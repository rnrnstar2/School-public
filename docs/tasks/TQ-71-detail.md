## 🔴 根本原因発見（フィードバック2）

### リダイレクトルートの問題

**現状:**
1. `/plan/lesson/[lessonId]/page.tsx` が `/lessons/[lessonId]` にリダイレクト
2. **しかし `/lessons/[lessonId]` ページが存在しない！**

**コード:**
```typescript
// apps/web/src/app/(app)/plan/lesson/[lessonId]/page.tsx
const redirectPath = searchParams.taskId
  ? `/planner/lesson/${params.lessonId}?taskId=${searchParams.taskId}`
  : `/lessons/${params.lessonId}`  // ← このパスが存在しない！

return redirect(redirectPath)
```

**問題:**
- `taskId` がない場合、存在しない `/lessons/[lessonId]` にリダイレクト
- これが「Undefinedにリダイレクト」の原因

**修正案:**
- `/lessons/[lessonId]` を作成
- または `/plan/lesson/[lessonId]` を直接レンダリングページにする

## 根本原因の仮説

### 仮説1: プロンプトの問題
AIプラン生成プロンプトが「目的を決める」タスクを生成することを禁止していない。

### 仮説2: ヒアリング結果の未活用
ヒアリングで収集した「目的」「経験」「素材状況」がプラン生成に渡されていない。

### 仮説3: lesson-libraryの問題
lesson-library.tsに「目的を決める」タスクが存在し、常に最初に選ばれるようになっている。

### 仮説4: デフォルトパスの問題
「経験なし」ユーザーのデフォルトパスが「目的を決める」から始まるようハードコードされている。

---

## 修正方針

### 1. プラン生成プロンプト修正
```
最初のタスクは必ず具体的なアクションにすること。
「目的を決める」「範囲を固める」「計画を立てる」のような抽象タスクは生成しない。
ユーザーがすぐに手を動かせるタスクを提示すること。

ヒアリングで目的が明確な場合、その情報を使って最初のタスクを決定すること。
```

### 2. ヒアリング結果の活用
- ヒアリング結果をプラン生成APIに必ず渡す
- 目的・経験・素材状況を最初のタスク決定に使う
- 「経験なし」の場合はツール選択→インストールから開始

### 3. lesson-library.tsの修正
- 「サイトの目的を決める」タスクを削除または条件付きにする
- ヒアリング完了後はこのタスクをスキップするロジックを追加

### 4. first-step-determinerの実装確認
- ヒアリング結果から最適な最初のステップを決定するロジックが正しく動作しているか確認
- Directive 23の実装状況を確認
