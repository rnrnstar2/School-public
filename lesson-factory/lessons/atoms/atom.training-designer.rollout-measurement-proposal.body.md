# 展開計画と効果測定案を提案する

あなたが作った学習プログラムを、組織全体に広げるための「展開計画」と、その成果を確かめるための「効果測定案」を、AIツールを使って15分でまとめます。

たとえて言うなら、展開計画は「料理のレシピ」です。材料（誰が参加するか）、手順（いつ何をするか）、味見のタイミング（途中で確認するポイント）がセットになっています。効果測定は「食レポ」です。「美味しかった」だけでなく、「何が良くて、次はどう改良するか」を具体的に記録するものです。

このレッスンが終わると、上司やチームに共有できる「展開計画＋効果測定案」の1枚文書が手元に残ります。

## 前提を確認する

このレッスンを始める前に、次の2つを確認してください。

- **AIチャットツールが使える状態にある** — ChatGPT（chat.openai.com）、Claude（claude.ai）、またはCursor内のAIチャットなど、どれか1つをブラウザやアプリで開けばOKです
- **広めたい学習プログラムの概要を口頭で説明できる** — 「誰向けに」「何を学ぶ」「どれくらいの期間で」の3点をざっくり言えれば十分です

特別なソフトのインストールは不要です。

## ステップ1：展開計画の骨組みをAIに作ってもらう

まずはAIに「たたき台」を作ってもらいましょう。次のプロンプト（＝AIへの指示文）をそのままコピーして、AIチャットに貼り付けてください。

```
あなたは研修設計のプロです。以下の学習プログラムを組織に展開する4週間の計画テンプレートを作ってください。

【プログラム概要】
（ここにあなたのプログラムの説明を1〜3行で書く）

以下を含む表形式で出力してください：
- 対象者（誰が参加するか）
- 各週の具体的な目標とアクション
- 各週のチェックポイント（確認すること）
- 担当者の役割
```

> **ポイント**: 「（ここに〜書く）」の部分を、あなたの実際のプログラムに書き換えてから送信してください。

AIが返してきた表をそのまま使わず、あなたの現場の事情に合わせて修正します。

### 良い例
- 「第1週：営業部パイロットチーム10名で試行。金曜に15分のふりかえりミーティングを実施」
- 「第2週：パイロットの声を反映して教材を修正。全体展開のスケジュールを確定」

### 悪い例
- 「第1週：準備する」→ 誰が何を準備するのか不明
- 「第2週：実施する」→ 具体的に何をするのか分からない

## ステップ2：効果測定の指標をAIと一緒に選ぶ

展開計画ができたら、次は「何をもって成功と言うか」を決めます。料理で言えば「味見の基準」を先に決めておくイメージです。

次のプロンプトをAIに送ってください。

```
以下の学習プログラムの効果を測る指標（KPI）を5つ提案してください。
条件：
- 数値や回数で測れるものを優先する
- 専門知識がなくても集計できるもの
- 短期（1ヶ月以内）と中期（3ヶ月後）の両方を含める

【プログラム概要】
（ステップ1と同じ説明を貼り付ける）
```

AIの提案から、**あなたの現場で実際に測定できるものを2〜3個だけ選びます**。全部を測ろうとすると手が回らなくなるので、絞ることが大切です。

### 良い例（数値で測れる）
- 「プログラム参加者の80%が、業務でAIツールを週3回以上使用している（参加1ヶ月後にアンケートで測定）」
- 「参加者の自己評価スコア（5点満点）が受講前と比べて平均0.5点以上向上」

### 悪い例（測定できない）
- 「みんなが満足する」→ どうやって測るか不明
- 「AIスキルが向上する」→ 何をもって向上とするか分からない

## ステップ3：一枚の提案文書にまとめる

展開計画と効果測定案を1つの文書にまとめます。AIに整形を頼みましょう。

```
以下の「展開計画」と「効果測定指標」を、上司に共有できる1枚の提案文書にまとめてください。

構成：
1. 目的（1〜2文）
2. 展開スケジュール（表形式）
3. 効果測定の指標と測定方法（箇条書き）
4. 想定リスクと対策（2〜3個）

【展開計画】
（ステップ1で作った表を貼り付ける）

【効果測定指標】
（ステップ2で選んだ2〜3個を貼り付ける）
```

## ステップ4：完成した文書をチェックする

まとめた文書が次のすべてを満たしているか、1つずつ確認してください。

- [ ] **対象者が明記されている**（「営業部の新入社員20名」のように具体的）
- [ ] **スケジュールに週単位の目標がある**（各週で何をするか書かれている）
- [ ] **チェックポイントが設定されている**（途中で「うまくいっているか」を確認するタイミングがある）
- [ ] **効果測定の指標が2つ以上ある**
- [ ] **指標が数値や回数で測定できる**（「〇〇が△△%以上」のような形）
- [ ] **測定方法が書かれている**（アンケート、ログ集計など、どうやって測るかが明記）

1つでも抜けていたら、AIに「〇〇が足りないので追加して」と頼んで補いましょう。

## つまずきやすいポイントと対策

| よくあるつまずき | 具体的な対策 |
|---|---|
| 計画が抽象的で行動に移せない | 各週の目標を「誰が・何を・いつまでに」の3点セットで書く |
| 指標を5個以上選んでしまう | 最初は2〜3個に絞る。慣れてから追加すればOK |
| AIの提案をそのまま採用してしまう | 必ず「自分の現場で本当に測定できるか？」と確認する |
| スケジュールが詰め込みすぎ | 4週間計画なら1週間の余白（バッファ）を入れる |
| 上司に見せたら「で、結局いくらかかるの？」と聞かれた | コスト項目（AIツール利用料、研修時間の工数）も1行追加しておくと安心 |

> **ヒント**: つまずいたときは、AIに「この計画の弱い部分を3つ指摘して、改善案を出して」と聞くと、自分では気づけない穴を見つけてくれます。
