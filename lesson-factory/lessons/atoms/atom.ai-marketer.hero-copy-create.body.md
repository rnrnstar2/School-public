# LPファーストビューの文案をつくる

LP（ランディングページ＝商品やサービスを紹介する1枚の Web ページ）を開いたとき、最初に目に飛び込んでくる部分を「ファーストビュー」と呼びます。お店でいえばショーウィンドウのようなもので、ここで「自分に関係がありそうだ」と思ってもらえないと、読者はすぐにページを閉じてしまいます。

この Atom では、AI チャット（ChatGPT や Claude など）に適切な指示を出して、ファーストビューに必要な 3 つの文案――キャッチコピー・サブコピー・CTA ボタン文言――を 15 分以内に仕上げます。

## 理解する：ファーストビューに必要な 3 つの要素

ファーストビューの文案は、大きく 3 つのパーツに分かれます。

1. **キャッチコピー**：一番大きな文字で表示される一文。読者の悩みや願望をズバリ言い当てる役割です
2. **サブコピー**：キャッチコピーの下に置く 1〜2 行の補足。「どうやって？」「なぜ？」を短く伝えます
3. **CTA（コール・トゥ・アクション）ボタン文言**：「無料で試す」「資料をダウンロード」など、読者に次の行動を促すボタンの言葉です

![ファーストビュー構成図](/lesson-assets/atom.ai-marketer.hero-copy-create/diagram.png)

たとえばカフェの新メニューを宣伝する LP なら、こんなイメージです。

- キャッチコピー：「朝 5 分で、一日が変わるスムージー」
- サブコピー：「管理栄養士監修・季節のフルーツを毎月届けます」
- CTA：「初回半額で始める」

## 準備する：AI に渡す情報を整理する

AI に良い文案を書いてもらうコツは、「誰に・何を・なぜ」を先に整理しておくことです。料理でいえば材料を切っておく「下ごしらえ」にあたります。

以下の 4 項目をメモ帳やテキストファイルに書き出してください。

| 項目 | 書くこと | 例 |
|------|----------|----|
| ターゲット | どんな人に届けたいか | 30代の働くママ |
| 商品・サービス | 何を提供するか | 時短スムージーの定期便 |
| ベネフィット | 読者が得られる嬉しい結果 | 朝の準備が5分短縮できる |
| 差別化ポイント | 他と何が違うか | 管理栄養士が毎月レシピ監修 |

**良い例**：「30代の働くママが、朝の準備を5分短縮できる時短スムージー定期便。管理栄養士が毎月レシピを監修している」

**悪い例**：「スムージーを売りたい」（情報が少なすぎて AI が的確な文案を出せません）

## つくる：AI にプロンプトを送って文案を生成する

ここからが本番です。先ほど整理した情報を使って、AI に具体的な指示（プロンプト）を送ります。

### ステップ 1：プロンプトを組み立てる

以下のテンプレートをコピーして、【 】の部分を自分の情報に書き換えてください。

```
あなたはLP専門のコピーライターです。
以下の情報をもとに、LPファーストビューの文案を3パターン作ってください。

■ ターゲット：【30代の働くママ】
■ 商品・サービス：【時短スムージーの定期便】
■ ベネフィット：【朝の準備が5分短縮できる】
■ 差別化ポイント：【管理栄養士が毎月レシピを監修】

各パターンに以下を含めてください：
- キャッチコピー（15文字以内）
- サブコピー（40文字以内）
- CTAボタン文言（10文字以内）

文体は親しみやすく、専門用語は使わないでください。
```

### ステップ 2：AI チャットに送る

ChatGPT、Claude、または普段使っている AI チャットを開いて、上のプロンプトをそのまま貼り付けて送信します。

### ステップ 3：結果を受け取って選ぶ

AI が 3 パターン返してくれたら、次の基準でベストを選びます。

- **ターゲットの言葉になっているか**：専門用語や業界用語が混ざっていないか
- **ベネフィットが伝わるか**：「何ができるようになるか」が一文で分かるか
- **文字数は守れているか**：キャッチコピーが長すぎるとスマホで見切れます

![AI出力の選定画面](/lesson-assets/atom.ai-marketer.hero-copy-create/screen_capture.png)

**良い例**：「朝5分、キレイが届く」（短く、ベネフィットが伝わる）

**悪い例**：「管理栄養士監修の厳選オーガニックフルーツを使用した健康スムージー定期便サービス」（長すぎ・特徴の羅列になっている）

## 磨く：AI に改善を依頼する

最初の出力がそのまま完成形になることはほとんどありません。「もう一声」を AI に頼むのが上手な使い方です。

改善プロンプトの例：

```
パターン2のキャッチコピーが気に入りました。
これをベースに、以下の方向で3つバリエーションを出してください。
- もっと感情に訴える表現にする
- 「朝」というキーワードは残す
- 数字を入れて具体性を高める
```

このように「どこが良かったか」「どう変えたいか」を具体的に伝えると、AI の精度がぐっと上がります。

## 仕上げる：成果物をまとめる

最終的に選んだ文案を、以下のフォーマットで 1 つのドキュメントにまとめてください。これがこの Atom の成果物（＝あなたが作った完成品）です。

```markdown
# LP ファーストビュー文案

## ターゲット
（ここに書く）

## キャッチコピー
（ここに書く）

## サブコピー
（ここに書く）

## CTA ボタン文言
（ここに書く）

## 選んだ理由（一言メモ）
（なぜこの文案を選んだかを 1〜2 行で）
```

## 確認する：セルフチェックリスト

提出前に、以下を自分で確認しましょう。

- [ ] キャッチコピーは 15 文字以内に収まっているか
- [ ] サブコピーは 40 文字以内か
- [ ] CTA ボタン文言は 10 文字以内か
- [ ] ターゲットが読んで「自分のことだ」と感じられる言葉になっているか
- [ ] 専門用語や業界用語が混ざっていないか

## つまずきやすいポイント

| よくある問題 | 対処法 |
|-------------|--------|
| AI の出力が長すぎる | プロンプトに「〇文字以内」と文字数制限を明記する |
| どのパターンも似たり寄ったり | 「感情訴求」「数字訴求」「疑問形」など切り口を指定して再生成する |
| 自分の商品がまだ決まっていない | 架空の商品（例：猫カフェの会員サービス）で練習しても OK |
| キャッチコピーが説明的になる | 「特徴ではなくベネフィットで書いて」と AI に指示する |
