# 執筆用プロンプトテンプレート集を作る

料理をするとき、レシピノートがあると便利ですよね。「カレーを作るときは玉ねぎを炒めてから…」のように、手順をまとめたカードがあれば、毎回ゼロから考え直す必要がありません。

AI に文章を書いてもらうときも同じです。「ブログ記事を書いて」とそのまま伝えるより、目的や読者、長さなどをまとめた「ひな形（テンプレート）」を用意しておけば、いつでも安定して良い文章が得られます。

このレッスンでは、あなたがよく書く文章のパターンごとにプロンプト（＝AI への指示文）のテンプレートを作り、1つのまとまった「テンプレート集」を完成させます。最終的に、3種類以上のテンプレートが入った1つのドキュメントがあなたの手元に残ります。

![テンプレート集の全体像](/lesson-assets/atom.ai-writer.prompt-template-library/diagram.png)

## 前提を確認する

このレッスンを進める前に、次の準備が整っていることを確認してください。

- ChatGPT、Claude などの AI チャットツール（＝文章で会話しながら指示を出せる AI サービス）を使える状態にしておく
- メモ帳、Google ドキュメント、Notion など、文章を書き残せる場所がある

プログラミングの知識は一切不要です。

## 書く文章のパターンを洗い出す

まずは、あなたが普段よく書く文章の種類を 5 個以上リストアップしましょう。手書きのメモでも構いません。

### 良い例

- 商品紹介のブログ記事
- メルマガ（＝メールで届くお知らせマガジン）の本文
- SNS の告知文（Instagram・X の投稿テキスト）
- お客様への返信メール
- セミナー告知ページの文章

### 悪い例

- 「なんか文章全般」…範囲が広すぎてテンプレートにまとまりません
- 「いい感じの文章」…何が「いい感じ」かが人によって違うので AI も困ります

コツは「1つのテンプレートで1つの目的」に絞ることです。種類が多くても、1つずつ作ればOKです。

> **AI に手伝ってもらうコツ**: 思いつかない場合は、AI に聞いてみましょう。以下のプロンプトをそのままコピーして送ってください。
>
> ```
> 私は（あなたの仕事や活動を書く）をしています。
> 普段よく書きそうな文章の種類を10個リストアップしてください。
> ```

## テンプレートの構成要素を理解する

プロンプトのテンプレートには、次の **5つの要素** を含めると、AI の回答品質が安定しやすくなります。

| 要素 | 役割 | 記入例 |
|---|---|---|
| **役割** | AI に「誰として振る舞うか」を伝える | 「プロの料理ブロガーとして」 |
| **目的** | どんな文章を書きたいか | 「初心者向けのレシピ紹介記事」 |
| **読者** | 誰に向けた文章か | 「料理を始めたばかりの20代女性」 |
| **条件** | 長さやトーンなどの制約 | 「800文字以内、親しみやすい文体、絵文字なし」 |
| **形式** | 出力の構造や見せ方 | 「見出し3つ＋箇条書きのまとめ」 |

「役割」と「目的」だけでもテンプレートは作れますが、「読者」「条件」「形式」を加えるほど、あなたのイメージに近い文章が返ってきます。

## テンプレートを1つ書いてみる

メモ帳や Google ドキュメントを開いて、次のひな形をコピーしてください。

```text
## （文章の種類）用テンプレート

【役割】（AI に演じてほしい人物像）
【目的】（どんな文章を作りたいか）
【読者】（この文章を読む人はどんな人か）
【条件】（文字数、文体、禁止事項など）
【形式】（見出し構成、箇条書き、段落数など）
```

`（ ）` の部分を、あなたの実際の用途に合わせて埋めましょう。

### 良い例

```text
## 商品紹介ブログ用テンプレート

【役割】料理好きの主婦ブロガー
【目的】新しい調理器具の良さを伝える紹介記事
【読者】30〜40代の料理初心者
【条件】600〜800文字、親しみやすい文体、専門用語禁止
【形式】導入（共感）→ 商品の特徴3つ → まとめ（読者への一言）
```

### 悪い例

```text
## テンプレート

ブログ書いて。お願いします。
```

→ 役割や条件がないため、AI の回答が毎回バラバラになります。

## AI でテンプレートを試す

テンプレートが1つ書けたら、実際に AI ツールで試しましょう。ここでは ChatGPT と Claude それぞれの使い方を示します。どちらか一方だけでもOKです。

### ChatGPT で試す場合

1. [ChatGPT](https://chat.openai.com) を開く
2. チャット欄に、作ったテンプレートの `（ ）` を埋めたものを貼り付ける
3. 送信ボタンを押す
4. 出てきた文章を読んで、期待通りか確認する

### Claude で試す場合

1. [Claude](https://claude.ai) を開く
2. メッセージ欄にテンプレートを貼り付けて送信する
3. 結果を読んで確認する

![AIツールでテンプレートを試している画面](/lesson-assets/atom.ai-writer.prompt-template-library/screen_capture.png)

### 結果がイマイチだったときの直し方

AI の返答が期待と違った場合は、次のように条件を足して再送信してください。

| 困りごと | 追加する条件の例 |
|---|---|
| 文章が長すぎる | 「400文字以内で書いてください」 |
| 硬すぎる・堅苦しい | 「友だちに話すようなカジュアルな文体で」 |
| 具体性が足りない | 「数字や具体的な商品名を入れてください」 |
| 構成がバラバラ | 「形式」に見出しの数と順番を書く |

2〜3回やり取りして納得のいく結果が出たら、**最終版のテンプレートをメモ帳に保存**してください。この「調整して保存」の流れがテンプレート作りの基本です。

> **AI に改善点を聞くプロンプト例**:
>
> ```
> 上の結果を踏まえて、私のプロンプト（指示文）をもっと良くするには
> どこを変えればいいですか？ 改善案を3つ教えてください。
> ```

## テンプレートを3つ以上にまとめる

同じ手順で、文章の種類ごとにテンプレートを追加していきましょう。目標は **3つ以上** です。

すべて作り終えたら、1つのファイルにまとめます。見出しで種類別に分けておくと、後から探しやすくなります。

```text
# 執筆用プロンプトテンプレート集

---

## 1. ブログ記事用

【役割】…
【目的】…
【読者】…
【条件】…
【形式】…

---

## 2. SNS 告知用

【役割】…
【目的】…
【読者】…
【条件】…
【形式】…

---

## 3. メルマガ用

【役割】…
【目的】…
【読者】…
【条件】…
【形式】…
```

> **AI にテンプレートの叩き台を作ってもらうプロンプト例**:
>
> ```
> 以下の文章の種類ごとに、プロンプトテンプレートの叩き台を作ってください。
> 各テンプレートには【役割】【目的】【読者】【条件】【形式】を含めてください。
>
> 1. ブログ記事
> 2. SNS投稿文
> 3. メルマガ本文
> ```
>
> AI が出した叩き台を、あなたの仕事に合わせて修正すれば時短になります。

## 完成を確認する

次のチェックリストで最終確認しましょう。すべてにチェックが入れば、このレッスンは完了です。

- [ ] テンプレートが **3つ以上** 揃っている
- [ ] 各テンプレートに5つの要素（役割・目的・読者・条件・形式）が含まれている
- [ ] 各テンプレートを実際に AI に送って、期待に近い文章が返ってきた
- [ ] すべてのテンプレートが **1つのファイル（ドキュメント）** にまとまっている
- [ ] ファイルを見返したとき、どのテンプレートが何用かすぐわかる見出しがついている

## つまずきやすいポイント

| つまずき | 原因 | 対策 |
|---|---|---|
| テンプレートが長くなりすぎる | 条件を詰め込みすぎている | 1テンプレート150文字以内を目安にする。細かい指示は別の会話で伝える |
| AI の返答が毎回違う | 条件や形式があいまい | 「800文字以内」「見出し3つ」のように数字で具体的に指定する |
| どの文章種類を選べばいいかわからない | 普段の作業を振り返っていない | 直近1週間に書いた文章を思い出す。メールの送信履歴やSNS投稿を見返すのも有効 |
| ChatGPT と Claude で結果が違う | AIツールごとに得意分野が異なる | どちらか気に入った方に統一するか、用途ごとに使い分ける。テンプレート自体はどちらでも使えます |
| テンプレートの「読者」が書けない | 読む人を具体的にイメージできていない | 実在する1人（お客様、友人など）を思い浮かべて「その人に向けて書く」と考える |
