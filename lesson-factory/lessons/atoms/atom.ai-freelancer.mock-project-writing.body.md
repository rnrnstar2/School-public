# 模擬案件でライティング納品を実践する

あなたはこれから、本物の案件を模した「模擬ライティング案件」に取り組みます。

これは、スポーツの「練習試合」のようなものです。本番の試合（＝実際のクライアントからの依頼）に出る前に、試合形式の練習をして流れを体に覚えさせるイメージです。この1回の模擬案件をやりきるだけで、「自分にもできそうだ」という手応えが得られます。

![模擬案件の全体フロー](/lesson-assets/atom.ai-freelancer.mock-project-writing/diagram.png)

## 前提を確認する

このレッスンを始める前に、次の2つが手元にあるか確認してください。

- **AIチャットツール**: ChatGPT・Claude・Gemini など、どれか1つをブラウザで開けること（無料版で十分です）
- **テキスト保存先**: Googleドキュメント・メモ帳・Notionなど、文章をコピペして保存できる場所

特別なソフトのインストールは不要です。ブラウザだけで最後まで完結します。

## ステップ1: 模擬案件のお題を受け取る

まずは「依頼内容」を確認しましょう。今回のお題はこちらです。

> **お題**: 「初心者向けに、毎朝5分でできる簡単ストレッチを3つ紹介するブログ記事（800文字程度）」

これは、クラウドソーシング（＝インターネット上で仕事の依頼と受注ができる仕組み。ランサーズやクラウドワークスが有名です）でよく見かけるライティング案件と同じ形式です。

### 良い依頼と悪い依頼を見分ける

実際の案件でも「依頼内容が具体的かどうか」は大事なポイントです。

- **良い例**: 「初心者向け・800文字・ストレッチ3つ・語りかけ口調」→ 具体的な条件があるので、何を書けばいいか迷わない
- **悪い例**: 「ストレッチについて書いて」→ 条件がないので、何をどう書いたらいいかわからない

今回のお題は「良い例」に近い形にしてあります。安心して取り組んでください。

## ステップ2: AIに指示を出して原稿を書く

AIチャットツールを開いて、次の文章をそのままコピペして送信してください。

```
次の条件でブログ記事を書いてください。

【テーマ】毎朝5分でできる簡単ストレッチを3つ紹介
【読者】運動習慣のない初心者
【文字数】800文字程度
【文体】です・ます調、親しみやすい語りかけ口調
【構成】タイトル → 導入 → ストレッチ3つ（各ストレッチに名前・やり方・ポイント）→ まとめ
```

これは「レストランで注文する」のと同じです。「ラーメン」とだけ伝えるより、「味噌ラーメン、麺かため、ネギ多め」と伝えたほうが思い通りの一杯が出てきますよね。AIへの指示も、具体的であるほど良い結果が返ってきます。

### なぜこの指示の書き方がいいのか

- **良い指示**: テーマ・読者・文字数・文体・構成がすべて明記されている → AIが迷わず書ける
- **悪い指示**: 「ストレッチの記事を書いて」だけ → 出てくる文章が毎回バラバラになる

> 💡 **AIツール別のコツ**
> - **ChatGPT**: 上の指示をそのまま貼り付ければOKです
> - **Claude**: 同じ指示で動きます。長文を頼むときは「箇条書きを使って見やすく」と追加すると読みやすくなります
> - **Gemini**: 同様に使えます。「日本語で」と明示すると確実です

![AIに指示を送る画面の例](/lesson-assets/atom.ai-freelancer.mock-project-writing/screen_capture.png)

## ステップ3: 原稿をチェックして修正する

AIが出力した文章を、次の3つの視点で確認しましょう。

| チェック項目 | 確認すること | OKの基準 |
|---|---|---|
| 文字数 | 長すぎ・短すぎでないか | 600〜1,000文字に収まっている |
| 読みやすさ | 専門用語を使いすぎていないか | 中学生が読んでも意味がわかる |
| 指示どおり | 3つのストレッチがちゃんとあるか | タイトル・やり方・ポイントが各ストレッチにある |

### 修正をAIにお願いする方法

気になる部分があったら、具体的に伝えて直してもらいましょう。以下はコピペして使える修正指示の例です。

**文字数を調整したいとき:**
```
全体を800文字ちょうどに収まるように調整してください。
```

**一部を書き直したいとき:**
```
2つ目のストレッチの説明が長すぎるので、2文以内に短くしてください。
```

**トーンを変えたいとき:**
```
もう少しカジュアルに、友達に話しかけるような口調に変えてください。
```

![原稿チェックの3ポイント](/lesson-assets/atom.ai-freelancer.mock-project-writing/diagram.2.png)

> 💡 **よくある修正パターン**: 1回目の出力で完璧になることはまずありません。2〜3回やりとりして仕上げるのが普通です。プロのライターも編集者と何度もやりとりして原稿を仕上げます。

## ステップ4: 納品形式にまとめる

原稿が仕上がったら、次の手順で「納品物」（＝クライアントに渡す完成品）にまとめます。

1. Googleドキュメント（またはお好みのテキストエディタ）を新しく開く
2. 完成した原稿をコピペして貼り付ける
3. 以下のチェックリストで最終確認する

### 納品前の最終チェックリスト

- [ ] タイトルがついているか
- [ ] 見出しで内容が区切られているか
- [ ] 誤字脱字がないか（AIの出力にもたまに誤字があります）
- [ ] 指定された文字数の範囲内か
- [ ] 読み返して意味が通るか

実際の案件では、このファイルの共有リンクをクライアントに送って「納品完了」になります。

## ステップ5: 振り返る

ここまでの流れを振り返りましょう。あなたは次の4ステップを完了しました。

1. **お題を確認した** — 依頼内容を読み、何を求められているかを理解した
2. **AIに具体的な指示を出した** — テーマ・読者・文字数・文体・構成を明記してプロンプトを送った
3. **原稿をチェック・修正した** — 文字数・読みやすさ・指示どおりかの3点で確認した
4. **納品形式にまとめた** — ドキュメントに貼り付け、最終チェックを行った

この4ステップが、実際のライティング案件の基本パターンです。今回の模擬案件をやりきったあなたは、もう本番の案件でも同じ流れで進められます。

![お題確認→AI指示→チェック修正→納品の4ステップ](/lesson-assets/atom.ai-freelancer.mock-project-writing/diagram.3.png)

## つまずきポイントと解決法

| よくある困りごと | 解決法 |
|---|---|
| AIの出力が長すぎる・短すぎる | 指示に「800文字程度で」と文字数を明記する。出力後に「もう少し短く」「あと200文字足して」と追加指示する |
| 文章のトーンが堅すぎる・軽すぎる | 「です・ます調で」「友達に話すように」など語尾や雰囲気の指定を追加する |
| 何から始めればいいか分からない | まずは上のお題とプロンプトをそのままコピペして送信する。出てきた文章を読むところから始めればOK |
| AIが的外れな内容を返してくる | 指示の【構成】部分を具体的にする。「タイトル→導入→本文→まとめ」のように骨組みを伝える |

## 確認する: あなたの成果物をチェック

以下がすべて「はい」なら、このレッスンは完了です。

1. AIチャットツールで、お題に沿ったブログ記事の原稿を1本作成できた
2. 最低1回はAIに修正指示を出して、原稿を改善した
3. 完成した原稿をGoogleドキュメントなどに保存した
4. 保存したドキュメントのスクリーンショットを撮れた（これが証拠になります）
