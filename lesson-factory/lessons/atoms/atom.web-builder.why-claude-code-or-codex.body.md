# Claude Code / Codex を使う理由を決める

## このレッスンで身につくこと

料理をするとき、すべてを一人でこなすのもよいですが、便利な道具や助手がいればずっとラクにおいしいものが作れますよね。Webアプリづくりも同じです。AI（＝人工知能で、あなたの指示を理解して作業を手伝うプログラム）を「料理の助手」として活用すると、プログラミング経験がないあなたでも本格的なWebアプリを作れるようになります。

このレッスンを終えると、次の3つができるようになります。

- Claude Code と Codex CLI の違いを自分の言葉で説明できる
- あなたの作業スタイルに合った道具を一つ選べる
- 「なぜAIを使うのか」を自分の言葉で書き出せる

**所要時間**: 約10〜15分

## AI道具を知る

まずは、この学習パスで使う2つのAI道具を紹介します。どちらも「ターミナル（＝文字だけでコンピューターに指示を出す画面）」で動くAIの助手です。

### Claude Code（クロード・コード）

Anthropic社が作ったAI「Claude」を、あなたのパソコン上で動かせる道具です。ターミナルの中で対話しながら、コードを書いたり直したりしてくれます。

- **特徴**: 日本語で話しかけると、丁寧に返事しながら作業を進めてくれる
- **向いていること**: 対話しながら少しずつ作り上げたいとき
- **たとえるなら**: 隣にいる料理の先生。「次は何をしますか？」と一つずつ確認しながら一緒に作る

### Codex CLI（コーデックス・シーエルアイ）

OpenAI社が作ったAIを、同じくターミナルから使える道具です。コマンド（＝コンピューターへの短い指示文）一つで、まとまった作業を一気に進めてくれます。

- **特徴**: 指示を与えると自動でバシバシ作業を進める
- **向いていること**: ある程度方針が決まっていて一気に進めたいとき
- **たとえるなら**: レシピを渡すと全部作ってくれるシェフ。完成品をドンと出してくれる

![ターミナルでの操作イメージ](/lesson-assets/atom.web-builder.why-claude-code-or-codex/screen_capture.png)

## どちらを選ぶか考える

どちらも「AIがコードを書く」という点では同じですが、進め方の好みに合わせて選べます。

### 選び方のポイント

| あなたの好み | 向いている道具 |
|---|---|
| 対話しながら確認しながら進めたい | Claude Code |
| 方針が決まったら一気に進めたい | Codex CLI |
| どちらも試してみたい | まずは Claude Code から |
| とにかく早く成果物がほしい | Codex CLI |

### 良い例と悪い例

**良い例**: 「まずはWebアプリを1つ完成させたい。プログラミングは初めてなので、対話しながら進められる Claude Code を選ぼう」

**悪い例**: 「どっちでもいいからとりあえず入れてみる」（→ 途中で「自分に合っていなかった」と感じて挫折しやすい）

**良い例**: 「Codex CLI の方が効率的らしいけど、まずは仕組みを理解したいから Claude Code で始めて、慣れたら切り替えよう」

**悪い例**: 道具選びだけで満足して、実際に使い始めない

## AIに聞いてみる（プロンプト例）

まだ迷っている場合は、AIに相談してみましょう。ChatGPTやClaudeの無料版（ブラウザで使えます）に、次のように聞くことができます。

> **プロンプト例 1**（自分に合う道具を聞く）:
> 「私はプログラミング経験がない初心者で、Webアプリを作りたいです。Claude Code と Codex CLI のどちらが向いているか、私の状況に合わせて教えてください」

> **プロンプト例 2**（違いをもう少し詳しく知りたいとき）:
> 「Claude Code と Codex CLI を使ってWebアプリを作る場合、それぞれの作業の流れを具体的に教えてください。プログラミング用語は使わず、日常の言葉で説明してください」

AIの答えを読んで、「こっちの方が自分に合いそうだな」と思えたらOKです。

## あなたの理由を書き出す

最後に、「なぜAIの力を借りるのか」をあなた自身の言葉で書き出してみましょう。メモ帳やノートアプリに、次の3つを書いてみてください。

### 書くこと

1. **あなたが作りたいもの**（例: 「自分のポートフォリオサイト」「カフェの予約ページ」）
2. **選んだ道具**（Claude Code または Codex CLI）
3. **その道具を選んだ理由**（1〜2文でOK）

### 書き方の例

> 私は自分のポートフォリオサイトを作りたい。プログラミングは初めてなので、日本語で話しかけながら進められる Claude Code を選んだ。対話しながらだと、一つずつ理解しながら作れると思ったから。

これがこのレッスンの成果物（＝あなたが完成させるもの）です。

## つまずきやすいポイント

| よくあるつまずき | 対処法 |
|---|---|
| 「どっちがいいか全然わからない」 | 迷ったらまず Claude Code を選びましょう。対話型なので初心者に向いています。あとから変更もできます |
| 「理由がうまく書けない」 | 完璧な文章でなくて大丈夫。「○○を作りたいから」の一言だけでもOKです |
| 「そもそもAIを使う必要があるのかわからない」 | AI無しだとプログラミング言語を一から学ぶ必要があります。AIを使えば日本語の指示だけでコードが書けるので、最初の一歩が格段にラクになります |

## 確認する

次の3つすべてに「はい」と言えたら、このレッスンは完了です。

- [ ] Claude Code と Codex CLI のそれぞれの特徴を、友だちに説明できる
- [ ] あなたが使う道具を一つ選んだ（迷ったら Claude Code でOK）
- [ ] 「作りたいもの」「選んだ道具」「選んだ理由」を書き出した
