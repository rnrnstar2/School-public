# v0 で UI を 1 ショット生成して Vercel に公開する

> **Status**: skeleton — `/lesson-improve` で本文化予定
>
> 想定 persona: `web-builder` / `ai-app-builder` / `p-noneng-webapp`（コードを書かずに画面を出したい層）

## なぜこれをやる？

「サイトを作りたい」と思った瞬間に、HTML / CSS / フレームワーク選定で止まる人が多い。
v0.dev は「作りたい画面」を日本語 1 文で書くと、現代的な React + Tailwind UI を 1 ショットで返す。
Vercel に直接 deploy できるので、初手で「自分の URL が世界に出る」体験まで一気に行ける。
コード理解は後からでよい。まず "動く何か" を画面に出すのが目的。

## 何を準備する？

- v0.dev のアカウント（GitHub ログインで OK、無料枠あり）
- Vercel のアカウント（GitHub 連携でほぼ自動）
- 作りたいものを 1 文で書いたメモ（例: 「フリーランスのポートフォリオ。トップ・実績・連絡先の 3 セクション。落ち着いた配色」）

## やってみよう

1. v0.dev にログインし、プロンプト欄に「作りたいもの」を 1 文で投入
2. 出てきた候補から 1 つ選び、配色・文言を 2-3 回プロンプトで微調整
3. 「Deploy to Vercel」ボタンを押し、Vercel 側でプロジェクト名を決める
4. ビルド完了後、`*.vercel.app` の URL を開いて手元の端末で表示確認
5. URL を README / SNS / メモに保存（卒業証明として使う）

## できたかチェック

- [ ] 自分専用の `*.vercel.app` URL がブラウザで開ける
- [ ] スマホでも崩れず表示される（responsive ベース）
- [ ] プロンプト 2-3 回で意図した雰囲気に近付けられた

## つまづいたら

- **「思った絵と違う」** → 1 つの prompt に詰め込まず「配色だけ」「セクションだけ」と分割して指示する
- **Vercel deploy が失敗** → v0 が出した依存関係を変更しないでそのままデプロイする。手で `package.json` を触らない
- **無料枠の上限** → 同じ画面で何度も生成しない。気に入った版を保存してから次へ
