# 長文コンテンツの構造を整理する

## このレッスンでできるようになること

このレッスンを終えると、あなたは**長い文章を書く前に、全体の「骨組み」をAIと一緒に整理して作れる**ようになります。

旅行の計画を思い浮かべてみてください。いきなり荷造りから始める人はいませんよね。まず「どこに行くか」「何日間か」「予算はいくらか」を決めて、それから詳細を詰めます。長い文章も同じです。いきなり書き始める前に、全体の構造（＝骨組み）を先に決めることで、内容がブレず、読みやすくなります。

![構造化フロー](/lesson-assets/atom.ai-writer.long-form-structuring/diagram.png)

## 前提条件を確認する

- AIチャットツール（ChatGPT、Claude、Geminiなど）をブラウザで開ける状態にしてください
- プログラミングの知識は一切不要です
- 書きたいテーマが1つあると進めやすいです（例：ブログ記事、社内報告書、メルマガなど）

## 全体の流れを把握する

このレッスンでは、次の4つのステップを進めます（目安：15分）。

1. **アイデアを書き出す**（3分）— 思いついたことをそのまま書き出します
2. **AIに構造化を頼む**（5分）— AIに書き出した内容を整理してもらいます
3. **見出しの階層を確認する**（4分）— 大見出し・中見出しのバランスを見直します
4. **最終チェックして保存する**（3分）— 構造が目的に合っているか確認して保存します

## 1. アイデアを書き出す

まず、あなたが書きたいことを自由に書き出しましょう。この時点では綺麗にまとめる必要はありません。箇条書きでも、長い文章でも構いません。

メモ帳、ノートアプリ、Googleドキュメントなど、好きな場所に書いてください。

**良い例：**
> 「初心者向けのブログ記事を書きたい。内容はAIを使った文章作成のコツ。読者は30代の会社員。伝えたいことは3つ。①AIは書くのが苦手な人の助けになる ②コツは具体的な指示を出すこと ③出てきた文章は必ず自分で読み直すこと」

**悪い例：**
> 「ブログ」だけ
> （情報が少なすぎて、AIも整理のしようがありません）

### なぜ書き出しが大事なのか

料理でいうと「冷蔵庫にある食材を全部テーブルに出す」作業です。材料が揃っているか確認してからでないと、レシピは考えられませんよね。書きたいことが3〜5個以上出ればOKです。

## 2. AIに構造化を頼む

AIチャットツールに、書き出したアイデアを渡して構造化を頼みます。次のプロンプト（＝AIへの指示文）をコピーして、AIの入力欄に貼り付けてください。

### 使うプロンプト例

```
以下のアイデアを、ブログ記事の構造（見出しの一覧）に整理してください。
条件：
- 読者は初心者です
- 大見出し（##）は3〜5個にしてください
- 各見出しは「読者が内容を想像できる具体的な言葉」にしてください
- 導入→本題→まとめの流れにしてください

[ここにステップ1で書き出したアイデアを貼る]
```

**何をしているか：** AIに「私のアイデアを読みやすい記事の構造に整理して」とお願いしています。条件を付けることで、AIが出す構造の質がぐっと上がります。

![AIに構造化を頼む画面](/lesson-assets/atom.ai-writer.long-form-structuring/screen_capture.png)

### AIの結果を読む

AIから返ってきた構造案を確認しましょう。

**良い構造の例：**
```
## AIで文章を書くってどういうこと？
## 知っておきたい3つのコツ
### 具体的な指示を出す
### 最初は短い文章から頼む
### 結果は必ず自分の目で読み直す
## まとめ：AIは「下書き係」として使おう
```
（読むだけで内容が想像できる見出しが並んでいる ✅）

**悪い構造の例：**
```
## 概要
## 詳細
## その他
## 結論
```
（内容が想像できず、読者が迷ってしまう ❌）

### AIの回答がイマイチだったら

一度で完璧な回答が返ってくるとは限りません。そのときは、追加で具体的に指示してみましょう。

**修正プロンプト例：**
```
ありがとうございます。次の点を修正してください。
- 「概要」「詳細」のような抽象的な見出しを、内容がわかる具体的な見出しに変えてください
- 見出しの数を5個以内に絞ってください
```

## 3. 見出しの階層を確認する

AIが提案した構造を見て、次の3つのポイントをチェックしましょう。

| チェック項目 | 確認のしかた |
|---------|------|
| 大見出し（##）は3〜5個か | 指で数えて確認する |
| 同じレベルの見出しに重複がないか | 似た言葉が2つ並んでいないか見る |
| 読む順番が自然か | 「導入→本題→まとめ」の流れになっているか確認する |

もしおかしな箇所があれば、AIに追加で指示を出せます。

**修正プロンプト例：**
```
見出し「AIを使う3つのコツ」の下に「具体的な指示を出す」がありますが、
「3つのコツ」自体が「具体的な指示」と同じ内容に見えます。
重複しないように整理してください。
```

**何をしているか：** AIに「この部分とこの部分がダブっているから直して」と伝えて、構造をより綺麗にしてもらっています。「どこが・なぜおかしいか」を伝えるのがコツです。

## 4. 最終チェックして保存する

構造が固まったら、次の3つの質問に「はい」と答えられるか確認します。

- ✅ この見出しの一覧だけで、記事の全体像が想像できるか？
- ✅ 読者が「知りたい」と思う順番になっているか？
- ✅ 一番伝えたいことが本題部分（真ん中あたり）にきているか？

全部「はい」なら、構造化は完成です！

### 仕上げの確認プロンプト

自信がないときは、AIにダブルチェックを頼めます。

```
以下の構造を読者目線でチェックしてください。
- 読む順番は自然ですか？
- 抜けている話題はありますか？
- 見出しは具体的でわかりやすいですか？

[完成した構造を貼る]
```

### 成果物を保存する

このレッスンの成果物は、**AIを使って作った記事の見出し構造（アウトライン）**です。

完成した構造をテキストファイルやメモアプリに保存してください。スクリーンショットを撮っておくと、あとで振り返るときに便利です。次のレッスンで、この構造に沿って本文を書いていきます。

## つまずきポイントと対策

| つまずき | 対策 |
|---------|------|
| AIが複雑すぎる構造を出す | 「大見出しは3〜5個に絞って」と追加指示する |
| 見出しが抽象的すぎる | 「読者が内容を想像できる具体的な見出しにして」と追加指示する |
| どの見出しが重要かわからない | 一番伝えたい見出しに「★重要」とマークを付けて整理する |
| AIに何を頼めばいいかわからない | まずは「ブログ記事の目次を作って」とシンプルに頼む |
| AIの回答が毎回違って迷う | 条件（見出し数・読者層・流れ）を具体的に指定すると安定する |
| 構造が正しいか自信が持てない | 仕上げの確認プロンプト（上記）でAIにダブルチェックを頼む |
