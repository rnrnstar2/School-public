# 既存教材の不足箇所を特定する

たとえば、あなたが料理教室の先生だとします。生徒に配るレシピ集を見直したとき、「卵アレルギーの生徒への代替材料が書いていない」「火加減の説明がない」と気づいたら、それは「不足箇所（ギャップ）」です。

この Atom では、あなたが関わる教材やマニュアルについて、同じように「これが足りない」という箇所を体系的に見つけ出す方法を学びます。AI ツール（＝人工知能を使った便利なソフト。ChatGPT や Claude など）を活用して、効率よく不足を見つけましょう。

![教材ギャップ特定フロー](/lesson-assets/atom.training-designer.content-gap-identification/diagram.png)

## 前提を確認する

この Atom に取り組む前に、次の2つを用意してください。

- **見直したい教材**が手元にある（テキスト、PDF、スライド、動画の台本など何でもOK）
- **ChatGPT または Claude** にアクセスできる（無料プランで十分です）

どちらかが用意できない場合は、まず教材を一つ選ぶところから始めましょう。社内マニュアルや研修資料など、普段使っているものがおすすめです。

## 教材全体をスキャンする

まず、教材の全体像をざっと把握します。料理でいえば「レシピを最初から最後まで一度通して読む」ステップです。

### やること

1. 教材の目次や見出しをすべて書き出す
2. 各見出しが「何を教えているか」を一文でまとめる
3. 学習者が最終的にできるようになる目標（ゴール）を確認する

### AI に手伝ってもらうプロンプト例

教材が長い場合、AI にまとめてもらうと効率的です。ChatGPT や Claude に次のように入力してください。

```
あなたは教材レビューの専門家です。
次の教材テキストの構成を分析し、以下の3点を整理してください。
1. 各章・セクションが教えている内容（一文ずつ）
2. この教材全体のゴール（学習者が最終的にできるようになること）
3. ゴールに対して構成上足りなさそうな部分

【教材テキスト】
（ここにテキストをコピー＆ペーストする）
```

**良い例:** 「第1章ではログインの方法、第2章では検索の方法を教えている。ゴールは『自分で情報を探せること』」

**悪い例:** 「なんとなく読んで、大体の雰囲気をつかむ」

## 学習者の視点で疑問を洗い出す

次に、教材を読んだ学習者が「ここでつまずきそう」と感じる箇所を探します。次の3つのチェック質問を教材の各セクションに当てはめてみてください。

- 「初めて聞く言葉は説明されているか？」
- 「手順は途中で飛んでいないか？（例：手順3の次がいきなり手順5になっていないか）」
- 「失敗したときの対処法は書いてあるか？」

### AI にチェックしてもらうプロンプト例

```
次の教材テキストを「完全な初心者」の目線で読んでください。
以下の観点で、つまずきそうな箇所を最大7つ挙げてください。
- 専門用語が説明なく使われている箇所
- 手順が飛んでいる（前提知識が暗黙になっている）箇所
- 失敗したときの対処法が書かれていない箇所
各箇所について「どこ」「なぜつまずくか」「改善案」の3点で答えてください。

【教材テキスト】
（ここにテキストをコピー＆ペーストする）
```

**良い例:** 「手順3から手順4への移行で、『保存ボタンを押す』という前提が省略されていると指摘された」

**悪い例:** 「AI が『全体的に良く書けています』と返しただけで終わる」→ この場合はプロンプトをもっと具体的にしましょう。「問題がない」と返ってきたら、「初めてパソコンを触る60代の方が読む場合、つまずくポイントはありませんか？」のように対象者を明示して聞き直してみてください。

## ゴールとの差を特定する

洗い出した疑問を、教材のゴールと照らし合わせます。「ゴールに到達するために必要だけど、教材に書かれていないこと」が不足箇所です。

1. ゴールを達成するために必要なステップを上から下まで書き出す
2. 各ステップが教材のどの箇所でカバーされているか確認する
3. カバーされていないステップを不足箇所としてリスト化する

### AI を使ったギャップ分析プロンプト例

```
この教材のゴールは「○○ができるようになること」です。
そのゴールを達成するために必要なステップを順番に書き出し、
各ステップが教材のどこでカバーされているか表にしてください。
カバーされていないステップがあれば「未カバー」と明記してください。

【教材テキスト】
（ここにテキストをコピー＆ペーストする）
```

## 不足箇所をまとめる

見つけた不足箇所を一つのドキュメントにまとめます。次の形式で記録すると、あとで見直しやすくなります。

| 箇所 | 不足している内容 | 影響度 | 優先度 |
|------|----------------|--------|--------|
| 第2章 | 用語「〇〇」の説明がない | 読者が先に進めなくなる | 高 |
| 第4章 | エラー時の対処法がない | 失敗時に自力復帰できない | 中 |
| 第1章 | 画像が古く現在の画面と違う | 混乱するが手順自体は合っている | 低 |

この表を Markdown（＝シンプルな記号ルールで文書を整形する書き方。メモ帳でも作れます）で書き、完成したらスクリーンショット（＝画面の写真）を撮って提出しましょう。

### 完成イメージ

あなたが作るドキュメントは、以下のような構成になります。

```
# 教材ギャップ分析レポート

## 対象教材: （教材名を入れる）
## 分析日: （日付）
## ゴール: （教材のゴール）

## 不足箇所一覧
（上の表を入れる）

## 改善の優先順位
1. （最優先の改善点とその理由）
2. （次に重要な改善点）
```

## つまずきやすいポイントを押さえる

- **「全体は問題ない」と思い込みがち** — 全体がきれいに見えても、一つずつ確認すると意外な穴があります。省略せず全セクションを確認しましょう
- **AI の指摘を鵜呑みにする** — AI が挙げた不足箇所も、あなたの目で「本当に不足か？」を確認してください。AI は文脈を完全には理解できないことがあります
- **優先度をつけない** — すべての不足を「同じくらい重要」と扱うと、後でどこから直せばよいか迷います。必ず高・中・低で優先度をつけましょう
- **教材が長すぎて AI に一度に渡せない** — 教材が長い場合は章ごとに分けて AI に渡してください。最後に結果をまとめれば OK です

## 確認する

以下の3つすべてに「はい」と言えたら、この Atom は完了です。

1. **教材のゴールと各セクションの対応表を作れた** — 各セクションが何をカバーしているか一覧にできている
2. **不足箇所を3つ以上見つけて表にまとめた** — 箇所・内容・優先度を含む表ができている
3. **AI ツールを使って分析した痕跡がある** — スクリーンショットに AI とのやりとりが写っている
