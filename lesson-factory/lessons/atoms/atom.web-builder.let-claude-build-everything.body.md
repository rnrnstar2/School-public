# Claude Code に「全部作って」と委譲する

> **Status**: skeleton — `/lesson-improve` で本文化予定
>
> 想定 persona: `web-builder` / `ai-app-builder` / `p-noneng-webapp`
> 代替ツール: Cursor / Codex CLI（同じ流儀で動く）

## なぜこれをやる？

非エンジニアが詰まる最大の原因は「細かい手順を AI に投げる粒度」。
Claude Code は "1 ファイルを直す" よりも "この機能を全部作る" 粒度の方が得意。
ここでは「最初の 1 画面を全部作って」を 1 つのプロンプトで委譲し、
人間は完成形を触ってフィードバックするだけ、という働き方を体験する。
他ツール（Cursor / Codex CLI）でも同じ流儀が通用する。

## 何を準備する？

- Claude Code（または Cursor / Codex CLI）が動く環境（インストール済みでよい、初期セットアップは別 atom）
- 「最初に出したい 1 画面」を 1 段落でメモ（誰が見る・何が出る・どうクリックする）
- 空のプロジェクトディレクトリ 1 つ

## やってみよう

1. プロジェクトディレクトリで Claude Code を起動し、最初のメッセージに 1 段落の要件を投入
2. 「全部任せる」ことを明示する（例: "Next.js + Tailwind + shadcn でこの 1 画面を完成形まで作って。質問があれば私に聞いて"）
3. AI が出してきたファイル変更をそのまま受け入れ、`pnpm dev` 等で起動
4. 動かない箇所はエラーメッセージごと AI に貼り付ける（自分で直さない）
5. 1 画面が動いたら、スクショと URL（localhost で OK）を成果物として保存

## できたかチェック

- [ ] 自分は 1 行もコードを書かずに 1 画面が動いた
- [ ] エラーは AI に丸投げで解決できた
- [ ] AI に「全部作って」と委ねるプロンプトのテンプレを 1 つ自分用に持てた

## つまづいたら

- **AI が止まって質問してくる** → そのまま日本語で答える。詰まったら "判断は任せる" でも OK
- **生成範囲が広すぎて壊れる** → "1 画面分だけ" と範囲を区切る
- **環境が無い** → このレッスンは別 atom（CLI セットアップ系）の後に来る前提でもよい
