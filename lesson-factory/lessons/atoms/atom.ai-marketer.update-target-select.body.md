# リライト候補を選定する

あなたが書いたブログ記事やWebページは、時間が経つと情報が古くなったり、読まれにくくなったりします。
このレッスンでは、**どの記事を書き直す（＝リライトする）べきか**をAIと一緒に選ぶ方法を15分で身につけます。

> たとえば、冷蔵庫の整理を想像してみてください。
> 賞味期限が近いもの、よく使うもの、まだ大丈夫なものを仕分けますよね。
> リライト候補の選定もこれと同じで、「どの記事を優先して手入れすべきか」を整理する作業です。

![候補選定フロー](/lesson-assets/atom.ai-marketer.update-target-select/diagram.png)

## 前提を確認する

このレッスンを進める前に、次の条件を満たしているか確認してください。

- すでにどこかに公開した記事やWebページが **3本以上** ある
- Google Analytics（＝Googleが無料で提供する、サイトへの訪問者数などを調べるツール）か、類似のアクセス解析ツールでページごとの訪問者数を見られる状態にある

どちらもまだ無い場合は、まず記事を数本公開してからこのレッスンに戻ってきてください。

## 候補を探す

### アクセス数データを書き出す

あなたのサイト管理画面（Google Analyticsなど）を開き、**過去3〜6ヶ月**の記事ごとの訪問者数を確認します。数字を手でメモしてもよいですし、CSVダウンロード（＝表形式のファイルとして保存すること）が使えればそちらが楽です。

**良い例**: 「月に200人来ていたのに、今は50人に減った記事」を見つけてメモする
**悪い例**: 「とりあえず全部の記事をリストアップする」→ 数が多すぎて結局手が回らなくなる

### AIに分析を頼む

訪問者数のデータをAIに渡して、リライト候補を選んでもらいましょう。以下のプロンプト（＝AIへの指示文）をコピーして、ChatGPTやClaudeに貼り付けてください。

```
あなたはWebマーケティングの専門家です。
以下に私のブログ記事の一覧と、過去6ヶ月の月別訪問者数を貼ります。

次の3つの観点でリライト候補を上位5つ選び、それぞれ理由を添えてください。
1. アクセス数の減少幅が大きい記事
2. もともとアクセスが多かったが最近落ちている記事
3. 内容が古くなっている可能性が高い記事

【データ】
（ここにあなたのデータを貼り付ける）
```

> **ポイント**: AIは「どのくらい減ったか」の数字を比べるのが得意です。人間だと見落としがちな傾向もグラフなしで見つけてくれます。

![AIへの分析依頼画面](/lesson-assets/atom.ai-marketer.update-target-select/screen_capture.png)

### AIの回答を確認する

AIが候補を返してきたら、次の点を確かめましょう。

- 理由が具体的な数字に基づいているか（「なんとなく古そう」ではなく「3ヶ月で60%減少」など）
- あなたが知っている事実と矛盾していないか（たとえば季節商品の記事は時期的に減るのが当然）

もしAIの回答に納得できない点があれば、追加で質問しましょう。

```
候補1の「○○入門ガイド」について、アクセスが減った原因として考えられることを3つ挙げてください。
```

## 優先順位をつける

AIが出した候補に対して、あなた自身でも次の **3つの観点** で最終チェックをします。

1. **情報の古さ**: 法律・価格・サービス仕様など、すでに変わっている情報が含まれていないか
2. **アクセスの落ち込み**: 3ヶ月前と比べて訪問者が30%以上減っているか
3. **今の目標との関係**: 今あなたが力を入れたいテーマやサービスに関連しているか

**良い例**: 「昨年はアクセスが多かった初心者向けガイドが、最近は半分以下に減っている。価格情報も古い。これはリライト優先度が高い」
**悪い例**: 「個人的に思い入れがある記事だからリライトする」→ 感覚ではなくデータで決める

## 結果をまとめる

最後に、リライト候補の一覧をドキュメント（＝文章ファイル）にまとめます。
以下のプロンプトをAIに送ると、表のひな形を作ってくれます。

```
以下のリライト候補5つを、Markdown形式の表にまとめてください。
列は「優先順位」「記事名」「選定理由」「予定リライト時期」の4つです。

（ここにAIが選んだ候補と理由を貼り付ける）
```

出来上がりのイメージはこのようになります。

```
| 優先順位 | 記事名         | 選定理由                         | 予定リライト時期 |
|----------|---------------|--------------------------------|----------------|
| 1        | ○○入門ガイド    | アクセス60%減、価格情報が古い       | 今週            |
| 2        | △△の選び方      | アクセス30%減、競合記事が増加       | 来週            |
| 3        | □□活用術        | 情報は最新だがCTAが弱い           | 再来週          |
```

この表をスクリーンショットで保存しておくと、あとで提出物として使えます。

## 確認する

作業が終わったら、次のチェックリストで自己確認してください。すべてにチェックが入れば完了です。

- [ ] リライト候補が **3〜5本** に絞られている（多すぎない）
- [ ] それぞれに **データに基づいた理由** が書いてある（「なんとなく」はNG）
- [ ] 優先順位1位の記事が、あなたの **今の目標** に関連している
- [ ] 候補一覧が **表形式のドキュメント** としてまとまっている
- [ ] AIに投げたプロンプトと返答を、あとで見返せるように保存してある

## つまずきやすいポイント

| つまずき | 対策 |
|---------|------|
| 候補が多すぎて絞れない | AIに「最も優先度が高い上位5つだけに絞って」と明確に指示する |
| どの数字を見ればよいかわからない | まずは「月間ページビュー（＝ページが表示された回数）の変化」だけに注目する |
| リライトと新規記事のどちらを優先すべきか迷う | 既存記事のリライトは、ゼロから書くより短期間で効果が出やすい。まずはリライトから始める |
| Google Analyticsの使い方がわからない | AIに「Google Analyticsで記事ごとの訪問者数を見る手順を教えて」と聞く |
| AIの回答が的外れに感じる | データの貼り方が不十分な可能性がある。「記事名・URL・月別PV数」の3点セットを揃えて渡し直す |

お疲れさまでした！候補一覧ができたら、次はこの中から実際にリライトに取り組むステップに進みましょう。
