# Search Consoleで改善点を分析する

ウェブサイトを育てるのは、お店の品ぞろえを整えることに似ています。お客さんが「どんな言葉で検索してお店にたどり着いたか」「入り口まで来たのに入らなかった人はどれくらいいるか」を把握できれば、次に何を改善すべきかが見えてきます。Google Search Console（サーチコンソール）は、まさにそのデータを無料で教えてくれるツールです。

この Atom では、Search Console の「検索パフォーマンス」画面からデータを読み取り、AIに分析を手伝ってもらって **改善すべきページと検索キーワードをまとめたレポート** を1枚作ります。所要時間は約15分です。

## 前提を確認する

始める前に、以下が済んでいることを確認してください。

- Google Search Console にあなたのサイトが登録済みであること（「プロパティ」と呼ばれます）
- 少なくとも1週間分のデータが溜まっていること（登録直後はデータがほぼゼロです）

まだの場合は、先に Search Console のプロパティ登録を済ませてから戻ってきてください。

## ギャップ分析とは何かを知る

「ギャップ分析」と聞くと難しそうですが、やることはシンプルです。

> **表示回数は多いのにクリックされていないキーワード**（＝もったいないキーワード）を見つけること

たとえ話で言えば、「お店の前を毎日100人が通るのに、5人しか入ってこない」状態です。看板やディスプレイ（＝ページのタイトルや説明文）を変えれば、来店数を増やせる可能性が高いですよね。

## Search Console でデータをエクスポートする

1. [Search Console](https://search.google.com/search-console) を開き、対象のプロパティを選びます
2. 左メニューから **「検索パフォーマンス」** → **「検索結果」** をクリックします
3. 画面上部で期間を **「過去3か月」** に変更します
4. **「平均CTR」** と **「平均掲載順位」** のチェックを ON にして、4つの指標（クリック数・表示回数・CTR・掲載順位）すべてを表示させます
5. 下のテーブルが「クエリ」タブになっていることを確認し、右上の **エクスポートボタン（↓アイコン）** → **「Googleスプレッドシート」** を選びます

![Search Console のエクスポート手順](/lesson-assets/atom.ai-marketer.sc-gap-analyze/screen_capture.png)

**良い例**: 期間を「過去3か月」にしてデータ量を確保する
**悪い例**: 「過去7日間」で見ると季節変動やたまたまの偏りに引っ張られる

## AIにギャップ分析を依頼する

エクスポートしたスプレッドシートの内容を、AIに貼り付けて分析してもらいます。以下のプロンプト例をそのまま使えます。

### プロンプト例（Claude / ChatGPT 共通）

```
以下はGoogle Search Consoleから過去3か月分エクスポートした検索クエリデータです。

[ここにスプレッドシートの内容を貼り付け]

次の条件で「もったいないキーワード」を抽出してください:
1. 表示回数が100回以上なのにCTRが3%未満のクエリ
2. 掲載順位が4〜20位（1ページ目の下位〜2ページ目）のクエリ

それぞれについて、以下をMarkdownの表形式でまとめてください:
- クエリ（検索キーワード）
- 表示回数
- 現在のCTR
- 掲載順位
- 改善アクション案（タイトル変更 / メタディスクリプション改善 / コンテンツ追加 のいずれか）
```

> **ポイント**: データ量が多い場合は「上位50件だけ貼る」か、CSVファイルをそのままアップロードしてください。ChatGPTやClaudeはファイル添付にも対応しています。

## AIの分析結果を確認する

AIが返してくれた表を見て、次の観点でチェックしましょう。

1. **表示回数が特に多いのにCTRが低いキーワード** → 最優先で改善する価値があります
2. **掲載順位が10位前後（1ページ目と2ページ目の境界）** → 少しの改善で1ページ目に上がる可能性が高いです
3. **自分のサイトと関係が薄いキーワード** → 無理に対応しなくてOKです。AIが機械的に拾ったノイズの場合もあります

**良い例**: 「表示1,200回・CTR 1.2%・順位8位」のキーワード → タイトルを改善すればクリック数が大きく伸びる可能性
**悪い例**: 「表示30回・CTR 0%・順位45位」のキーワード → そもそも上位表示されていないので、タイトル変更だけでは効果が薄い

## 改善レポートを仕上げる

AIの分析結果をもとに、以下の構成で改善レポート（Markdownドキュメント）を完成させましょう。

```markdown
# Search Console ギャップ分析レポート

## 分析期間
YYYY/MM/DD 〜 YYYY/MM/DD

## サマリー
- 分析クエリ数: ○件
- もったいないキーワード数: ○件
- 最優先改善キーワード: 「○○○」

## もったいないキーワード一覧
| キーワード | 表示回数 | CTR | 順位 | 改善アクション |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |

## 次のアクション
1. 最優先キーワードのページタイトルを変更する
2. ...
```

> AIに「このレポートの雛形を埋めて」と追加で依頼すれば、コピペで完成します。

## できあがりを確認する

完成したレポートを見て、以下を確認してください。

- [ ] もったいないキーワードが **3件以上** リストアップされている
- [ ] 各キーワードに対して **具体的な改善アクション** が書かれている
- [ ] レポートのサマリーに **最優先の1件** が明記されている
- [ ] 分析期間が記載されている

すべてチェックできたら、この Atom は完了です。

## つまずきやすいポイント

| よくある問題 | 対処法 |
|---|---|
| データが少なすぎて分析できない | サイト登録から最低2週間、できれば1か月待ってから再挑戦してください |
| エクスポートボタンが見つからない | テーブルの右上にある下向き矢印（↓）です。画面を下にスクロールすると見えます |
| AIが的外れなキーワードを優先している | 「自社サービスと関連が薄いキーワードは除外して」と追加で指示してください |
| CTRや掲載順位の列が表示されない | 画面上部の「平均CTR」「平均掲載順位」ボタンをクリックしてONにしてください |
