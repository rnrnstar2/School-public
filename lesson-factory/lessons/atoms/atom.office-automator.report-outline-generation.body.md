# レポートの構成案をAIに作らせる

レポートや企画書を書くとき、いきなり本文から書き始めると途中で迷子になりませんか？
料理にたとえると、カレーを作る前に「何人分か」「材料は何が必要か」をメモしておくのと同じで、レポートも全体の骨組み（＝構成案）を先に決めておくと、あとの作業がぐっとラクになります。

このレッスンでは、AIチャットツール（＝ChatGPTやClaudeのように、言葉で対話できるAI）を使って、**15分以内**にレポートの構成案を完成させる方法を学びます。完成した構成案はテキストファイルとして保存し、あなただけの「レポート設計図」になります。

## 前提を確認する

始める前に、次の2つを用意してください。

- **AIチャットツール**にアクセスできること（ChatGPT、Claude、Geminiなど、どれでもOK）
- **書きたいレポートの情報**を頭のなかで整理しておくこと：
  - 何のためのレポートか（目的）
  - 誰に読んでもらうか（想定読者）

どのAIツールを使えばよいか迷ったら、普段使い慣れているものを選んで大丈夫です。このレッスンの手順はどのツールでも同じように使えます。

## ステップ1：AIへの指示文（プロンプト）を準備する

![構成案作成フロー](/lesson-assets/atom.office-automator.report-outline-generation/diagram.png)

AIに良い構成案を作ってもらうコツは、「何を・誰に・どのくらいの量で」を明確に伝えることです。旅行で言えば、「どこかに行きたい」よりも「来月、2泊3日で京都に行きたい。予算は3万円」と伝えるほうが、良い提案をもらえますよね。

次のひな形をそのままコピーして、`[ ]` の部分だけ書き換えてAIに入力してください。

```
以下の条件で、レポートの構成案を作成してください。

- レポートの目的: [例: 新規プロジェクトの提案]
- 想定読者: [例: 社内の意思決定者]
- 章数・長さ: [例: 全5章、各章300字程度]
- 特に盛り込みたい要素: [例: 競合比較、ROI試算]

各章には「章タイトル」と「その章で書く内容の要約（2〜3行）」を付けてください。
```

### 良い例と悪い例

**良い例**:
> 「目的: 社内研修の効果測定レポート、読者: 人事部長、全4章、前回研修との比較を含める」

→ 具体的な情報が揃っていて、AIが的確な構成を出しやすい

**悪い例**:
> 「レポートの構成案を作って」

→ 情報が足りず、一般的すぎる構成案しか出てこない。「誰に向けたレポートか」だけでも追加すると改善します

## ステップ2：AIの出力を3つの観点で確認する

AIが返してきた構成案を、次の3つの観点でチェックしましょう。

| チェック観点 | 確認すること | OKの目安 |
|---|---|---|
| 1. 目的との一致 | ゴールに沿った章構成になっているか | 目的と直接関係ない章がない |
| 2. 読者レベル | 想定読者の知識レベルに合っているか | 専門用語が多すぎない |
| 3. 過不足 | 必要な要素の漏れ・不要な章がないか | 盛り込みたい要素がすべて含まれている |

3つともOKなら、ステップ4に進んでください。1つでも気になる点があれば、次のステップで修正します。

## ステップ3：AIに修正を依頼する

最初の出力で完璧な構成案ができることは少ないです。気になる点をAIに伝えて調整しましょう。

**修正指示のコツ**: 「何を」「どこに」「どうしてほしいか」の3点を伝えます。

**良い修正指示の例**:

```
第3章に「競合サービス比較表」を追加して、全体で5章に再構成してください。
第1章の導入はもう少し短くして、背景を3行以内にまとめてください。
```

**悪い修正指示の例**:
> 「もっといい感じにして」

→ 何を直したいのかAIに伝わりません。「第2章の内容が薄いので、具体例を1つ追加して」のように具体的に伝えましょう

修正は **2〜3回** にとどめるのがコツです。何度直しても思い通りにならない場合は、最初のプロンプトの前提条件を見直したほうが早道です。

## ステップ4：完成した構成案を保存する

最終的な構成案が決まったら、テキストファイルやメモアプリにコピー＆ペーストして保存します。

![構成案の保存手順](/lesson-assets/atom.office-automator.report-outline-generation/screen_capture.png)

保存するとき、ファイル名に日付とレポート名を入れておくと後で見つけやすくなります。

**良いファイル名の例**: `20260413_研修効果レポート_構成案.txt`

**悪いファイル名の例**: `メモ.txt`（何のファイルか後からわからなくなる）

この構成案が、本文を書くときの「道しるべ」になります。「この章には何を書くんだっけ？」と迷ったときにすぐ取り出せる場所に置いておきましょう。

## つまずきやすいポイントに対処する

| こんなとき | 対処法 |
|---|---|
| AIが長すぎる構成案を出してきた | 「各章を100字以内で要約して」と追加で指示する |
| 専門用語が多すぎる | 「小学生にもわかる言葉で書き直して」と依頼する |
| 目的から外れた提案が来た | 会話をリセット（新しいチャットを開始）して、目的をより具体的に書き直す |
| どの章から書き始めればいいかわからない | AIに「この構成案で最初に書くべき章はどれですか？理由も教えてください」と聞く |
| レポートのテーマ自体が決まっていない | まず「最近の業務で課題に感じていることを3つ挙げて」とAIに相談してからテーマを決める |

## 完成を確認する

次の3つすべてに当てはまれば、このレッスンは完了です。

- [ ] AIとのやりとりで構成案を作成できた（スクリーンショットを撮っておくと振り返りに便利）
- [ ] 構成案に「章タイトル」と「各章の内容要約」が含まれている
- [ ] 構成案をテキストファイルまたはメモアプリに保存した

おめでとうございます！ あなたはAIを使ってレポートの骨組みを設計できるようになりました。次は、この構成案をもとに実際のレポート本文をAIと一緒に書いていくステップに進みましょう。
