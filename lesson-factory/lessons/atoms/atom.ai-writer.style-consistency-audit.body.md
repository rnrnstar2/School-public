# 文体の一貫性を監査する

## このレッスンで得られること

あなたが書いた文章やAIに作ってもらった文章を読み返すと、途中から「です・ます」が「だ・である」に変わっていたり、同じものを「ユーザー」と呼んだり「お客様」と呼んだりしていることはありませんか？

これは**文体のブレ**と呼ばれる問題で、読み手に「この文章、なんだか読みにくいな」という印象を与えてしまいます。料理にたとえると、和食のコース料理の途中でいきなりケチャップ味の一品が出てくるようなものです。一品ずつは美味しくても、全体の統一感が崩れると違和感が残ります。

このレッスンでは、AIチャット（Claude や ChatGPT など）を使って、文章の文体が統一されているかを15分以内でチェックする方法を身につけます。

## 前提として必要なもの

- AIチャットツール（Claude、ChatGPT、Gemini など）をブラウザで開ける状態
- チェックしたい文章（ブログ記事、提案書、メルマガなど。500文字〜3000文字程度がやりやすい）

## 文体の「監査」とは何かを理解する

「監査（かんさ）」というと難しそうですが、やることはシンプルです。書いた文章を**決めたルールに照らし合わせてチェックする**だけです。

チェックするポイントは主に3つあります。

1. **語尾の統一**: 「です・ます」調か「だ・である」調かが混在していないか
2. **用語の統一**: 同じ意味の言葉を複数の表記で使っていないか（例：「お客様」と「ユーザー」）
3. **トーンの統一**: 親しみやすい口調とビジネス的な硬い口調が混ざっていないか

## AIに文体チェックを依頼する

まず、AIチャットに文体チェックを頼むためのプロンプト（＝AIへの指示文）を用意します。

### 使うプロンプト例

以下のプロンプトをコピーして、AIチャットに貼り付けてください。`【ここにあなたの文章を貼り付け】` の部分を、チェックしたい文章に差し替えます。

```
以下の文章の「文体の一貫性」を監査してください。
チェック観点は次の3つです。

1. 語尾の統一（です・ます / だ・である の混在がないか）
2. 用語の統一（同じ意味の言葉に複数の表記がないか）
3. トーンの統一（親しみやすい / ビジネス的 が混在していないか）

問題があれば、該当箇所を引用し、どう修正すべきか具体的に提案してください。
問題がなければ「一貫性に問題はありません」と回答してください。

---
【ここにあなたの文章を貼り付け】
```

### 良い例と悪い例

**良い例（チェック対象が明確）**:
> 「以下のブログ記事の文体を監査してください。記事は『です・ます調』で統一する方針です。」

**悪い例（曖昧すぎる）**:
> 「この文章を直してください。」

悪い例だと、AIは文体だけでなく内容・構成・表現すべてを変えようとしてしまい、元の文章が大きく書き換わるリスクがあります。

## AIの指摘を確認して修正する

AIから返ってきた結果を見て、以下の手順で修正します。

1. **指摘を一覧にする**: AIが見つけた問題箇所をリストアップする
2. **修正方針を決める**: 「です・ます」に統一する、「ユーザー」に表記を揃える、など
3. **一括で修正する**: 方針が決まったら、AIに「上記の方針で文章全体を修正してください」と依頼する

### もう一歩踏み込むプロンプト

修正が終わったあと、もう一度同じチェックを走らせると安心です。

```
修正後の文章をもう一度同じ3観点でチェックしてください。新たな問題があれば指摘してください。
```

このように**チェック→修正→再チェック**のサイクルを回すのがコツです。

## 成果物を仕上げる

最終的に、以下のような **文体監査レポート** を Markdown 形式でまとめてみましょう。

```
# 文体監査レポート

## 対象文書
（文書名やURLなど）

## チェック結果

### 1. 語尾の統一
- 問題なし / ○件の混在あり（修正済み）

### 2. 用語の統一
- 「ユーザー」と「お客様」の混在 → 「ユーザー」に統一（修正済み）

### 3. トーンの統一
- 問題なし / ○件の混在あり（修正済み）

## 修正方針
（どのルールで統一したか）
```

このレポートが完成していれば、このレッスンのゴールは達成です。

## つまずきやすいポイント

- **「問題がなかった」と言われたけど不安**: 文章が短すぎると（100文字以下など）AIが判断しにくいことがあります。500文字以上の文章で試してみましょう
- **AIの修正案が元の文章と大きく変わってしまった**: チェック依頼と修正依頼を分けるのがポイントです。まず「問題箇所だけ指摘して」と頼み、次に「指摘箇所だけ修正して」と段階を分けましょう
- **「です・ます」「だ・である」どちらが正解？**: 正解はありません。大切なのは**どちらかに統一すること**です。読者や用途に合わせて選んでください

## まとめ

このレッスンでは、AIを使って文章の文体をチェックし、統一する方法を学びました。

- 文体チェックの3観点（語尾・用語・トーン）を覚える
- AIに具体的なチェック指示を出すプロンプトを使う
- チェック→修正→再チェックのサイクルで品質を高める

次に文章を書いたり、AIに文章を作ってもらったりしたとき、この監査ステップを挟むだけで、読み手にとってぐっと読みやすい文章になります。
