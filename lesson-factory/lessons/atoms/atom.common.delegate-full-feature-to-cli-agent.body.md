# Codex CLI / Claude Code に feature 単位で丸投げする

> **Status**: skeleton — `/lesson-improve` で本文化予定
>
> 想定 persona: `web-builder` / `ai-app-builder` / `p-noneng-webapp`（"1 行ずつ AI に書かせる" を卒業したい層）

## なぜこれをやる？

非エンジニアが AI コーディングで疲弊するパターン:
「次は何を直せばいい？」と 1 行ずつ聞き続けて、自分が結局判断役に詰む。
解決策は粒度を上げること――「ログイン機能を一式作って」「在庫管理ページを一式作って」と
**feature まるごと** を 1 タスクとして委譲し、
AI に計画・実装・テストまで連続で実行させる。
このスタイルは Codex CLI / Claude Code 両方で同じく機能する。

## 何を準備する？

- Codex CLI もしくは Claude Code がローカルで動く状態
- 「いま欲しい feature」を 1 段落で書いたメモ（入力・出力・触る画面）
- リポジトリ（既存 or 直前 atom で作ったもの）

## やってみよう

1. CLI を起動し、最初のメッセージに feature 1 段落要件を投入
2. プロンプトで明示する 4 要素: **目的 / 触る範囲 / 完了条件 / 質問は私に聞く**
3. AI が計画案を出したら、合意するか 1-2 点だけ修正してから `go`
4. 実装が終わったら自分で動作確認し、エラーは丸ごと貼り付けて再委譲
5. 自分用「feature 委譲プロンプトテンプレ」を 1 つ md にメモして保存

## できたかチェック

- [ ] 1 つの feature を 1 つの委譲プロンプトで完了させた
- [ ] 自分用テンプレ（目的 / 範囲 / 完了 / 確認）を 1 つ持てた
- [ ] 自分は実装ではなくレビュー側に回れた

## つまづいたら

- **粒度が大きすぎる** → 1 feature = 1 画面 + 1 API くらいに刻む
- **AI が暴走する** → "完了条件" を最初に明文化、合意してから実行
- **テンプレが固まらない** → 1 周目はそのまま走り、2 周目に同じ枠組みを使い回せたか確認
