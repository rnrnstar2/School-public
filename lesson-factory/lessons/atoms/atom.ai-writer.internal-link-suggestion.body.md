# 内部リンク構造をAIに提案させる

ブログやWebサイトの記事を何本か書いたあと、こんな悩みが出てきませんか？「この記事とあの記事、つながりがあるのに読者が気づけない」「どこにリンクを貼ればいいか分からない」——これは本棚の整理に似ています。本がバラバラに並んでいると目当ての本が見つからないように、記事どうしのつながり（＝内部リンク）が整っていないと、読者は関連する情報にたどり着けません。

このレッスンでは、AIに「どの記事からどの記事へリンクを貼るべきか」を提案してもらい、その結果をもとにリンク構造の設計シートを完成させます。所要時間はおよそ15分です。

## 知っておくこと：内部リンクとは

内部リンクとは、同じサイト内の別のページへのリンクのことです。たとえばブログ記事Aの中に「詳しくはこちらの記事Bをご覧ください」と書いてリンクを貼る、あれが内部リンクです。

内部リンクが整っていると、こんないいことがあります。

- **読者にとって便利**: 関連する記事をすぐ読める
- **検索エンジンにも好印象**: Googleなどがサイトの構造を理解しやすくなる（＝SEO効果）
- **サイト全体の回遊率が上がる**: 1人の読者がたくさんのページを見てくれる

でも、記事が増えてくると「どこに何を貼ればいいのか」を人力で考えるのは大変です。ここでAIの出番です。

## 準備する

始める前に、次のものを用意してください。

- **記事リスト**: あなたのサイトの記事タイトルと、各記事の内容を1-2行でまとめたメモ（5本以上あると効果的）
- **AIツール**: ChatGPT、Claude、またはお好みのAIチャットツール

記事リストの例:

```
1. 「初心者向けSEOの基本」— SEOとは何か、基礎用語を解説
2. 「キーワードの選び方」— 検索キーワードの調べ方と選定基準
3. 「読まれるタイトルの付け方」— クリック率を上げるタイトル術
4. 「ブログの文章構成テンプレート」— PREP法など構成の型を紹介
5. 「AIでブログ記事を書く方法」— ChatGPTを使った記事作成手順
```

## AIに内部リンクを提案してもらう

### ステップ1：記事リストを渡す

AIチャットに、次のようなプロンプト（＝AIへの指示文）を送ります。コピーしてそのまま使えます。

**プロンプト例：**

```
以下は私のブログの記事リストです。各記事の間で「この記事からこの記事へ内部リンクを貼るべき」という提案を、理由付きで出してください。

優先度（高・中・低）も付けてください。

【記事リスト】
1. 「初心者向けSEOの基本」— SEOとは何か、基礎用語を解説
2. 「キーワードの選び方」— 検索キーワードの調べ方と選定基準
3. 「読まれるタイトルの付け方」— クリック率を上げるタイトル術
4. 「ブログの文章構成テンプレート」— PREP法など構成の型を紹介
5. 「AIでブログ記事を書く方法」— ChatGPTを使った記事作成手順
```

### ステップ2：AIの回答を確認する

AIはこんな形で提案してくれます。

```
■ 優先度：高
・記事1「SEOの基本」→ 記事2「キーワードの選び方」
  理由：SEOの基本を学んだ読者が次に知りたいのはキーワード選定

・記事5「AIでブログ記事を書く方法」→ 記事4「文章構成テンプレート」
  理由：AI記事作成の前に構成の型を知っておくと質が上がる

■ 優先度：中
・記事2「キーワードの選び方」→ 記事3「タイトルの付け方」
  理由：キーワードを決めた後、それをタイトルに活かす流れが自然
```

### ステップ3：提案を吟味する

AIの提案を鵜呑みにせず、次の観点でチェックしてください。

**良い内部リンクの例：**
- 「SEOの基本」の記事中に「具体的なキーワードの選び方はこちら」とリンク → 読者の次の疑問に自然に答えている

**悪い内部リンクの例：**
- 「タイトルの付け方」の記事中に「AIでブログを書く方法はこちら」とリンク → 話題が飛びすぎて読者が混乱する

判断のコツは「この記事を読んでいる人が、次に知りたくなることは何か？」と考えることです。

## リンク設計シートを作る

### ステップ4：AIに表形式で整理させる

提案内容が固まったら、次のプロンプトで設計シートにまとめてもらいましょう。

**プロンプト例：**

```
上の提案をもとに、内部リンク設計シートをMarkdownの表形式で作ってください。
列は「リンク元記事」「リンク先記事」「リンクを貼る場所の目安」「優先度」の4つでお願いします。
```

AIが出力する表の例：

```
| リンク元記事 | リンク先記事 | リンクを貼る場所の目安 | 優先度 |
|---|---|---|---|
| SEOの基本 | キーワードの選び方 | 「キーワード」に初めて言及する段落 | 高 |
| AIでブログ記事を書く方法 | 文章構成テンプレート | 「構成を考える」ステップの冒頭 | 高 |
| キーワードの選び方 | タイトルの付け方 | 「選んだキーワードの活用法」セクション | 中 |
```

![完成した設計シートの画面](/lesson-assets/atom.ai-writer.internal-link-suggestion/screen_capture.png)

### ステップ5：設計シートを保存する

表をコピーして、Googleドキュメントやメモアプリに貼り付けて保存してください。これがあなたの「内部リンク設計シート」になります。記事を追加するたびに、同じ手順でAIに提案を更新してもらえます。

## 確認する

次のことができていれば、このレッスンは完了です。

- [ ] AIに記事リストを渡して、内部リンクの提案を受け取れた
- [ ] 提案を自分の目で確認し、不自然なリンクを除外できた
- [ ] Markdown表形式の内部リンク設計シートが手元にある

## つまずきやすいポイント

**Q. 記事が少なすぎてAIがうまく提案してくれない**
A. 最低3本あればOKですが、5本以上あるとAIの提案精度が上がります。記事が少ない場合は「今後書く予定の記事タイトル」も含めてリストに入れてみましょう。

**Q. AIの提案が的外れに感じる**
A. 記事の内容説明が短すぎるかもしれません。各記事の要約を2-3行に増やしてみてください。AIは渡された情報だけで判断するので、情報が多いほど精度が上がります。

**Q. リンクを貼る場所が分からない**
A. ステップ4のプロンプトで「リンクを貼る場所の目安」を出力させています。AIの提案をヒントに、実際の記事を読み返して「ここで読者が次の情報を欲しがるな」と感じる箇所に貼りましょう。
