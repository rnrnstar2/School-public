# リスク説明資料を作る

旅行に出かける前に「もし雨が降ったら？」「もし財布をなくしたら？」と備えを考えますよね。サービスやプロジェクトでも同じように、「もし○○が起きたらどうするか」をあらかじめまとめておくのがリスク説明資料です。

このレッスンでは、AIツール（＝ChatGPTやClaudeなどの対話型AI）を使って、**15分以内**でリスク説明資料を完成させます。完成後は「自分でリスク資料を作れる」という実感を持てるようになります。

![リスク説明資料の作成フロー](/lesson-assets/atom.nocode-builder.create-risk-explanation-doc/diagram.png)

## 前提を確認する

このレッスンを進める前に、次の2つが揃っているか確認してください。

- **AIツールのアカウント**: ChatGPT、Claude、Gemini など、どれか1つでOKです
- **プロジェクトの概要**: 自分のプロジェクトやサービスを1〜2文で説明できる状態にしておく

特別なプログラミングの知識は一切必要ありません。

## ステップ1：リスクを洗い出す

まず、あなたのプロジェクトで「何が起こるかもしれないか」をリストアップします。

### AIに手伝ってもらう

AIツールを開いて、次のプロンプト（＝AIへの指示文）をコピー＆ペーストしてください。`〔　〕`の部分だけ自分の情報に書き換えます。

```
私は〔プロジェクト名〕というプロジェクトを進めています。
概要：〔プロジェクトの概要を1〜2文で〕
このプロジェクトで想定されるリスクを、次の3つの観点から5〜10個挙げてください。
- 事業リスク（売上・競合・市場の変化など）
- 技術リスク（システム停止・データ消失など）
- 法務リスク（法令変更・契約トラブルなど）
各リスクは1行で簡潔に書いてください。
```

### AIの回答を自分でチェックする

AIが出したリスクをそのまま使わず、次の基準で取捨選択してください。

**残すもの（良い例）：**
- 「ユーザーの個人情報が漏洩（＝勝手に外部に出てしまうこと）する可能性」
- 「システムが1時間以上止まる可能性」
- 「法令変更でサービス内容を変えなければならない可能性」

**削除するもの（悪い例）：**
- 「何か悪いことが起きる」← 具体的ではないので不採用
- 「隕石が落ちる」← 現実的に備えようがないので対象外
- あなたのプロジェクトに明らかに当てはまらないもの

最終的に **5〜8個** に絞るのがおすすめです。

## ステップ2：資料の構成を決める

リスクが洗い出せたら、次は資料の形を整えます。以下のプロンプトをAIに送りましょう。

```
次のリスク一覧を、社内説明用の資料として整理してください。
表形式で、次の列を含めてください：
- リスク名
- 影響度（高/中/低）
- 発生可能性（高/中/低）
- 対策（具体的に何をするか）

〔ステップ1で絞り込んだリスクを貼り付け〕
```

![AI出力の表を整理している画面](/lesson-assets/atom.nocode-builder.create-risk-explanation-doc/screen_capture.png)

基本的な資料の構成は次の3パートです。

1. **概要** — プロジェクトの簡単な説明（2〜3文）
2. **リスク一覧** — 各リスクの名称・影響度・対策（表形式）
3. **対応優先度** — どのリスクから対処すべきか（影響度×発生可能性で判断）

## ステップ3：対策を具体化する

AIが出した「対策」欄が抽象的な場合は、追加で質問して具体化します。

```
上の表の対策をもっと具体的にしてください。
各対策に「誰が・何を・いつまでに」の3点を含めてください。
```

**良い例：**
- 「影響度：高 / 対策：運用担当がバックアップを毎日自動取得し、復旧手順書を月1回確認する」

**悪い例：**
- 「影響度：高 / 対策：気をつける」← 具体的な行動がないので改善が必要

## ステップ4：資料としてまとめる

ここまでの内容を1つのドキュメントにまとめます。次のプロンプトをAIに送ってください。

```
ここまでの内容を、次の構成でMarkdown形式の資料にまとめてください。

1. タイトル：〔プロジェクト名〕リスク説明資料
2. 作成日：〔今日の日付〕
3. プロジェクト概要（2〜3文）
4. リスク一覧（表形式）
5. 対応優先度の説明（1段落）
6. まとめ（2〜3文）

専門用語には括弧書きでわかりやすい説明を添えてください。
```

AIが出力した文章をコピーして、お好みのツール（Googleドキュメント、Notion、Wordなど）に貼り付ければ完成です。

## 最終確認する

資料ができたら、次のチェックリストで確認しましょう。すべてにチェックが入れば完了です。

- [ ] リスクが5件以上書かれている
- [ ] 各リスクに「影響度」と「対策」が書かれている
- [ ] 対策に「誰が・何を・いつまでに」が含まれている
- [ ] 専門用語に簡単な説明が添えられている
- [ ] 15分以内に作成できた

## つまずきポイントに対処する

| こんなとき | こうしてみましょう |
|---|---|
| リスクが思いつかない | AIに「〔業界名〕でよくあるリスク例を10個教えて」と聞いてから、自分に当てはまるものを選ぶ |
| AIの回答が難しすぎる | 「中学生にもわかる言葉で書き直して」とAIに頼む |
| 対策が抽象的なまま | 「『誰が・何を・いつまでに』の形で具体的にして」とAIに追加指示する |
| 影響度の判断に迷う | 「このリスクが実際に起きたら、売上・信用・業務のどれに一番影響するか教えて」とAIに聞く |
| 資料のボリュームが多すぎる | リスクを「影響度：高」のものだけに絞って、まず5件で資料を仕上げる |
