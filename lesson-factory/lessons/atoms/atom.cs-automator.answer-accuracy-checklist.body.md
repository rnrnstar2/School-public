# 回答精度を確認するチェックリストを作る

あなたがカスタマーサポートで AI に回答を作ってもらったとき、「この回答、本当に合ってる？」と不安になったことはありませんか？

料理でいうと、レシピ通りに作った料理を最後に味見するようなものです。味見をしないでお客さんに出すのは怖いですよね。このレッスンでは、AI の回答を「味見」するためのチェックリストをあなた自身の手で作ります。完成すると、AI が作った回答を**5つの観点でサッと確認できる一覧表**が手元に残ります。

## 準備する

このレッスンに必要なものは次の2つだけです。

- **AI ツール**（ChatGPT、Claude など）にアクセスできること
- **メモ帳やドキュメントツール**（Google ドキュメント、Notion、メモ帳アプリなど何でも OK）

特別なインストールは不要です。いつも使っているツールで大丈夫です。所要時間はおよそ **15 分**です。

## チェックリストの5つの確認項目を知る

AI の回答を確認するときは、次の5つに注目します。頭文字を取って「**じ・こ・ト・み・き**」と覚えると便利です。

![チェックリスト5項目の一覧図](/lesson-assets/atom.cs-automator.answer-accuracy-checklist/diagram.png)

| # | 確認項目 | ひとことで言うと | 具体的に見るところ |
|---|---------|----------------|------------------|
| 1 | **事実は正しいか（じ）** | 数字や日付が合っているか | 金額・日付・仕様を元の資料と見比べる |
| 2 | **顧客の質問に答えているか（こ）** | 聞かれたことに答えているか | 質問を読み直してから回答を読む |
| 3 | **トーンは適切か（ト）** | 丁寧で会社の雰囲気に合うか | 「お客様に見せて恥ずかしくないか」を基準にする |
| 4 | **見落としはないか（み）** | 複数の質問すべてに答えているか | 質問文の「？」の数を数えて照合する |
| 5 | **規約に触れていないか（き）** | 会社のルールに違反しないか | 返品条件・保証範囲など社内ルールと突き合わせる |

### 良い例と悪い例を比べる

**顧客からの質問**: 「返品期限はいつですか？購入から2週間経っています。」

- ❌ **悪い例の AI 回答**: 「返品はいつでも可能です。」
  - → 項目1「事実は正しいか」で不合格。会社の返品規約と食い違っている可能性が高い
- ✅ **良い例の AI 回答**: 「通常、返品は購入後7日以内にお受けしています。すでに2週間経過しているため、個別にご相談を承ります。サポート窓口までご連絡ください。」
  - → 事実が正しく、顧客の状況に合わせた代替案も提示している

## AI にチェックリストのたたき台を作ってもらう

5つの項目を覚えたら、AI ツールを使ってチェックリストの文書を効率よく作りましょう。ゼロから自分で書くより、AI にたたき台を出してもらい、自分で手直しする方が速くて確実です。

### ステップ1: AI にプロンプトを送る

以下のプロンプトをコピーして、ChatGPT や Claude に送ってください。

```
カスタマーサポートで AI が生成した回答の精度を確認するチェックリストを作ってください。
次の5項目を必ず含めてください：
1. 事実は正しいか（日付・金額・仕様）
2. 顧客の質問に答えているか
3. トーンは適切か
4. 見落としはないか（複数質問への対応）
5. 規約に触れていないか
各項目に「確認ポイント」と「NGの例」を1つずつ付けてください。
フォーマットはチェックボックス付きの箇条書きでお願いします。
```

### ステップ2: 出力を自分の会社に合わせて手直しする

AI が出力したチェックリストを、ドキュメントツール（Google ドキュメントなど）にコピーして、以下の点を修正します。

- **自社の返品期限や保証条件**など、正しい数字に差し替える
- **社内で使っている言い回し**（「お客様」「ご利用者様」など）に統一する
- **不要な項目があれば削除**、**足りない項目があれば追加**する

> **ポイント**: AI が作ったものをそのまま使うのではなく、「自社の事情に合わせて手直しする」ひと手間が品質を上げるコツです。

### 良い例と悪い例

- ✅ **良い例**: 「事実は正しいか → 日付・金額・仕様を元の資料と見比べる。NG例: 返品期限を『30日以内』と書いたが実際は7日以内だった」と具体的な確認方法と失敗例が書かれている
- ❌ **悪い例**: 「内容を確認する」とだけ書かれていて、何をどう確認するか分からない

## チェックリストを実際に使ってみる

作ったチェックリストを実際に試してみましょう。ここが一番大事なステップです。

### ステップ1: AI にサンプル回答を生成させる

以下のプロンプトを AI に送ってください。

```
カスタマーサポート担当として、以下の顧客の質問に回答してください。
「先週注文した商品がまだ届きません。配送状況を教えてください。また、届かない場合は返金してもらえますか？」
```

### ステップ2: 5項目でチェックする

AI の回答が出たら、作ったチェックリストの5項目を上から順に確認します。

| 確認項目 | チェック結果の例 |
|---------|----------------|
| 事実は正しいか | 「通常3〜5営業日でお届け」→ 自社の実際の配送日数と合っているか？ |
| 質問に答えているか | 配送状況の確認方法 **と** 返金条件の **両方** に答えているか？ |
| トーンは適切か | 丁寧でお客様に失礼のない文体か？ |
| 見落としはないか | 質問は2つ（配送状況＋返金）。両方に触れているか？ |
| 規約に触れていないか | 返金条件が社内規定と一致しているか？ |

### ステップ3: 問題箇所を修正する

チェックで見つかった問題があれば、AI にこう伝えて修正してもらいます。

```
先ほどの回答について、以下を修正してください。
- 配送日数を「3〜5営業日」から「2〜4営業日」に変更
- 返金条件について「商品到着後14日以内」という条件を追記
```

![チェックリストを使って回答を確認している様子](/lesson-assets/atom.cs-automator.answer-accuracy-checklist/screen_capture.png)

## 成果物を確認する

完成したチェックリストが次の条件をすべて満たしているか、最終確認しましょう。

- [ ] 5つの確認項目（事実・質問対応・トーン・見落とし・規約）がすべて含まれている
- [ ] 各項目に**具体的な確認ポイント**が書かれている（「確認する」だけでは不十分）
- [ ] 各項目に**NG例**が1つ以上書かれている
- [ ] 自社の情報（返品期限、配送日数など）に合わせて数字を差し替えてある
- [ ] 実際の AI 回答に適用して、少なくとも1つは問題を見つけられた

すべてにチェックが付いたら、このレッスンの成果物は完成です。

## つまずきやすいポイントに備える

| つまずき | 対処法 |
|---------|-------|
| **「何を確認すればよいか分からない」** | まずは「事実は正しいか」と「質問に答えているか」の2つだけから始めましょう。慣れてきたら残り3つを足します |
| **「AI が自信満々に間違える」** | AI はもっともらしい文章を書くのが得意です。特に数字（日付・金額・期限）は**必ず**元の資料と突き合わせましょう |
| **「チェック項目が多すぎて大変」** | 最初は3項目から始めて OK。1週間使ってみてから、見落としが多い項目を追加していきましょう |
| **「AI の出力をどこまで手直しすればいいか分からない」** | 「自社固有の数字」と「社内用語」だけ変えれば最低限 OK です。完璧を目指さなくて大丈夫 |
| **「チェックリストを毎回見るのが面倒」** | ドキュメントツールのテンプレート機能を使って、ワンクリックで呼び出せるようにしましょう |
