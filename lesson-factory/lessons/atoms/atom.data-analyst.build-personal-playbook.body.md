# 自分の分析手順書を作る

料理をしているとき、「あれ、次は何を入れるんだっけ？」と迷った経験はありませんか？
そんなとき、レシピ（＝手順書）があれば安心ですよね。

データ分析でも同じです。あなたがこれまでにやった分析作業を「レシピ」としてまとめておけば、次に同じ作業をするときに迷わずスムーズに進められます。
このレッスンでは、AIチャットツール（＝人工知能と会話しながら使える便利な道具。ChatGPTやClaudeなど）の力を借りて、あなただけの**分析手順書（＝プレイブック）**を15分で作ります。

![分析手順書の全体像](/lesson-assets/atom.data-analyst.build-personal-playbook/diagram.png)

## 前提を確認する

このレッスンを始める前に、次の2つだけ確認してください。

- 何か一つ、過去にデータを使って調べた経験がある（エクセルで表を眺めた程度でOKです）
- ChatGPT（https://chat.openai.com）やClaude（https://claude.ai）などのAIチャットツールをブラウザで開ける状態にある

特別なソフトのインストールは不要です。ブラウザさえあれば始められます。

## 手順書のひな形を知る

まずは、手順書の枠組み（＝テンプレート）を決めましょう。良い分析手順書には次の**4つの項目**が入っています。

| 項目 | 一言でいうと | 例 |
|------|-------------|----|
| **目的** | 何を知りたいのか | 「先月と今月の売上を比べたい」 |
| **手順** | どうやって調べるのか | 「エクセルのA列をコピーして…」 |
| **確認ポイント** | 結果が正しいか確かめる方法 | 「合計金額が元データと一致するか見る」 |
| **つまずき対策** | よくある失敗と対処法 | 「空欄セルがある場合は除外する」 |

この4項目をテンプレートとして使います。

## 分析テーマを一つ選ぶ

あなたが過去にやった分析、または今後やりたい分析を**一つだけ**選んでください。

### 良い例

- 「月ごとの売上を調べて、先月より増えたか減ったかを報告する」
- 「アンケート回答を年代別に集計して、傾向をまとめる」

→ **何を・どうする**が具体的で、手順書に書きやすいテーマです。

### 悪い例

- 「売上を分析する」
- 「データを見る」

→ ざっくりしすぎていて、何から始めればいいかわかりません。

**コツ**: 「○○のデータを使って、△△を調べて、□□に報告する」のように、3つの要素が入っていると具体的になります。

## AIに手順書の下書きを作ってもらう

テーマが決まったら、AIチャットツールを開いて次のプロンプト（＝AIへの指示文）をコピー＆ペーストしてください。`〇〇` の部分だけ、あなたのテーマに書き換えます。

```
私はデータ分析の初心者です。
「〇〇」について分析する手順書を作りたいです。

次の4項目で、初心者にもわかるように手順書を書いてください。
各手順は「エクセルのどのボタンを押す」レベルで具体的にお願いします。

1. 目的（この分析で何がわかるか）
2. 手順（ステップごとに番号をつけて）
3. 確認ポイント（結果が正しいか確かめる方法）
4. つまずき対策（初心者がハマりやすい失敗と対処法）
```

![AIチャットにプロンプトを入力する画面](/lesson-assets/atom.data-analyst.build-personal-playbook/screen_capture.png)

### AIの回答が物足りなかったら

AIの返答を読んで「ここがよくわからない」「もっと詳しく知りたい」と感じたら、追加で質問しましょう。具体的に聞くほど、的確な回答が返ってきます。

**良い追加質問の例**:

```
手順の3番目をもっと詳しく教えてください。
特に「どうやってデータを絞り込むか」を、
エクセルの操作手順で説明してください。
```

**悪い追加質問の例**:

```
もっと詳しく教えて
```

→ 「何を」「どう」詳しくしてほしいかが伝わらないと、AIも的確に答えられません。

## 内容を3つのチェックポイントで確かめる

AIが作った手順書を、以下の3つの視点でチェックします。一つでも「いいえ」があれば、次のステップで修正しましょう。

| チェック項目 | OK の基準 | NG の例 |
|---|---|---|
| **手順が具体的か** | 「エクセルのA列からB列までを選ぶ」のように操作が書かれている | 「データを集める」だけで具体的な操作がない |
| **確認ポイントがあるか** | 「合計が前月と比べてどう変わったかを確認する」など正しさの確かめ方がある | 確認方法の記載がない |
| **つまずき対策があるか** | 「データに空欄がある場合は、その行を除外する」のように対処法が書かれている | 「気をつける」だけで具体的な対処法がない |

### 良い手順書と悪い手順書を見比べる

**良い手順書の特徴**: 手順のそれぞれに「**なぜこれをするのか**」が一言添えてある
→ 状況が少し変わっても、理由がわかっているので応用できます

**悪い手順書の特徴**: 操作だけが並んでいて、理由がわからない
→ 手順通りにしか動けず、想定外のデータが来ると対応できません

## 手順書を修正・仕上げする

チェックで気づいた改善点があれば、AIに修正をお願いします。

**修正依頼のコツ**: 「どこを」「どう直したいか」を具体的に伝えることです。

```
手順書の「つまずき対策」に、以下を追加してください。
- データが文字と数字で混ざっている場合の対処法
- ファイルが開けない場合の確認ポイント
```

修正が終わったら、もう一度3つのチェックポイントで確認してください。

## 手順書を保存する

完成した手順書を保存しましょう。

1. AIチャットの回答を**全文コピー**する（Ctrl+A → Ctrl+C、またはCommand+A → Command+C）
2. メモ帳やGoogleドキュメントなどのテキストエディタ（＝文字を入力・保存するソフト）に貼り付ける
3. ファイル名を **「分析手順書＿〇〇＿作成日」** にして保存する（例:「分析手順書＿月次売上＿20260413」）

**保存場所のコツ**: デスクトップに「分析手順書」フォルダを作っておくと、後から探しやすくなります。

## できたか確認する

以下の**すべてにチェック**がつけば、このレッスンは完了です！

- ✅ 「目的」「手順」「確認ポイント」「つまずき対策」の4項目がすべて書かれている
- ✅ 手順が具体的で、次にやるときも同じように進められる
- ✅ 手順の各ステップに「なぜこれをするのか」の理由が添えてある
- ✅ ファイルとして保存し、ファイル名に内容と日付が入っている

おつかれさまでした！ この手順書は、これから何度でも使い回せるあなたの財産です。
テーマを変えてもう一枚作れば、あなた専用の「分析レシピ集」がどんどん育っていきますよ。
