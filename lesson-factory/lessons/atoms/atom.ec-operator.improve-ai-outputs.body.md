# AI出力を改善する

AIに何かを頼んだとき、「うーん、なんか違うな…」と感じたことはありませんか？　たとえばレストランで料理を注文するとき、「パスタください」だけだと、トマトソースが来るかクリームソースが来るかわかりません。「トマトソースのパスタで、辛さは控えめ、チーズ多めで」と伝えれば、ぐっとイメージに近い一皿が届きます。

AIへの指示もまったく同じです。このレッスンでは、AIの出力が思い通りでなかったときに「どこを直してほしいか」を伝えて、出力をどんどん良くしていく方法を学びます。

## このレッスンで身につくこと

- AIの回答を読んで「何が足りないか」を見分けられるようになる
- 追加の指示（フィードバック）を出して、出力を自分の目的に合わせて改善できるようになる
- 最終的に、ECサイト運営で使える実用レベルの文章をAIと一緒に仕上げられるようになる

## 準備するもの

- AIチャットツール（ChatGPT、Claude など）が使える状態であること
- 改善したい題材を1つ決めておくこと（例：商品説明文、お客様へのメール文、SNS投稿文など）

---

## ステップ1：まず「そのまま」頼んでみる

まずはシンプルにAIに頼んでみましょう。ここではECサイトの商品説明文を例にします。

**プロンプト例（最初の指示）：**

```
ハンドメイドの革製トートバッグの商品説明文を書いてください。
```

AIはそれらしい文章を返してくれますが、おそらく「もっとこうしたい」と思う箇所が出てくるはずです。ここで大事なのは、**最初の出力が完璧でなくて当たり前**ということです。

## ステップ2：出力を「3つの視点」でチェックする

AIの回答を受け取ったら、次の3つの視点で読んでみてください。

![出力チェックの3視点](/lesson-assets/atom.ec-operator.improve-ai-outputs/diagram.png)

| チェック視点 | 確認すること | 例 |
|---|---|---|
| **情報の過不足** | 必要な情報が入っているか？余計な情報はないか？ | サイズ・素材・価格帯が抜けている |
| **トーン（文章の雰囲気）** | 読む人に合った言葉づかいか？ | カジュアルすぎる／硬すぎる |
| **具体性** | ふわっとした表現になっていないか？ | 「高品質」→ どう高品質なのか不明 |

## ステップ3：フィードバックを伝えて改善する

チェックで見つけた「直してほしいポイント」を、AIに伝えます。ここが今回のレッスンで一番大事なステップです。

### 良い例と悪い例

**悪い例（あいまいなフィードバック）：**

```
もっと良くしてください。
```

これだと、AIはどこをどう直せばいいかわかりません。レストランで「もっと美味しくして」と言うようなものです。

**良い例（具体的なフィードバック）：**

```
ありがとうございます。次の3点を修正してください。
1. サイズ（縦30cm×横40cm×マチ10cm）と重さ（約500g）を冒頭に追加してください
2. 「上質な」「高品質な」といった抽象的な表現を、「イタリア・トスカーナ産のヌメ革を使用」のように具体的にしてください
3. 文体を「です・ます調」で統一し、30代女性が読みやすい柔らかいトーンにしてください
```

ポイントは、**「何を」「どう」直してほしいかをセットで伝える**ことです。

### フィードバックのテンプレート

迷ったときは、このテンプレートに当てはめてみてください。

```
以下の点を修正してください。
1. 【追加】○○の情報を△△の位置に入れてください
2. 【変更】□□という表現を、◇◇に変えてください
3. 【削除】××の部分は不要なので削除してください
```

## ステップ4：2〜3回やりとりして仕上げる

フィードバック→改善のやりとりは、2〜3回繰り返すのがちょうどいいバランスです。

![改善サイクルの流れ](/lesson-assets/atom.ec-operator.improve-ai-outputs/screen_capture.png)

- **1回目のフィードバック**: 情報の過不足と大きな方向性を修正
- **2回目のフィードバック**: トーンや細かい表現を調整
- **3回目（必要なら）**: 最終チェックと微調整

4回以上やりとりしても良くならない場合は、最初の指示（プロンプト）自体を見直したほうが早いことが多いです。そのときは、新しいチャットを開いて、改善点を最初から盛り込んだ指示をもう一度書いてみましょう。

## ステップ5：完成した文章を保存する

満足のいく出力ができたら、テキストファイルやメモアプリにコピーして保存しましょう。

**保存するときに一緒にメモしておくと便利なもの：**

- 最初に出した指示（プロンプト）
- 特に効果的だったフィードバックの文面
- 次に似たものを作るときのポイント

これを貯めていくと、自分だけの「AIへの指示メモ帳」ができあがり、次回からもっと速く良い出力を得られるようになります。

---

## よくあるつまずきと対処法

| つまずき | 対処法 |
|---|---|
| フィードバックしても同じような文が返ってくる | 修正指示の数を絞る（1回に1〜2点）。一度に多すぎるとAIが混乱します |
| 途中で話がずれていく | 「最初の商品説明文の修正に戻ってください」と明示する |
| どこが悪いかわからない | 3つの視点（情報・トーン・具体性）に戻って1つずつチェックする |
| 完璧を目指して終わらない | 「80点で十分」と割り切る。公開後に修正もできます |

## 確認してみよう

以下ができていれば、このレッスンは完了です。

- [ ] AIに最初の指示を出して、出力を受け取った
- [ ] 3つの視点（情報・トーン・具体性）で出力をチェックした
- [ ] 具体的なフィードバックを1回以上伝えて、改善された出力を得た
- [ ] 完成した文章を保存した
