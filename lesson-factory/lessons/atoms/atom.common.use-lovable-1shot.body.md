# Lovable に「作りたい」を 1 文書いてアプリを得る

> **Status**: skeleton — `/lesson-improve` で本文化予定
>
> 想定 persona: `web-builder` / `ai-app-builder` / `p-noneng-webapp`（"アイデアだけ持っている人" を最短で動かす）

## なぜこれをやる？

「アイデアはあるけど、フレームワーク選び・DB 接続・認証で止まる」――この層に最も効くのが Lovable。
1 文で目的を投げると、UI・データ保存・必要であれば認証まで一括で組み立ててくれる。
仕上がりはそのままプロトタイプとして他人に共有できる URL になる。
ゴールは「アイデアを 1 文で書く → 30 分後に他人に触ってもらえる」体験を 1 周まわすこと。

## 何を準備する？

- Lovable のアカウント（GitHub 連携で OK、無料枠あり）
- 1 文の「誰が・何のために・どう使うか」メモ
- 共有相手 1 人（家族・友人）の存在（フィードバック収集用）

## やってみよう

1. Lovable を開き、ホーム画面のプロンプト欄に 1 文で要件を投入
2. 出てきたプレビューを触り、欲しい挙動と違う部分を「〇〇を××にして」と追加プロンプトで修正（最大 3 ターン）
3. 「Publish」/「Share」ボタンで公開 URL を取得
4. その URL を共有相手 1 人に送って実際に触ってもらう
5. 返ってきたフィードバックを 3 行でメモして atom 完了

## できたかチェック

- [ ] 公開 URL を他人がブラウザで触れる
- [ ] 1 つ以上の機能（フォーム送信・一覧表示など）が動く
- [ ] 他人から 1 件以上の感想 / 改善点を得た

## つまづいたら

- **要件を盛りすぎて壊れる** → 「最初の 1 画面だけ」で確定してから次の機能を頼む
- **UI が英語で出る** → プロンプトに「日本語 UI で」を明示
- **認証が必要かわからない** → 最初は認証なしで作り、後から「ログインを追加」と頼む
