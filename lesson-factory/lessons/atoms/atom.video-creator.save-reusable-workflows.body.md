# 再利用できる制作手順を残す

たとえば料理を思い出してみてください。初めて作ったレシピをメモに残しておけば、次回からも同じ味で作れますよね。動画制作もまったく同じです。一度うまくいった手順を「テンプレート（＝何度も使い回せるひな形）」として残しておけば、次の動画づくりがぐっとラクになります。

しかも、AIツールを使った動画制作では「AIに入力した指示文（プロンプト）」こそが再利用のカギになります。うまくいったプロンプトをそのまま保存しておけば、まるでレシピカードを見て料理するように、何度でも同じクオリティの動画が作れるのです。

この Atom では、あなたが一度成功した制作工程を、次回からも使い回せる手順書（テンプレート）としてまとめる方法を学びます。所要時間は約15分です。

![制作手順テンプレートの全体像](/lesson-assets/atom.video-creator.save-reusable-workflows/diagram.png)

## 前提を確認する

この Atom に取り組む前に、次の条件を満たしているとスムーズです。

- すでにAIツールを使って、少なくとも1本の動画を完成させた経験がある
- パソコンでテキストファイルを作成・保存できる

まだ動画を1本も作ったことがない場合は、先にトラック内の他の Atom で制作体験を積んでから戻ってきてください。

## 制作工程を書き出す

まずは、あなたがこれまでに動画を作ったときの工程を振り返りましょう。ここでは ChatGPT や Claude などのAIチャットツールに手伝ってもらうと、抜け漏れなく思い出せます。

**AIに聞いてみるプロンプト例：**

> 「私は YouTube のショート動画を以下の流れで作りました。抜けている工程がないか確認して、番号付きリストに整理してください。
> ①台本をAIに書かせた → ②音声を生成した → ③画像素材を作った → ④編集ソフトで合成した」

**良い例**: 「①AIに台本を書かせる → ②音声を生成する → ③画像を揃える → ④編集ソフトで組み合わせる → ⑤サムネイルを作る」と、工程を番号付きで書き出す

**悪い例**: 「なんとなくAIを使って動画を作った」と、漠然とした記憶だけに頼る

紙でもメモアプリでもよいので、5〜7ステップ以内で工程を書き出してみてください。多すぎると後で使わなくなるので、大きな区切りだけにするのがコツです。

## 各工程のプロンプトを書き残す

次に、それぞれの工程でAIに入力した指示文（プロンプト）を、そのままコピーして残します。

たとえば台本を作る工程なら：

> 「YouTube向けの60秒ショート動画の台本を書いて。テーマは〇〇。語り口は親しみやすく。冒頭3秒でフックを入れて。」

このように、具体的な指示文をそのまま保存するのがコツです。

**AIに聞いてみるプロンプト例（プロンプトの改善）：**

> 「以下のプロンプトをもっと具体的にしてください。出力の文字数、トーン、形式を指定するように直してほしいです。
> 元のプロンプト：『台本を書いて』」

**良い例**: 実際に使って結果がよかったプロンプトを一字一句そのまま残す。使ったAIツール名（ChatGPT、Claude など）も添える

**悪い例**: 「なんかいい感じに台本を書かせた」だけ書いて、プロンプト本体を残さない

## テンプレートにまとめる

振り返った工程とプロンプトを、ひとつのテンプレート（ひな形）にまとめます。以下の構成で書くと、次回すぐに使えて便利です。

```
## テンプレート名：〇〇動画の制作手順
## 最終更新日：2026-04-13
## 想定完成時間：約〇〇分

### ステップ1：台本作成
- 使うツール：ChatGPT（または Claude）
- プロンプト：「……」
- 確認ポイント：動画の長さ（60秒以内か）、トーン（親しみやすいか）

### ステップ2：音声生成
- 使うツール：〇〇
- 設定：言語＝日本語、話速＝ふつう
- 確認ポイント：聞き取りやすさ、イントネーション

### ステップ3：画像・映像素材の準備
- 使うツール：〇〇
- プロンプト：「……」
- 確認ポイント：解像度、著作権の問題がないか

### ステップ4：編集・合成
- 使うツール：〇〇
- 注意点：BGMの音量バランス、テロップの読みやすさ

### ステップ5：書き出し・公開前チェック
- 書き出し形式：MP4、1080p
- 公開前確認：サムネイル、タイトル、概要欄
```

**AIに聞いてみるプロンプト例（テンプレート作成の補助）：**

> 「以下の動画制作ステップを、再利用しやすいテンプレートに整形してください。各ステップに『使うツール』『プロンプト』『確認ポイント』の項目を入れてください。
> ①台本作成 ②音声生成 ③画像準備 ④編集」

## テンプレートを保存する

まとめた手順書を、次回も見つけやすい場所に保存します。

**良い例**: 「動画制作テンプレート_ショート動画.md」というファイル名で、「動画テンプレート」フォルダに保存する

**悪い例**: デスクトップに「無題.txt」のまま置いて、後で探せなくなる

おすすめの保存場所：
- Google ドキュメント（スマホからもすぐ見られる）
- Notion（テンプレート管理に向いている）
- パソコンの専用フォルダ（「動画テンプレート」など分かりやすい名前をつける）

保存場所は1か所だけに決めるのがポイントです。あちこちに散らばると、結局どれが最新か分からなくなります。

## テンプレートを試して改善する

保存したテンプレートを見ながら、実際にもう一度動画を作ってみます。このステップが一番大事です。テンプレートを使って初めて「手順が足りなかった」「プロンプトが曖昧だった」という漏れに気づけます。

試したあとは、気づいた点をテンプレートに書き足しましょう。とくに以下を意識してください。

- 抜けていたステップはないか？
- プロンプトの結果が期待と違った箇所はないか？
- 確認ポイントに追加すべき項目はないか？

## つまずきポイントを対策する

| つまずき | 原因 | 対策 |
|---------|------|------|
| プロンプトの結果が毎回違う | 指示が曖昧で、AIが解釈を変えている | プロンプトに条件を具体的に書き足す（文字数、トーン、形式、例文など） |
| 手順が長すぎて使わなくなる | すべてを1つのテンプレートに詰め込んでいる | 工程を5〜7ステップに絞る。細かい設定は「設定メモ」として別枠にする |
| どのファイルに保存したか忘れる | 保存場所がバラバラ | テンプレート専用の場所を1か所だけ決めて、そこにまとめる |
| テンプレートが古くなって使えない | 更新の習慣がない | テンプレート冒頭に「最終更新日」を入れ、使うたびに日付を更新する |

## 完成を確認する

以下のチェックリストをすべて満たしていれば、このAtomは完了です。

- [ ] 制作工程が5〜7ステップの番号付きリストになっている
- [ ] 各ステップに「使うツール」と「プロンプト（またはやること）」が書かれている
- [ ] テンプレートにファイル名をつけて、決めた場所に保存できた
- [ ] 保存したテンプレートを見ながら、実際にもう一度制作を試した（全工程でなくても、最初の2ステップだけでもOK）

すべてチェックできたら、テンプレートの画面をスクリーンショットで撮って証拠として残しましょう。
