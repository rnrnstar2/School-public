# コピーのABテスト案を設計する

## このAtomでできるようになること

あなたはこのAtomを終えると、キャッチコピー（＝商品やサービスを紹介する短い文句）の「どっちがいいか」を確かめるテスト計画を、AIツールと一緒に自分で組み立てられるようになります。完成までの目安は**15分**です。

## たとえでイメージする

レストランのメニューを思い浮かべてください。「本日のスペシャル」と書くのと「シェフこだわりの一品」と書くのでは、どちらでお客さんがもっと注文してくれるでしょうか？ 両方を実際に出してみて注文数を比べる——それがABテスト（＝2つの案を同時に試して良い方を見つける方法）の基本的な考え方です。

このAtomでは「テスト計画を立てる」ところまでを扱います。実際にテストを走らせるのは次のステップです。

![ABテスト設計フロー](/lesson-assets/atom.ai-marketer.copy-tests-design/diagram.png)

## 前提を確認する

このAtomを進める前に、次の準備が整っているとスムーズです。

- 比べたいキャッチコピー案がすでに2つ以上ある（AIツールで出してもOK）
- 比べる対象が1つ決まっている（例：メールの件名、広告の見出し、LPのボタン文言）

まだコピー案がない場合は、ChatGPTやClaudeに「○○のキャッチコピーを5案出して」と頼めばすぐに用意できます。

## ステップ1：テストの目的を決める

まず「何を比べたいのか」を**1つだけ**に絞ります。ここが曖昧だと結果が出ても「で、何がわかったの？」となってしまいます。

### 良い例
- 「メールの件名で『期間限定』と『今だけ』を使い、開封率を比べる」
- 「広告見出しで『簡単3ステップ』と『たった5分で完了』を使い、クリック数を比べる」

### 悪い例
- 「なんとなく2つの広告を比べる」→ 何を測るかが不明
- 「件名も本文も画像も全部変える」→ 変更点が多すぎて原因が分からない

**鉄則：変える部分は1つだけ。** 料理で言えば、塩を変えたときの味を見たいのに、塩も砂糖も油も全部変えてしまっては何が効いたか分かりません。

## ステップ2：測る指標を選ぶ

ABテストでは「どの数字を見て判断するか」をあらかじめ決めておきます。

| 指標名 | 意味 | 向いているテスト |
|--------|------|------------------|
| **開封率** | 送ったメールのうち開かれた割合 | メール件名のテスト |
| **クリック率** | 見た人のうちリンクを押した割合 | 広告見出し・ボタン文言のテスト |
| **成約率** | 見た人のうち購入など行動をとった割合 | LP全体のコピーテスト |

最初は**指標を1つだけ**選びましょう。複数見たくなりますが、まずは「この数字が上がったら成功」と言い切れる状態にします。

## ステップ3：AIにテスト案のたたき台を作ってもらう

ここからAIツールの出番です。ChatGPTやClaudeを開いて、次のプロンプト（＝AIへの指示文）をコピー＆ペーストしてください。

```
あなたはマーケティングのABテスト設計のプロです。
以下の条件でABテスト案を作成してください。

【条件】
- テスト対象：（例：メールマガジンの件名）
- 現在の文言（案A）：（例：「期間限定のお知らせ」）
- 変えたい方向性：（例：もっと緊急感を出したい）
- 想定読者：（例：30代女性、美容に関心あり）

【出力してほしいもの】
1. 案B（新しいコピー案を3つ）
2. おすすめの指標（1つ）
3. テスト期間の目安
4. 判断基準（案Bを採用する条件）
5. テストの注意点
```

**AIから返ってきた案を鵜呑みにしないこと。** あなたの商品やお客さんのことは、あなたが一番よく知っています。AIの提案は「たたき台」として使い、違和感があれば修正してください。

![AIでテスト案を作成する画面](/lesson-assets/atom.ai-marketer.copy-tests-design/screen_capture.png)

## ステップ4：テスト案を7項目で整理する

AIの出力をもとに、次の7項目を埋めてテスト案を完成させます。スプレッドシートでもメモ帳でも構いません。

1. **テスト名** — 何のテストか分かる短い名前
2. **対象** — メールの件名、広告の見出し、ボタン文言など
3. **案A（いまの案）** — 現在使っている文言
4. **案B（新しい案）** — 変えたい部分だけ変えた文言
5. **指標** — 開封率、クリック率など（1つ）
6. **期間** — どのくらいテストするか（例：3日間）
7. **判断基準** — どちらが良かったと判断するか（例：案Bが案Aより5%上回れば採用）

### 記入例

| 項目 | 内容 |
|------|------|
| テスト名 | メール件名「期間限定」vs「今だけ」 |
| 対象 | メールマガジンの件名 |
| 案A | 「期間限定のお知らせ」 |
| 案B | 「今だけのお知らせ」 |
| 指標 | 開封率 |
| 期間 | 3日間（配信100通以上が目安） |
| 判断基準 | 案Bが案Aより開封率3%以上上回れば案Bを採用 |

### 悪い例
- 案Aと案Bで件名も本文も画像も全部変えている → どの変更が効いたか分からない
- 期間が「1通だけ送る」→ データが少なすぎて判断できない
- 判断基準が「なんとなく良さそうな方」→ 感覚になってしまい再現性がない

## ステップ5：AIにレビューしてもらう

7項目を埋めたら、もう一度AIに見てもらいましょう。次のプロンプトを使います。

```
以下のABテスト案をレビューしてください。
マーケティングのベストプラクティスに照らして、改善すべき点があれば具体的に教えてください。

【テスト案】
（ここに7項目を貼り付ける）

特に以下の観点でチェックしてください：
- 案Aと案Bの違いは1カ所だけか
- 指標は適切か
- テスト期間は十分か（必要なデータ数が集まるか）
- 判断基準は明確か
```

AIが「サンプルサイズ（＝テストに必要なデータの件数）が足りないかも」と指摘したら、期間を長めに設定し直しましょう。目安として、**最低でも各案50件以上のデータ**が集まる期間を確保します。

## 成果物をまとめる

完成したテスト案のシートが、このAtomの成果物です。次の内容が入っていれば合格です。

- 7項目（テスト名・対象・案A・案B・指標・期間・判断基準）が揃っている
- 変える部分が1カ所だけになっている
- 判断基準が具体的な数字で書かれている

スクリーンショット（＝画面の写真）を撮って提出してください。

## 確認する：セルフチェックリスト

提出前に、次のポイントを1つずつ確認しましょう。すべてチェックがついたら完了です。

- [ ] 案Aと案Bは**1カ所だけ**違うか？
- [ ] 指標は**1つ**に決まっているか？
- [ ] 期間中にどちらの案も**同じ条件**で届く（または表示される）ようになっているか？
- [ ] 判断基準が**具体的な数字**（例：「○%以上」）で書かれているか？
- [ ] AIツールに一度はレビューしてもらったか？

## つまずき対策

| つまずき | 対策 |
|----------|------|
| 何をテストすればいいか思いつかない | いま使っているキャッチコピーをそのまま案Aにする。AIに「この案Aに対して、もっと○○なニュアンスの案Bを3つ提案して」と聞く |
| 結果が同じになりそうで不安 | 文言のニュアンスを大きく変える。「簡単」vs「プロも使う」のように対照的な表現にすると差が出やすい |
| テスト期間の決め方が分からない | メールなら送信後3日、広告なら1週間が目安。AIに「○○人に配信する場合、何日テストすればよいか」と聞くと具体的な回答が返る |
| AIの回答が漠然としている | プロンプトに「具体的な数字を入れて」「改善点を3つ挙げて」と条件を書き足す |
| 案Bを何パターンも作りたくなる | まず1組（AとB）だけで回す。結果が出てから次の案Cを試す方が、学びが着実に積み上がる |
