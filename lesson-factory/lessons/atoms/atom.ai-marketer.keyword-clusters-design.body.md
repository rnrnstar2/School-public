# キーワードクラスターを設計する

キーワードクラスター（＝関連する検索ワードのまとまり）を作ると、ブログやWebサイトで「何について書けばいいか」がすっきり整理できます。

たとえばスーパーで買い物するときを想像してください。「今夜はカレー！」と決めたら、にんじん・じゃがいも・玉ねぎ・カレールーをまとめてカゴに入れますよね。バラバラに棚を探すのではなく、「カレーをつくる」という目的で材料をまとめる――これがクラスターの考え方です。

検索キーワードも同じです。「カレー レシピ」「カレー 人気」「カレー 簡単」というワードは、どれも「カレーについて知りたい人」が打つ言葉のまとまりです。このまとまりを意識して記事を計画するのが、キーワードクラスター設計です。

![キーワードクラスター設計フロー](/lesson-assets/atom.ai-marketer.keyword-clusters-design/diagram.png)

## 目標を確認する

このレッスンを終えると、あなたは次のことができるようになります。

- ひとつのテーマについて、関連する検索ワードをまとまり（クラスター）に整理できる
- そのクラスター設計書を、記事の企画に使える形でドキュメントにまとめられる

**完成イメージ:** 1つのメインキーワードに対して3〜5個のクラスターがあり、それぞれに3〜10個の検索ワードが入った一覧表ができあがります。所要時間は約15分です。

## テーマを決める

まずは、あなたが記事で扱いたい大テーマを1つ決めます。これを「メインキーワード」と呼びます。

### よい例
- 「初心者 筋トレ」――具体的で、関連ワードが豊富
- 「犬 お留守番」――読者の悩みが明確
- 「一人暮らし 節約術」――検索する人の状況がイメージしやすい

### 悪い例
- 「筋トレ」――広すぎて、どこから手をつけるか迷う
- 「健康」――テーマが大きすぎてまとまらない
- 「ライフハック」――意味が広く、クラスターの切り分けが困難

**ポイント:** 自分が詳しく書けそうなテーマを選びましょう。「なんとなく知っている」程度だと、クラスターの切り分けが難しくなります。

## AIで関連キーワードを集める

メインキーワードが決まったら、AIを使って関連ワードを一気に集めます。ChatGPT、Claude、どちらでもかまいません。

次のプロンプト（＝AIへの指示文）をコピーして、AIのチャット画面に貼り付けてください。「〇〇」の部分だけ、あなたのメインキーワードに置き換えます。

```
「〇〇」について、Googleで検索されそうな関連キーワードを30個リストアップしてください。
検索する人の目的（情報収集、購入検討、やり方を知りたい等）ごとに分類してください。
```

**AIの返事が返ってきたら、ここをチェックしてください。**

- ✅ キーワードが20個以上出ていますか？（少なすぎたら「あと10個追加で出して」と聞く）
- ✅ メインキーワードと明らかに関係ないものが混ざっていませんか？（あれば手動で削除）

### よい例
- 「初心者 筋トレ メニュー」「筋トレ 何から始める」など、目的が似ているワードが自然に集まっている

### 悪い例
- メインキーワードが「筋トレ」なのに「病院 整形外科」が大量に混ざっている（AIに「メインキーワードから離れないで」と指示し直す）

## キーワードをクラスターにまとめる

集めたキーワードを、意味のまとまりごとにグループ分けします。

冷蔵庫の食材を「今日のカレー用」「明日のサラダ用」「おやつ用」と分けるイメージです。分け方の基準は「検索した人の目的が同じかどうか」です。

AIに分類を頼みましょう。次のプロンプトをコピーして使ってください。

```
以下のキーワードを、検索した人の目的が同じもの同士で3〜5つのグループに分けてください。
それぞれのグループに短い名前をつけてください。
グループの数は3〜5個に収めてください。

[ここに前のステップでAIが出力したキーワード一覧をそのまま貼り付ける]
```

![AIでクラスター分類した画面例](/lesson-assets/atom.ai-marketer.keyword-clusters-design/screen_capture.png)

### よいクラスター分けの例
- クラスター名「筋トレの始め方」：筋トレ 初日／筋トレ 何から／筋トレ 1日目 メニュー
- クラスター名「筋トレの効果」：筋トレ 効果 いつから／筋トレ 痩せる／筋トレ 体力 向上

### 悪いクラスター分けの例
- 1つのクラスターに20個以上のキーワードが入っている（まだ細かく分けられる）
- クラスター名が「その他」になっている（分類の基準があいまい）

**うまくいかないときのプロンプト例:**

```
クラスター「[名前]」のキーワードが多すぎます。
このクラスターをさらに2〜3個のサブグループに分けてください。
```

## クラスターの品質を確認する

あなたのクラスターが次の条件をすべて満たしているかチェックしてください。

- ✅ クラスターは合計3〜5個に収まっている
- ✅ 各クラスターに3〜10個のキーワードが入っている
- ✅ クラスター名を見ただけで「どんな検索意図の人向けか」がすぐわかる
- ✅ メインキーワードと関係ないクラスターが混ざっていない
- ✅ 同じキーワードが2つ以上のクラスターに重複して入っていない

ひとつでもチェックが外れたら、該当箇所をAIに相談して調整してください。

```
以下のクラスター設計をレビューしてください。
各クラスターが3〜10個のキーワードに収まっているか、
重複がないか、名前がわかりやすいかを確認して、
改善点があれば提案してください。

[ここにクラスター設計を貼り付ける]
```

## 成果物をまとめる

次のテンプレートを使って、クラスター設計書を1つのドキュメント（＝文書）にまとめます。メモ帳、Googleドキュメント、Notionなど、どのツールでもかまいません。

```
# キーワードクラスター設計書
## メインキーワード：〇〇〇〇

### クラスター1：[クラスター名]
- キーワード1
- キーワード2
- キーワード3

### クラスター2：[クラスター名]
- キーワード1
- キーワード2
- キーワード3

### クラスター3：[クラスター名]
- キーワード1
- キーワード2
- キーワード3
```

このドキュメントを保存したら、画面のスクリーンショット（＝画面の写真）を撮ってください。そのスクリーンショットがこのレッスンの成果物です。

**スクリーンショットの撮り方:**
- Mac：`Command + Shift + 3`（画面全体）または `Command + Shift + 4`（範囲選択）
- Windows：`Windows + Shift + S`（範囲選択）

## つまずきポイントを知っておく

| つまずき | 対策 |
|---------|------|
| キーワードが少なすぎる（20個未満） | AIに「あと10個追加で出して」と追加で聞く |
| クラスターが多すぎる（6個以上） | 似ているクラスターをひとつにまとめる。AIに「これとこれは統合できますか？」と聞く |
| クラスターが少なすぎる（2個以下） | 「検索する人の目的」がもっと細かく分かれないか再検討する |
| どのクラスターに入れるか迷うワードがある | 「このワードで検索した人は何を知りたいか？」を考えて判断する。迷ったらAIに聞く |
| AIがテーマと関係ない方向に広げてしまう | 「メインキーワード『〇〇』の範囲内で答えて」と指示し直す |
| 英語のキーワードが混ざって出てくる | 「日本語のキーワードだけでお願いします」と伝える |

## まとめ

おつかれさまでした！ あなたは今、1つのテーマについてキーワードをまとまりごとに整理する力を手に入れました。この設計書があれば、「次にどんな記事を書けばいいか」が一目でわかるようになります。

次のステップとして、各クラスターから実際に記事を1本書いてみると、学んだことがさらに定着します。
