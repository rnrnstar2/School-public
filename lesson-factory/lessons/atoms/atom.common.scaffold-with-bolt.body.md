# Bolt.new でブラウザ完結の Web アプリを作る

> **Status**: skeleton — `/lesson-improve` で本文化予定
>
> 想定 persona: `web-builder` / `ai-app-builder` / `p-noneng-webapp`（環境構築で挫折しない最短ルート）

## なぜこれをやる？

「Web アプリを作りたい」と思っても、ローカル環境構築 (Node / npm / git) で半日溶かすのは典型的な離脱点。
Bolt.new はブラウザだけで完結する。プロンプトを送ると、その場で
ファイルツリー・ターミナル・プレビューの 3 ペインが立ち上がり、AI が編集していく様子が見える。
「動く何か」をスマホからでも触れる URL として手に入れるのがゴール。

## 何を準備する？

- Bolt.new アカウント（無料枠あり）
- 「作りたいもの」を 1 段落でメモ（誰が・何を・どんな画面で）

## やってみよう

1. Bolt.new を開き、最初のプロンプトに「作りたいアプリ」を投入（例: 「読書記録アプリ。タイトル・著者・読了日を登録、一覧表示」）
2. 生成中はファイルツリーを眺めるだけで OK。完了したらプレビューで触る
3. 動かない箇所は「〇〇が動かない」と日本語でそのまま追記し、AI に直してもらう
4. 完成形を Bolt の「Deploy」 or `Download` で手元に保存
5. 動作確認の URL とスクショを成果物として保存

## できたかチェック

- [ ] プレビュー画面で意図した操作が 1 つ以上動く
- [ ] エラー時に AI に投げ直して 1 回以上修正できた
- [ ] 自分が「これを作った」と説明できる 1 段落のメモがある

## つまづいたら

- **無料枠を一気に使い切る** → 大規模な機能を一度に頼まず、画面 1 つずつに分割
- **生成途中で止まる** → ブラウザを再読込せず、Bolt の Retry を押す
- **何を作ればいいか思い付かない** → 自分の身近な「メモ・記録・予約」系を 1 つ選ぶ
