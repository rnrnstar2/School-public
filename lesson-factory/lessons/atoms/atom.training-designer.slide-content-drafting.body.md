# スライド本文をAIで下書きする

プレゼンの資料を作るとき、「白い画面を見つめて何から書き始めればいいかわからない」という経験はありませんか？それは料理でいうと、食材が冷蔵庫にあるのにレシピを思い出せない状態に似ています。

AIチャットツール（＝コンピューターと会話しながら文章を作れる仕組み）をうまく使えば、その「レシピ」を短時間で書き出せます。このレッスンでは、15分程度でスライドの本文をAIに下書きしてもらう方法を一緒に進めます。

![スライド下書きの全体フロー](/lesson-assets/atom.training-designer.slide-content-drafting/diagram.png)

## 前提を確認する

始める前に、次の2つを用意してください。

- **スライドのテーマが決まっていること**（例：「新入社員向けのマナー研修」「チームの月次報告」など）
- **AIチャットツールが使えること**（ChatGPT、Claude、Copilot など。無料版で構いません）

どちらも準備できたら次へ進みます。

## AIに「役割」と「目的」を伝える

AIに話しかけるとき、最初にやることは「あなたは誰で、何をお願いしたいか」をはっきり伝えることです。この指示文のことをプロンプト（＝AIへのお題）と呼びます。

次のテンプレートをコピーして、AIの入力欄に貼り付けてください。太字の部分をあなたのテーマに書き換えます。

```
あなたは研修デザインの専門家です。
**新入社員**向けに「**ビジネスマナーの基本**」というタイトルで
**6枚**のスライド資料を作ります。
各スライドに書く本文を、**1スライドあたり3〜5個の箇条書き**で提案してください。
```

この1行で「AIに専門家の役割を依頼し、対象読者・枚数・形式を指定する」ことをしています。

### 良い例と悪い例を比べる

| | プロンプト | なぜ？ |
|---|---|---|
| **良い例** | 「新入社員向けに6枚、各3〜5箇条書きで」 | 対象読者・枚数・形式の3つが揃っている |
| **悪い例** | 「スライドを作って」 | AIが勝手にテーマや枚数を決めてしまう |
| **悪い例** | 「最高のプレゼンにして」 | 「最高」の基準が曖昧でAIが困る |

### プロンプトのアレンジ例

テーマに応じて、以下のように書き換えると便利です。

- 社内報告の場合：「あなたはデータ分析の専門家です。経営層向けに『2024年Q3の売上報告』を5枚で…」
- ワークショップの場合：「あなたはファシリテーターです。参加者10名のチームビルディング研修を8枚で…」

## AIの出力を確認する

AIから返ってきた文章をざっと読みます。ここでチェックするのは次の3点です。

1. **スライドの枚数は合っているか**
2. **専門用語は少なめで、読みやすい言葉になっているか**
3. **話の流れが自然か**（前のスライドの話題が次につながっているか）

![AI出力結果の確認画面](/lesson-assets/atom.training-designer.slide-content-drafting/screen_capture.png)

### 修正をお願いするときのプロンプト例

もし気になる点があれば、AIにこう伝えます。

- 専門用語が多いとき → `専門用語を使わず、中学生でもわかる言葉で書き直してください。`
- 枚数がずれたとき → `ちょうど6枚になるように調整してください。`
- もっと具体例がほしいとき → `各スライドに1つずつ具体例を加えてください。`
- 文章が長すぎるとき → `各箇条書きを20文字以内に短くしてください。`

AIとのやりとりは「1回で完璧」を目指さなくて大丈夫です。2〜3回のやりとりで仕上げるのが普通です。

## 自分の言葉に整える

AIの出力をそのまま使うのではなく、あなたの現場に合わせて手直しします。これが一番大切なステップです。AIは「下書き係」、あなたは「最終編集者」と考えましょう。

手直しのポイント：

- **会社独自のルールや用語があれば差し替える**（例：「顧客」→ 自社では「お客さま」と呼ぶ）
- **長すぎる文は2つに分ける**（1つの箇条書きに情報を詰め込みすぎない）
- **箇条書きは1項目1行にまとめる**
- **数字や固有名詞は正確か確認する**（AIは事実と違う数字を出すことがあります）

### 良い手直しの例

| AIの出力 | 手直し後 |
|---|---|
| 「適切な敬語を使用することが重要です」 | 「まず相手の名前に『さん』をつけましょう」 |
| 「コミュニケーションスキルを向上させる」 | 「報告は結論→理由→補足の順で伝える」 |

## 成果物を保存する

整えた文章をMarkdown形式（＝見出しや箇条書きを記号で表す書き方）のテキストファイルとして保存します。

1. パソコンのメモ帳（Windows）やテキストエディット（Mac）を開く
2. AIとのやりとりで完成した内容を貼り付ける
3. `slide-draft.md` という名前で保存する

これがこのレッスンの成果物（＝完成した下書き）です。

## 検証する

保存したファイルを開いて、次のチェックリストで自己採点してください。すべて「はい」なら合格です。

- [ ] スライドの枚数が最初に決めた数と一致している
- [ ] 1枚のスライドにつき3〜5個の箇条書きになっている
- [ ] 最初から最後まで読んで「話が飛んでいる」と感じる箇所がない
- [ ] 読み手の知識レベルに合った言葉遣いになっている
- [ ] 自分の言葉で手直しした箇所が1つ以上ある

## つまずきやすいポイントを知っておく

| つまずき | 原因 | 対策 |
|---|---|---|
| AIが長すぎる文章を返す | 形式の指定が足りない | 「各スライド3行以内で」と追加で伝える |
| 枚数が合わない | 枚数指定が曖昧 | 「ちょうど6枚でお願いします」と明示する |
| 日本語が不自然 | AIの癖 | 該当部分を自分の言葉で書き直す |
| 内容が一般的すぎる | 具体的な情報を与えていない | 「対象は入社1年目の営業職」のように詳細を足す |
| AIが事実と違うことを書く | AIの「幻覚」 | 数字や固有名詞は必ず自分で確認する |
