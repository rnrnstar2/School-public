# スライド構成案をAIに出させる

研修やプレゼンのスライドづくりで、いちばん頭を悩ませるのが「何枚目に何を話すか」という構成の部分です。白紙の画面を前に手が止まった経験は、きっとあなたにもあるのではないでしょうか。

このレッスンでは、AIチャットツール（＝AIと会話できるウェブサービス。ChatGPTやClaudeなど）を使って、15分以内にスライド構成案を作れるようになります。料理でいえば、「レシピの目次」をAIに考えてもらうイメージです。あなたはテーマと条件を伝えるだけでOKです。

---

## スライド構成案をイメージする

レストランでコース料理を注文するときのことを想像してみてください。前菜、スープ、メイン、デザートと、食べる順番が決まっていますよね。スライドも同じで、聞き手を飽きさせず、伝えたいメッセージを最後まで届けるための「流れの設計図」が必要です。

この設計図＝構成案をAIに提案してもらうのが、今回のゴールです。

## 前提をそろえる

このレッスンを始める前に、次の2つを用意してください。

- **伝えたいテーマ**: 「新入社員向けの業務マナー」など、何について話すかが決まっていること
- **AIチャットツール**: ChatGPT（chat.openai.com）、Claude（claude.ai）、Gemini（gemini.google.com）など、ブラウザで使えるAIサービスのどれか一つにアクセスできること

どのツールでも構いません。無料プランで十分です。使いやすいものを一つ選んで、ブラウザで開いておいてください。

---

## AIにプロンプト（＝お願い文）を送る

AIにスライド構成案を作ってもらうには、あなたの要望を文章で伝えます。この要望文のことを「プロンプト」と呼びます。

### ステップ1: テンプレートをコピーする

以下のテンプレートをそのままコピーして、AIチャットの入力欄に貼り付けてください。`【 】`の部分だけ、あなたの内容に書き換えます。

```
以下の条件で、プレゼンテーション用スライドの構成案を作ってください。

テーマ: 【あなたのテーマを書く。例: 新入社員向けのビジネスメール基礎】
聞き手: 【誰が聞くか。例: 入社1ヶ月目の新入社員20名】
目的: 【何を持ち帰ってもらいたいか。例: ビジネスメールの基本フォーマットとよくあるミスを理解してもらう】
時間: 【発表時間。例: 15分】
スライド枚数: 【希望枚数。例: 10〜12枚程度】

各スライドには以下を含めてください:
- スライド番号とタイトル
- 盛り込むべき要点（箇条書き3つまで）
- そのスライドで使うと効果的な図やイラストの提案
```

### ステップ2: 送信して結果を待つ

テンプレートの `【 】` を書き換えたら、送信ボタン（紙飛行機アイコンや矢印ボタン）を押します。数秒〜十数秒で構成案が返ってきます。

### 良いプロンプトの例

```
以下の条件で、プレゼンテーション用スライドの構成案を作ってください。

テーマ: 新入社員向けのビジネスメール基礎
聞き手: 入社1ヶ月目の新入社員20名
目的: ビジネスメールの基本フォーマットとよくあるミスを理解してもらう
時間: 15分
スライド枚数: 10〜12枚程度

各スライドには以下を含めてください:
- スライド番号とタイトル
- 盛り込むべき要点（箇条書き3つまで）
- そのスライドで使うと効果的な図やイラストの提案
```

→ 「誰に」「何を」「どのくらいで」が明確なので、AIもピンポイントで回答できます。

### 悪いプロンプトの例

```
スライドの構成案を作って
```

→ 何について、誰に、何枚で作るのかがわからないので、AIからも当たり障りのない回答しか返ってきません。

**覚えておくコツ**: 「誰に」「何を」「どのくらいの分量で」の3つを書くだけで、回答の精度がぐんと上がります。

## AIの回答を確認する

AIが返してくれた構成案を、次の3つの観点でチェックしましょう。

### チェック1: 冒頭にゴール提示があるか

聞き手が「今日何を学ぶのか」を最初のスライドで把握できる構成になっているか確認します。

### チェック2: 本論が2〜4つのブロックに分かれているか

ひとつのスライドに情報を詰め込みすぎていないか見ます。1スライド=1メッセージが基本です。

### チェック3: まとめと次のアクションで終わっているか

最後に「今日話したこと」と「次にやること」がある構成になっているか確認します。

### 良い構成案の例

| スライド | 内容 |
|---|---|
| 1枚目 | あいさつ・今日のゴール |
| 2〜4枚目 | ビジネスメールの基本フォーマット |
| 5〜8枚目 | よくあるミスと直し方（具体例付き） |
| 9枚目 | まとめ |
| 10枚目 | 次のアクション・質問タイム |

### 悪い構成案の例

- すべてのスライドが「メールの書き方について」のような同じ見出しの繰り返し
- 1枚のスライドに箇条書きが7〜8個詰め込まれている

## AIに修正をお願いする

最初の回答で完璧なものが来るとは限りません。同じチャット画面でそのまま追加の指示を送れば、AIが修正してくれます。

### よく使う修正プロンプト

| やりたいこと | プロンプト例 |
|---|---|
| 特定のスライドを詳しくしたい | 「スライド3の内容をもう少し詳しくしてください」 |
| 具体例を足したい | 「スライド5に、悪いメールの具体例を2つ追加してください」 |
| 枚数を減らしたい | 「時間が15分なので、スライドを8枚に減らしてください」 |
| 聞き手のレベルを変えたい | 「聞き手を中堅社員に変えて、内容を調整してください」 |
| 別の切り口を試したい | 「ワーク（体験型の作業）を1つ入れた構成に変えてください」 |

AIは会話の文脈を覚えているので、最初のプロンプトを送り直す必要はありません。追加の指示だけ送ればOKです。

## 成果物を保存する

満足のいく構成案ができたら、以下の手順で保存しましょう。

1. AIの回答テキストを全選択してコピーする（Ctrl+A → Ctrl+C、Macの場合は⌘+A → ⌘+C）
2. メモアプリやドキュメントツール（Googleドキュメント、Notion、メモ帳など）に貼り付ける
3. ファイル名を「構成案_テーマ名_日付」のようにつけて保存する

**完成した成果物の目安**: スライド番号・タイトル・要点が揃った構成案が、1つのドキュメントとして保存されていればOKです。

## つまずきポイントと対策

| つまずき | 原因 | 対策 |
|---|---|---|
| AIの回答が一般的すぎて使えない | テーマや聞き手の情報が足りない | プロンプトに「聞き手は入社1ヶ月目の新入社員」のように具体的な対象を書き足す |
| スライド枚数が多すぎる/少なすぎる | 枚数の希望を伝えていない | 「10〜12枚で」のように枚数を明示する。修正プロンプトで調整もできる |
| 具体例が出てこない | AIに「具体例を入れて」と伝えていない | 修正プロンプトで「各スライドに具体例を1つずつ入れてください」と追加する |
| どこまで修正すればいいかわからない | 完璧を求めすぎている | 構成案は「たたき台」です。7〜8割の出来で十分。スライド作成時に調整できます |
| AIの回答が途中で切れた | 回答の文字数制限にかかった | 「続きをお願いします」と送るか、「スライド6以降を出力してください」と指定する |

## このレッスンでできるようになったこと

あなたはAIチャットツールにプロンプトを送ることで、スライド構成案を短時間で作れるようになりました。次のステップとして、この構成案をもとに実際のスライドを作成してみましょう。構成案という「設計図」があるので、スライド作成もスムーズに進むはずです。
