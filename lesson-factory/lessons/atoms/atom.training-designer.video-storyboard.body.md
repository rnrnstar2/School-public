# 動画教材の絵コンテをAIで作る

料理を始める前に「何を作るか」「どんな順番で進めるか」をメモしておくと、焦らずスムーズに調理できますよね。動画教材も同じです。撮影や編集に取りかかる前に、各シーンで「何を見せるか・何を伝えるか」をまとめた設計図があると、あとでの手戻りがぐっと減ります。

この設計図のことを「絵コンテ（ストーリーボード＝映像の設計図）」と呼びます。映画やCMの制作現場で昔から使われている手法ですが、AIチャットツール（＝AIと対話できるアプリ。ChatGPTやClaudeなど）を使えば、ゼロから自分で考える負担を大きく減らせます。

このレッスンでは、AIの力を借りて15分以内に「動画教材の絵コンテ」を1本完成させます。

## 前提を確認する

始める前に、次の準備が整っているか確認してください。

- AIチャットツール（ChatGPTやClaudeなど）をブラウザで開ける状態にあること
- 作りたい動画教材の「テーマ」が1つ決まっていること（例：「新人向けのメールマナー」「Excelの基本操作」など）
- 動画の対象者が誰か、ざっくりイメージできていること（例：「入社1年目の新人」「店舗スタッフ」）

プログラミングの知識は一切不要です。

## 絵コンテのイメージをつかむ

![絵コンテの構造図](/lesson-assets/atom.training-designer.video-storyboard/diagram.png)

絵コンテは、基本的に次の4つの要素をシーンごとにまとめた表です。

| 列 | 内容 |
|---|---|
| シーン番号 | 何番目の画面か |
| 映像・画面の内容 | 視聴者に何を見せるか |
| ナレーション（説明文） | 何を話す・書くか |
| 所要時間 | そのシーンは何秒か |

たとえば「新人向けメールマナー」の絵コンテなら、こんな感じになります。

**良い例**:
- シーン1：画面に「宛先の確認」の文字 → ナレーション「まずは宛先を確認しましょう」（15秒）
- シーン2：メールソフトの宛先欄の拡大図 → ナレーション「ToとCCの違い、おさえていますか？」（20秒）

**悪い例**:
- シーン1：とりあえずパソコン全体を映す → ナレーションなし（何を伝えたいか不明）
- シーン2：長文のテキストを一気に表示 → 読みきれない

## AIに絵コンテの雛形を作らせる

AIチャットツールに、次のような指示（プロンプト＝AIへの命令文）をコピーして貼り付けてください。「`___`」の部分だけ、自分の内容に書き換えます。

```
あなたは動画教材の設計プロです。次の条件で絵コンテを表形式で作成してください。

【テーマ】___（例：新人向けのメールマナー）
【対象者】___（例：入社1年目の事務職）
【動画の長さ】3〜5分
【シーン数】5〜8シーン

表の列：シーン番号 / 映像・画面の内容 / ナレーション（説明文） / 所要時間（秒）

シーンの冒頭にはタイトル画面、末尾にはまとめ画面を入れてください。
1シーンは30秒以内にしてください。
```

この指示をコピーして、空欄を自分のテーマに置き換えて送信するだけで、AIが絵コンテの雛形を出力してくれます。

![AIにプロンプトを送信した画面](/lesson-assets/atom.training-designer.video-storyboard/screen_capture.png)

### 指示の書き方のコツ

**良い例**: 「テーマ：新人向けのメールマナー。対象は入社1年目の社員。5シーンで、1シーン30秒以内。」
→ 具体的に書くほど、AIが的確な内容を返してくれます。

**悪い例**: 「メールについて教えて」
→ テーマが広すぎて、動画教材の絵コンテになりません。

### AIの出力が期待と違ったときの追加プロンプト

AIの回答がイマイチだった場合は、次のような追加指示を送ってみてください。やり直す必要はありません。同じチャットの続きで大丈夫です。

- **表形式にならなかった場合**: 「上の内容を、マークダウンの表形式に整えてください」
- **内容が一般的すぎる場合**: 「対象者は○○です。もっと具体的な場面で書き直してください」
- **シーン数が多すぎる場合**: 「重要度の低いシーンを削って、5シーンにまとめてください」
- **ナレーションが硬い場合**: 「もう少しカジュアルなトーンで、話しかけるように書き直してください」

## 内容を整える

AIが作った雛形を、そのまま使える部分と直したい部分に分けて確認します。この作業は、料理でいう「味見して調味料を足す」ステップです。AIが出したレシピを、自分好みに仕上げます。

次の3つの観点で見直してください。

1. **シーンの順番を確認する** — 話の流れが自然か見直す。「このシーンの前に、説明が必要では？」と感じたら順番を入れ替える
2. **言葉を自分の言葉に直す** — AIの表現がかたい場合は、普段使う言葉に置き換える。社内で使っている呼び方があればそちらに統一する
3. **時間を調整する** — 1シーンが長すぎる（30秒以上）なら、2つに分ける。合計時間が5分を超えないように調整する

修正したい箇所が見つかったら、AIに「シーン3のナレーションをもっと短くして」のように、ピンポイントで指示するのがコツです。全体をやり直すより効率的です。

## 完成した絵コンテを保存する

仕上がった絵コンテを保存します。次のどちらかの方法を使ってください。

- **方法A**: AIの回答をそのままコピーして、Googleドキュメントやメモアプリに貼り付ける
- **方法B**: AIに「この絵コンテをコピペしやすいように、プレーンテキストでもう一度出力して」と頼む

これが今回の成果物です。ファイル名に日付とテーマを入れておくと、あとで見つけやすくなります（例：「20260413_メールマナー絵コンテ.txt」）。

## 検証する

完成した絵コンテが「使えるもの」になっているか、次のチェックリストで確認してください。すべてにチェックが入れば合格です。

- [ ] 全シーンでテーマから外れていないか
- [ ] 対象者が知らない専門用語を使っていないか
- [ ] 各シーンのナレーションが30秒以内で読み上げられる長さか
- [ ] シーン同士のつながりが自然か（前のシーンの話題が次に続いているか）
- [ ] 合計時間が5分以内に収まっているか
- [ ] 冒頭にタイトル・導入のシーンがあるか
- [ ] 末尾にまとめ・振り返りのシーンがあるか

## つまずきやすいポイント

| つまずき | 原因 | 対策 |
|---|---|---|
| AIが表形式で返してくれない | 指示に「表形式で」が含まれていない | プロンプトに「表形式で作成してください」を追加する |
| 内容が一般的すぎて使えない | 対象者の情報が足りない | 「対象者は○○」と具体的に追記して再送信する |
| 1シーンが長すぎる | 時間の制約を伝えていない | 「1シーン30秒以内で」と指示に加える |
| シーン数が多すぎる | 上限を指定していない | 「5〜8シーン以内で」と指示に加える |
| AIが毎回違う形式で返す | ツールの仕様上正常な挙動 | 気に入った形式が出たら「この形式を維持して」と伝える |
