# 回答方針と引用表示を設計する

AI（人工知能）に社内の質問を答えてもらうとき、「なんだか機械的でピンとこない」「この情報、本当に正しいの？」と感じたことはありませんか？

ここは料理に似ています。美味しい料理を作るには、レシピに「味付けは薄めで」「材料の産地をメニューに書いて」と細かく指定しますよね。AIも同じです。あなたが「どう答えてほしいか」という方針（＝味付け）と、「どこから情報を持ってきたか」という引用（＝産地表示）をデザインしてあげると、グッと使いやすい回答になります。

![回答方針と引用の設計フロー](/lesson-assets/atom.nocode-builder.design-answer-policy-and-citations/diagram.png)

このレッスンでは、AIチャットツールを使いながら「回答方針ドキュメント」を1枚仕上げます。所要時間は約15分です。

## 前提を確認する

必要なものは次の2つだけです。

- パソコンとインターネット接続
- AIチャットツール（ChatGPT、Claude など。無料プランでOK）

プログラミングの経験は一切不要です。AIに質問しながら進めるだけで完了します。

## 回答トーンのルールを決める

まず「AIにどんな話し方で答えてほしいか」を決めましょう。これが**回答トーン**です。

考えるポイントは3つです。

1. **敬語かカジュアルか** — 社外向けなら「です・ます調」、チーム内なら「だ・である調」もあり
2. **専門用語を使うかどうか** — 読み手が専門家でなければ、やさしい言葉に言い換えるルールを入れる
3. **回答の長さ** — 「3行以内で要点をまとめる」など具体的な目安を決めておくと回答がブレにくくなる

### 良い例
- 「です・ます調で答えてください」
- 「専門用語は使わず、中学生でもわかる言葉で説明してください」
- 「回答は3〜5行にまとめ、詳細が必要なら箇条書きを付けてください」

### 悪い例
- 「適当に答えて」（何が適当かAIには判断できない）
- 「いい感じにまとめて」（基準がないので毎回バラバラになる）

> **AIに聞いてみよう**: ChatGPT や Claude に次のように聞くと、トーンのたたき台を作ってくれます。
>
> ```
> 社内FAQ用のAIチャットボットを作っています。
> 回答トーンのルールを5項目程度で提案してください。
> 対象読者は営業職の社員で、ITに詳しくない人が多いです。
> ```

## 引用表示のルールを決める

次に「AIが回答の根拠にした情報をどう見せるか」を決めます。これが**引用表示**です。

引用を付ける理由はシンプルです。読んだ人が「本当にそうなの？」と思ったときに、元の資料をたどれるようにするためです。スーパーの野菜コーナーに産地ラベルが貼ってあるのと同じ安心感を、AIの回答にも持たせましょう。

### 決めること

| 項目 | 具体例 |
|------|--------|
| 引用の位置 | 回答の最後にまとめる ／ 文中に入れる |
| 表示する情報 | 資料名＋ページ番号 ／ 資料名＋リンク |
| 直接引用の表し方 | カギカッコ「」で囲む ／ 引用ブロックで表示 |
| 資料にないときの対応 | 「この質問に該当する資料が見つかりませんでした」と明示する |

### 良い例
- 「回答の最後に、参照した社内マニュアルの名前とリンクを箇条書きで表示してください」
- 「マニュアルからそのまま引用した部分はカギカッコ（「 」）で囲んでください」
- 「資料に該当する情報がないときは『この質問に該当する情報は見つかりませんでした。担当者にお問い合わせください。』と表示してください」

### 悪い例
- 「リンクはいらない」（出どころが確認できず、誤った回答に気づけなくなる）
- 「適当に要約だけして」（どの資料を根拠にしたかわからなくなる）

> **AIに聞いてみよう**: 引用ルールもAIにたたき台を作ってもらえます。
>
> ```
> 社内FAQボットの引用表示ルールを考えています。
> 以下の条件で引用ルールを3〜5項目提案してください。
> - 回答の末尾に引用元を表示する
> - 資料にない場合の対応も含める
> - 読み手はITに詳しくない営業職の社員
> ```

## ルールをドキュメントにまとめる

ここまでで決めた内容を、1枚のドキュメントにまとめましょう。ノートアプリ（Notion、Googleドキュメントなど）を開いて、以下の構成で書きます。

```
【回答方針ドキュメント】

■ 回答トーンのルール
  1. （ここにルールを書く）
  2. ...

■ 引用表示のルール
  1. （ここにルールを書く）
  2. ...

■ 資料にない質問への対応
  - （対応方針を書く）
```

> **AIに聞いてみよう**: ドキュメント全体を一気に整えたいときは、ここまでに決めた内容をAIに渡して清書してもらいましょう。
>
> ```
> 以下のルールを「回答方針ドキュメント」として整理してください。
> 見出し付きで読みやすくまとめてください。
>
> 【回答トーン】
> - です・ます調で答える
> - 専門用語は使わない
> - 回答は3〜5行にまとめる
>
> 【引用表示】
> - 回答末尾に資料名とリンクを箇条書きで表示
> - 直接引用はカギカッコで囲む
> - 該当資料がないときは明示する
> ```

## テストして動作を確認する

作った方針がうまく機能するか、実際にAIで試してみましょう。

### テストの手順

1. ChatGPT や Claude を開く
2. 作成した回答方針ドキュメントの内容を、最初のメッセージとして貼り付ける
3. 続けて「この方針に従って、次の質問に答えてください：〇〇」と質問を送る
4. AIの回答を見て、トーンと引用表示がルール通りになっているか確認する

![テスト実行画面](/lesson-assets/atom.nocode-builder.design-answer-policy-and-citations/screen_capture.png)

### チェックポイント

- [ ] 回答のトーンがルール通りか（敬語、用語の平易さ、長さ）
- [ ] 引用元が指定した形式で表示されているか
- [ ] 資料にない質問をしたとき、想定どおりの対応が返ってくるか

結果がルールと違っていたら、方針ドキュメントの書き方を調整して、もう一度テストしてください。2〜3回やり取りすれば、十分に整ったルールに仕上がります。

## つまずきポイントを押さえる

| よくある失敗 | 原因 | 対処法 |
|-------------|------|--------|
| 毎回トーンがバラバラになる | ルールがあいまい | 「です・ます調」「3行以内」のように**数字や具体例**を入れる |
| 引用が表示されない | 引用指示を書いていない | 「回答の末尾に出典を箇条書きで表示して」と明記する |
| AIが勝手に情報を作る | 資料外の対応を決めていない | 「資料にない場合は『わかりません』と答えて」と追加する |
| ルールが長すぎてAIが守らない | 項目が多すぎる | 最初は5〜7項目に絞り、慣れてから追加する |

## 完了を確認する

以下の3つがそろえば、このレッスンは完了です。

1. **回答方針ドキュメント**が完成している（回答トーン＋引用表示＋資料外対応の3セクション）
2. **AIでテスト済み** — 実際にルールを渡して質問し、意図どおりの回答が返ってきた
3. **テスト画面のスクリーンショット**を1枚以上撮った（AIの回答にトーンと引用が反映されている画面）

スクリーンショットの撮り方：Windowsは `Win + Shift + S`、Macは `Command + Shift + 4` で画面の一部を写真として保存できます。
