# クライアント対応の定型文をAIで準備する

仕事の依頼が来たとき、すぐに丁寧な返事を返せますか？
「返信を書くのに毎回30分もかかってしまう」「失礼にならないか不安で何度も書き直す」——こんな悩みは珍しくありません。

このレッスンでは、AIチャットツール（＝ChatGPTやClaudeなど、文章を自動で作ってくれるサービス）を使って、クライアントに送る定型文を**15分で**準備する方法を学びます。

料理でたとえると、「万能の下味ダレ」をあらかじめ作っておくイメージです。メイン食材（＝相手の名前や案件内容）を変えるだけで、毎回おいしい一品がサッと出せるようになります。

## 完成イメージを確認する

このレッスンを終えると、次の3点がそろったテキストファイルが手元にできあがります。

1. **AIへの指示文（プロンプト）** の見本
2. **AIが出力した文章**のコピー
3. **自分の言葉に直した最終版**の定型文

## 前提を確認する

このレッスンを進める前に、次の2つを用意してください。

- **AIチャットツールのアカウント**: ChatGPT（chat.openai.com）でも Claude（claude.ai）でもOK。無料プランで十分です
- **過去にクライアントへ送った返信（あれば）**: AIの出力精度が上がります。なくてもゼロから作れるので安心してください

## 定型文が活きる5つの場面を知る

フリーランスの仕事では、同じようなやり取りが繰り返し発生します。代表的な場面は次の5つです。

| # | 場面 | 一言メモ |
|---|------|----------|
| ① | 依頼を受けたときの「お受けします」返信 | 第一印象を決める |
| ② | 見積もりを送るときの案内 | 金額と納期を明確に |
| ③ | 納品時の連絡 | 確認してほしい点を添える |
| ④ | 修正依頼への対応 | スピード感が信頼に直結 |
| ⑤ | お礼と次回の提案 | リピートにつなげる |

このレッスンでは **①「お受けします」返信** を一緒に作ります。同じやり方で②〜⑤も応用できます。

## プロンプト（＝AIへの指示文）を組み立てる

AIに「いい文章を書いて」とだけ伝えても、自分の仕事に合った文章にはなりません。ポイントは **4つの要素** を具体的に伝えることです。

| 要素 | 何を書くか | 例 |
|------|-----------|----|
| 役割 | あなたの職種 | フリーランスのWebデザイナー |
| 状況 | どんなメールが来たか | LP制作の依頼メール |
| 口調 | どんなトーンで | 丁寧で親しみやすい |
| 制約 | 文字数や形式 | 300字以内 |

### 良い例

```
あなたはフリーランスのWebデザイナーです。
クライアントから「ランディングページの制作を依頼したい」というメールを受け取りました。
以下の条件で返信文を書いてください。

- 口調: 丁寧で親しみやすい
- 文字数: 300字以内
- 含める内容: お礼、引き受ける旨、次のステップ（ヒアリング日程の提案）
```

### 悪い例

```
クライアントへの返信を書いて。
```

悪い例が失敗する理由は、AIが「誰の」「どんな状況の」返信かを想像で補うしかないからです。結果、自分の仕事とズレた文章が返ってきます。

## AIに定型文を作ってもらう

実際にやってみましょう。手順は4ステップだけです。

1. AIチャットツール（ChatGPTまたはClaude）をブラウザで開く
2. 入力欄に、上の「良い例」のプロンプトを**そのまま**貼り付ける
3. 送信ボタンを押して、数秒待つ
4. 返ってきた文章を**全文コピー**する（あとで編集するため）

![AIチャット画面でプロンプトを入力して結果を得る様子](/lesson-assets/atom.ai-freelancer.client-communication-ai/screen_capture.png)

> **ヒント**: もし結果がイマイチだったら、「もう少しカジュアルに」「納期の目安も入れて」のように **追加の一言を送る** だけで改善されます。最初から完璧を目指さなくて大丈夫です。

## 結果を自分の言葉に整える

AIが作った文章は「たたき台」です。必ず自分の言葉に書き換えてから使いましょう。

### チェックリスト

- [ ] 相手の名前（「○○様」）を入れたか
- [ ] 自分のサービス内容と合っているか
- [ ] 納期や次のアクション（ヒアリング日程など）を一言添えたか
- [ ] 自分の普段の言葉づかいに近づけたか

### 悪い例

- AIの出力をそのままコピペして送る → 機械的な印象を与えてしまう
- 専門用語（レスポンシブ、UXなど）を説明なしに残す → 相手が非エンジニアだと伝わらない

## 定型文を保存する

整えた定型文は、すぐに取り出せる場所に保存しましょう。おすすめは **1つのファイルに場面ごとの見出しをつけてまとめる** 方法です。

例（Googleドキュメントやメモ帳で作れます）:

```
# クライアント対応テンプレート集

## ① 依頼を受けたとき
（ここに最終版を貼る）

## ② 見積もりを送るとき
（ここに最終版を貼る）

…
```

ファイル名に「テンプレート」と入れておくと、あとから検索しやすくなります。

## 成果物を確認する

次の3つがファイルに含まれていれば、このレッスンは完了です。

1. プロンプト（AIへの指示文）の見本 → 入っている？
2. AIが出力した文章のコピー → 入っている？
3. 自分が修正した最終版の定型文 → 入っている？

証拠として、**AIチャット画面のスクリーンショットを1枚**撮ってください。プロンプトとAIの回答が両方写っていればOKです。

## つまずきやすいポイントと対処法

| つまずき | 対処法 |
|---------|--------|
| AIが的外れな回答をした | プロンプトに「役割」と「状況」が入っているか確認。足りなければ追記して再送信 |
| 文章が長すぎる | プロンプトの末尾に「300字以内で」と追加して再実行 |
| 敬語のレベルが合わない | 「ですます調で」「もう少しカジュアルに」など口調の指示を具体的に追記 |
| 何を書けばいいかわからない | 過去に送ったメールをAIに貼り付けて「これと同じ雰囲気で書いて」と指示する |
| AIの回答が英語で返ってきた | プロンプトの最初に「日本語で回答してください」と一行追加する |

## 次のステップ

「お受けします」の定型文ができたら、同じ手順で②〜⑤のパターンも作ってみましょう。5つのパターンが揃えば、クライアント対応の大部分をスムーズにこなせるようになります。

さらにステップアップしたい方は、「通る提案文をAIで作成する」レッスンに進んでみてください。
