# AIでSEO記事を執筆する

SEO（＝検索エンジン最適化。Googleなどの検索結果で上位に表示されるように工夫すること）記事を、AIツールに手伝ってもらいながら15分で仕上げる方法を学びます。

料理に例えると、あなたは**シェフ**です。AIは優秀な**キッチン助手**。レシピ（＝記事の構成）を考え、材料（＝キーワード）を選び、味見（＝検証）をするのはあなたの役目です。助手にまかせっきりにせず、最後はあなたが仕上げるのがコツです。

## 準備する

### 必要なものをそろえる

- AIチャットツール（＝ChatGPTやClaudeなど、文章のやり取りでAIを使えるサービス）のアカウント
  - 無料プランでOKです。ChatGPTなら [chat.openai.com](https://chat.openai.com)、Claudeなら [claude.ai](https://claude.ai) でアカウントを作れます
- 書きたいテーマのアイデア（まだキーワードが決まっていなくても大丈夫です）
- テキストエディタ（メモ帳、Googleドキュメントなど何でもOK）

### キーワードを1つ決める

キーワード（＝読者が検索窓に入力する言葉）は、記事の「主役」です。旅行に例えると、キーワードは**行き先**にあたります。行き先が決まらないと荷造りも計画もできないように、キーワードが決まらないと記事の方向性が定まりません。

**良い例**
- 「初心者 おすすめ ノートPC 2025」（具体的で、検索する人の悩みが見える）
- 「一人暮らし 安い 自炊 レシピ」（ターゲットと目的がはっきりしている）

**悪い例**
- 「パソコン」（広すぎて大手サイトに勝てない）
- 「今日の夕食」（検索する人が少なく、記事にしにくい）

> **コツ**: 「自分が困ったとき、Googleにどう打ち込むか」を声に出して言ってみてください。自然なキーワードが見つかります。

キーワード選びに迷ったら、AIにも相談できます。次のプロンプト（＝AIへの指示文）をコピーして貼り付けてみてください。

```
私は「副業」「在宅ワーク」に興味がある初心者です。
このジャンルで、検索ボリュームがありそうで、かつ個人ブログでも上位を狙えそうなキーワードを5つ提案してください。
それぞれ「なぜ狙い目なのか」も一言添えてください。
```

## AIに記事の構成を作ってもらう

キーワードが決まったら、次はAIに記事の骨組み（＝見出し構成）を作ってもらいます。

AIチャットツールを開き、次のプロンプトをコピーして貼り付けてください。「〇〇」の部分にあなたのキーワードを入れます。

```
あなたはSEOライティングの専門家です。
「〇〇」というキーワードで検索する読者に向けたブログ記事を書きたいです。

以下の条件で見出し構成を提案してください:
- H2（大見出し）とH3（小見出し）で5〜7個
- 読者が「基礎→実践→まとめ」の順で理解できる流れにする
- 各見出しに「この見出しで読者が得られること」を1行で添える
```

![AIに構成を作ってもらう画面](/lesson-assets/atom.ai-freelancer.seo-article-with-ai/screen_capture.png)

### 構成を自分の目で整える

AIの提案をそのまま使うのではなく、**シェフの目で味見**します。

**チェックポイント**:
- 見出しの順番は「基本 → 具体的な方法 → よくある質問」と読者にとって自然な流れか？
- 似た内容の見出しが重複していないか？
- 読者が本当に知りたいことが抜けていないか？

**良い例**: 見出しを読むだけで記事の全体像がつかめる構成

**悪い例**: 似た内容の見出しが3つ重複している／順番がバラバラで読者が混乱する

不要な見出しは削り、足りない視点があれば追加してください。ここがシェフの腕の見せどころです。

## AIに本文を書いてもらう

構成が決まったら、見出しを**1つずつ**AIに渡して本文を書いてもらいます。一度に全部書かせるより、小分けにした方が品質が上がります。

次のプロンプトをコピーして、見出しごとに繰り返し使ってください。

```
次の見出しについて本文を書いてください。

【見出し】: （ここに見出しを貼る）
【キーワード】: （あなたのキーワード）
【条件】:
- 初心者にもわかる丁寧な日本語で400〜500文字
- 専門用語には必ず括弧で説明を添える
- 具体的な数字や例を1つ以上入れる
- 「〜です。〜ます。」の繰り返しにならないよう文末を変化させる
```

### AIの出力を確認する3つのポイント

| チェック項目 | なぜ大事か | 対処法 |
|---|---|---|
| 事実は正しいか | AIは「もっともらしい嘘」をつくことがあります（ハルシネーションと呼びます） | 数字や固有名詞はGoogle検索で裏を取る |
| 難しい言葉はないか | 読者は専門家ではありません | AIが使った難しい言葉をあなたの言葉に書き換える |
| あなたらしさはあるか | AI文章だけだと他の記事と同じになります | 体験談や自分の感想を1〜2文足す |

## 記事を仕上げる

### 全体を通して読む

すべての見出しの本文が揃ったら、最初から最後まで**声に出して**読みます。声に出すと、つっかかる部分＝読者もつまずく部分がわかります。

**確認ポイント**:
- 文章の途中で話が飛んでいないか
- 同じ内容を繰り返していないか
- 読者が「わからない」と感じる専門用語がないか

### タイトルとメタディスクリプションを書く

記事のタイトルは、お店の看板です。中身がどんなに良くても、看板が地味だとお客さんは入ってきません。

メタディスクリプション（＝検索結果に表示される記事の要約文、約100〜120文字）も一緒に書きましょう。AIにお願いできます。

```
以下の記事内容をもとに、SEOに強いタイトルとメタディスクリプションを提案してください。

【キーワード】: （あなたのキーワード）
【記事の内容】: （見出し構成をここに貼る）
【条件】:
- タイトルは32文字以内
- メタディスクリプションは100〜120文字
- 読者が「読みたい」と思えるように具体的な数字やメリットを入れる
- タイトル案を3つ、メタディスクリプション案を2つ出す
```

**良い例**: 「【2025年版】予算3万円で失敗しないノートPCの選び方5つ」

**悪い例**: 「ノートPCについて解説します」（情報が少なすぎてクリックされない）

## 検証する

記事が完成したら、**公開前の最終チェック**として次の4点を確認します。

1. **キーワードは自然に使われているか**: タイトル・最初の段落・見出しにキーワードが含まれているか確認します。ただし、無理に詰め込むと読みにくくなるので、自然な文脈で使うことが大切です
2. **読者に新しい発見はあるか**: AIだけの一般的な内容だと他の記事と差がつきません。あなたの体験や意見を最低1箇所は入れましょう
3. **文字数は2,000〜4,000文字程度か**: 短すぎると検索エンジンが「内容が薄い」と判断することがあります。AIチャットに「この文章の文字数を教えて」と聞けばすぐ数えてくれます
4. **誤字脱字はないか**: AIに最終チェックをお願いしましょう。次のプロンプトが使えます:

```
以下の文章の誤字脱字、文法の間違い、不自然な表現をチェックしてください。
修正が必要な箇所は「修正前 → 修正後」の形で一覧にしてください。

（ここに記事全文を貼る）
```

## つまずきポイント

| つまずき | 原因 | 対策 |
|---|---|---|
| AIの文章が「他人の文章っぽい」 | AIは平均的で無難な文章を書きがち | まずAIの出力を土台にして、あなたの言葉で2〜3文書き直す。「私が試したところ…」のような一人称を足すだけで印象が変わります |
| キーワード選びで迷って進まない | 「正解」を探しすぎている | まず3つテーマを書き出し、一番書きたいものを選ぶ。最初から完璧を目指さなくてOKです |
| 15分で終わらない | 記事を完璧にしようとしている | ゴールは「完成」ではなく「下書き完成」。タイマーを15分にセットして、時間が来たら一度手を止めましょう |
| AIが同じような文章ばかり返す | プロンプトが曖昧すぎる | 「400文字で」「具体例を入れて」「小学生にもわかるように」など、条件を具体的に指定すると出力が変わります |

## 成果物を確認する

このレッスンのゴールは、**SEOを意識した記事の下書きを1本完成させること**です。

以下ができていれば合格です:
- [ ] キーワードを1つ決めた
- [ ] AIを使って見出し構成を作った
- [ ] 各見出しの本文をAIで作成し、自分の言葉で手直しした
- [ ] タイトルとメタディスクリプションを書いた
- [ ] 完成した記事全体をテキストエディタに保存した

完成した記事のスクリーンショットを撮影して、成果物として保存してください。
