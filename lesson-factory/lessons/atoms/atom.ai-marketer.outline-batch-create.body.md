# 記事構成案を量産する

ブログやSNSのネタはあるのに、「どういう順番で書けばいいか」が決まらず、書き出すまでに時間がかかってしまったことはありませんか？

料理に例えると、記事構成案は「レシピの手順書」のようなものです。食材（ネタ）はそろっているのに、調理の手順があいまいだと途中で止まってしまいます。逆に手順がはっきりしていれば、あとは材料を順番に並べるだけ。このレッスンでは、AIチャットツール（＝ChatGPTやClaudeなどの対話型AI。人間の言葉で質問すると答えてくれるツール）を使って、記事の構成案（アウトライン）を一度に複数作る方法を学びます。15分で終わるので、ぜひ一緒にやってみましょう。

![構成案量産の全体フロー](/lesson-assets/atom.ai-marketer.outline-batch-create/diagram.png)

## 前提を確認する

このレッスンを始める前に、次の2つだけ用意してください。

- **AIチャットツールのアカウント**: ChatGPT（無料版でOK）、Claude、Gemini など、どれでも大丈夫です。まだアカウントがない方は、ChatGPT（https://chat.openai.com）にアクセスして無料登録しましょう
- **書きたいテーマのメモ**: 3〜5個のキーワードだけでもOKです。例: 「在宅勤務」「時間管理」「ツール紹介」

プログラミングの知識は一切不要です。ブラウザでAIチャットを開くだけで進められます。

## プロンプト（指示文）を組み立てる

プロンプト（＝AIに渡す指示文のこと）は、料理でいう「注文書」です。注文が具体的なほど、出てくる料理があなたの希望に近くなります。

以下のテンプレートをそのままコピーして、`[  ]` の部分だけを自分の内容に書き換えてください。

```
次のテーマについて、ブログ記事の構成案を3パターン作成してください。

テーマ: [ ここにテーマを書く ]
読者: [ ここに対象読者を書く。例: 30代の会社員 ]
目的: [ ここに記事の目的を書く。例: 新しいツールの便利さを伝える ]
文字数の目安: [ ここに完成記事の文字数を書く。例: 2000文字 ]

各構成案は以下の形式で出力してください。
- タイトル案（読者が思わずクリックしたくなる具体的なもの）
- はじめに（読者の悩みに共感する1〜2文）
- 本文の見出し（3〜5個。各見出しの下に1行で内容の要約をつけてください）
- まとめ（読者の次のアクションを促す1文）
```

### 良いプロンプトと悪いプロンプトを比べる

**良い例**:
「テーマ: 在宅勤務の効率アップ。読者: リモートワーク歴1年未満の会社員。目的: 時間管理ツールの導入を促す。文字数の目安: 2000文字」
→ テーマ・読者・目的・文字数がそろっているので、AIが具体的な構成を提案しやすい

**悪い例**:
「ブログの構成案を作って」
→ 何について・誰に向けて・何を伝えたいのかが不明なので、AIが一般的すぎる内容を返してしまう

> **ワンポイント**: もし出力に満足できなかったら、「読者がもっと初心者の場合に書き直してください」のように追加の指示（フォローアップ）を送りましょう。AIは会話の流れを覚えているので、最初からやり直す必要はありません。

## AIに構成案を生成させる

用意したプロンプトをAIチャットの入力欄に貼り付けて、送信ボタンを押しましょう。

![ChatGPTで構成案を生成した画面](/lesson-assets/atom.ai-marketer.outline-batch-create/screen_capture.png)

30秒ほどで3パターンの構成案が返ってきます。返ってきたら、次の3つの観点で目を通してください。

1. **タイトル案**: 自分が「これを書きたい」と思えるものがあるか
2. **見出しの流れ**: 問題提起 → 解決策 → まとめ、のように自然な順番になっているか
3. **読者目線**: 読者の立場で「知りたいことが見出しからわかるか」

### 良い構成案と悪い構成案を見分ける

**良い例**:
タイトル「在宅勤務が1.5倍はかどる時間管理術」、見出しが「なぜ在宅だと時間が溶けるのか」「3つのスケジュール管理術」「今日からできる1分セットアップ」
→ 具体的な数字や行動が含まれていて、読者が「自分ごと」として読み進められる

**悪い例**:
タイトル「在宅勤務について」、見出しが「はじめに」「在宅勤務とは」「まとめ」
→ 抽象的すぎて、読者が「自分に役立ちそう」と感じにくい

### 構成案を改善するフォローアップ例

AIの出力がいまいちだった場合は、以下のような追加指示を送ると改善できます。

- 「タイトルにもっと具体的な数字を入れてください」
- 「見出しを読者のアクション（〜する、〜を試す）で始めてください」
- 「3つ目のパターンは、まったく違う切り口（例: 失敗談から始まる構成）で作り直してください」

## 出力をまとめて保存する

AIから返ってきた3パターンの構成案を、自分のメモ帳やGoogleドキュメントなどにコピーして保存します。

保存するときのコツ:

- 各パターンの先頭に **パターンA / B / C** とラベルをつけておく
- 気に入った部分にはメモ帳の太字やハイライトで印をつけておく
- どの構成案を使うかはまだ決めなくてOKです。まずは「ストック（＝作り置きのようなもの）」として持っておきましょう

## 完成を確認する

ここまでの作業がうまくいったか、次のチェックリストで確認してみましょう。すべてにチェックがつけば、このレッスンは完了です。

- [ ] 構成案が3パターン以上ある
- [ ] それぞれにタイトル・見出し・まとめがそろっている
- [ ] 対象読者が具体的にイメージできる内容になっている
- [ ] どれか1つを選んで、そのまま記事を書き始められそうだと感じる
- [ ] 構成案をメモ帳やドキュメントに保存した

> **確認のポイント**: 5つ目の「書き始められそう」が一番大事です。もし「まだ足りない」と感じたら、AIに「見出しごとに書く内容のヒントを3行ずつ追加してください」とフォローアップしてみましょう。

## つまずきポイントを知っておく

| よくあるつまずき | 原因 | 解決策 |
|---|---|---|
| AIが短すぎる構成案を返す | プロンプトの情報が足りない | 「読者」と「目的」を1文ずつ具体的に書き足す。例: 「読者はExcelしか使ったことがない事務職」 |
| 3パターンが似たような内容になる | 切り口の指定がない | 「1つ目は初心者向け、2つ目は経験者向け、3つ目は失敗談から始まる構成にしてください」と追加指示する |
| どの構成案を選べばいいかわからない | 判断基準がない | まずは一番「書きやすそう」と感じた1つを選ぶ。残りは次回のストックに |
| AIが英語で返してきた | 言語指定が漏れた | プロンプトの最後に「すべて日本語で回答してください」を追加する |
