# クライアント対応をAIで効率化する

## このレッスンでできるようになること

クライアント（＝仕事を依頼してくれるお客様）からの問い合わせや要望に対して、AIチャットツールを使ってスピーディーに返信文の下書きを作れるようになります。最終的に「AIに下書きを作ってもらい、自分で確認・修正して送る」という一連の流れを、実際のメッセージを使って体験します。

所要時間の目安: **約15分**

---

## イメージで理解する

レストランでウェイターがお客さんの注文を聞いて厨房に伝える場面を想像してみてください。

1. ウェイターはお客さんの注文を正確に聞き取る
2. 厨房（＝AI）に「こういう料理を作って」と具体的に伝える
3. 厨房が料理を出したら、ウェイターが味見をしてお客さんの好みに合わせて盛り付け直す
4. 最後にウェイター自身がお客さんのテーブルに届ける

AIを使ったクライアント対応もまったく同じです。あなたが「ウェイター」となって、クライアントの要望をAIに伝え、AIが作った下書きをあなたが最終確認してからクライアントにお届けします。**AIはあくまで厨房であり、最終判断はいつもあなたが行います。**

## 前提を確認する

以下の2つが揃っていれば、すぐに始められます。

- **AIチャットツールのアカウント**: 以下のどれか1つでOKです
  - [ChatGPT](https://chat.openai.com/)（無料プランで十分）
  - [Claude](https://claude.ai/)（無料プランで十分）
  - その他、文章生成ができるAIチャットツール
- **クライアントからの実際のメッセージ**: メールやチャットで届いた文章を1件、手元に用意してください。練習用なので、機密情報を含まない軽めのやり取りがおすすめです

> **注意**: クライアントの個人情報や機密情報をそのままAIに入力しないでください。名前を「Aさん」に置き換える、金額をぼかすなど、最低限の配慮をしましょう。

---

## ステップ1: クライアントの要望を整理する

まず、クライアントから届いたメッセージを読み返して、「何を求めているのか」を **3行以内のメモ** にまとめます。このメモが、AIへの指示の土台になります。

整理するポイントは3つです。

1. **何をしてほしいのか**（依頼内容）
2. **なぜそうしてほしいのか**（背景や理由）
3. **いつまでに必要か**（期限）

**良い例**:
> 「納期を3日早めてほしいとのこと。理由は社内レビューが前倒しになったため。今週金曜までに可否を回答してほしい」

**悪い例**:
> 「なんか急いでるっぽい」

良い例のように具体的にまとめると、次のステップでAIに出す指示の精度がぐんと上がります。

---

## ステップ2: AIに返信文の下書きを作ってもらう

AIチャットツール（ChatGPTやClaudeなど）を開き、次のようなプロンプト（＝AIへの指示文）を入力します。

### コピペで使えるプロンプトのテンプレート

```
あなたはビジネスメールの作成アシスタントです。
以下の状況に対する返信文を作成してください。

【状況】
{ステップ1で整理した3行のメモをここに貼り付ける}

【条件】
- トーン: 丁寧で誠実（ですます調）
- 文字数: 150〜250字程度
- 相手の要望に前向きな姿勢を見せつつ、確認が必要な点は正直に伝える
```

### 実際の入力例

```
あなたはビジネスメールの作成アシスタントです。
以下の状況に対する返信文を作成してください。

【状況】
クライアントから納期を3日早めてほしいと依頼されました。
理由は社内レビューが前倒しになったためです。
今週金曜までに可否を回答してほしいとのことです。

【条件】
- トーン: 丁寧で誠実（ですます調）
- 文字数: 150〜250字程度
- 相手の要望に前向きな姿勢を見せつつ、確認が必要な点は正直に伝える
```

![AIチャット画面での入力例](/lesson-assets/atom.ai-writer.client-communication-ai/screen_capture.png)

### プロンプトのコツ

| やること | 理由 |
|---------|------|
| **背景を書く** | AIが状況を正しく理解できる |
| **トーンを指定する** | 丁寧すぎ・くだけすぎを防げる |
| **文字数の目安を書く** | 長すぎ・短すぎを防げる |
| **役割を与える**（「あなたは〇〇です」） | AIの回答の質が安定する |

**悪い例の指示**: 「返事書いて」
→ これだけだとAIは何のことかわからず、的外れな文章が出てきます。

---

## ステップ3: AIの下書きを確認・修正する

AIが出力した文章をそのままコピペして送る……のは **絶対にNG** です。必ず自分の目で確認しましょう。

### 3つのチェックポイント

| チェック項目 | 確認すること | よくあるミス |
|-------------|-------------|------------|
| **1. 事実の正確さ** | 日付・数字・名前が合っているか | AIが勝手に日付を入れてしまう |
| **2. トーンの妥当性** | あなたとクライアントの関係性に合っているか | 初対面なのにくだけすぎている |
| **3. 過度な約束** | できないことを「できます」と書いていないか | AIが「問題ありません」と断言してしまう |

**良い修正の例**:
AIの下書きに「2営業日以内にお返事します」とあったが、実際には確認に時間がかかるため「3営業日以内にお返事します」に修正してから送信する。

**悪い例**:
AIの文章を一切読まずにそのまま送信する。→ 事実と違う約束をしてしまうリスクがあります。

### うまくいかなかったら？ AIに修正を頼むプロンプト

もしAIの出力がイマイチだったら、以下のように追加で指示を出しましょう。

```
もう少しくだけたトーンにしてください。
長年の付き合いのあるクライアント向けです。
```

```
「前向きに検討します」ではなく「社内で確認のうえ、
金曜日までにご連絡します」のように、具体的な行動を書いてください。
```

このように **何がどう違うのか** を具体的に伝えると、AIは的確に修正してくれます。「もっと良くして」のような漠然とした指示は避けましょう。

---

## ステップ4: 最終版を送信する

修正が終わったら、いつもの方法（メールやチャットツール）でクライアントに送信します。このステップはAIの出番はありません。**あなた自身の判断で送信ボタンを押します。**

送信前の最終チェック:
- [ ] 宛先は合っているか
- [ ] 添付ファイルの漏れはないか
- [ ] 誤字脱字はないか（AIの文章にも誤字はあり得ます）

---

## できたか確認する

以下の3つすべてに「はい」と言えれば、このレッスンは完了です。

1. **クライアントの要望を3行以内に整理できた** — メモを見返して、依頼内容・背景・期限が書かれていますか？
2. **AIに具体的な指示を出して返信文の下書きを作れた** — テンプレートを参考に、背景・トーン・文字数を指定しましたか？
3. **AIの下書きを3つの観点（事実・トーン・約束）で確認し、必要な修正を加えた** — 1箇所でも修正した箇所がありますか？

> **成果物**: 修正済みの返信文を、テキストファイルやメモアプリに保存しておきましょう。これがあなたの「AIと一緒に作った最初のビジネス文書」になります。

---

## つまずきやすいポイントと対処法

| つまずき | 対処法 |
|---------|--------|
| AIの返信が長すぎる／短すぎる | プロンプトに「150〜250字程度で」と文字数を明記する |
| トーンが合わない（丁寧すぎる・くだけすぎる） | 「ですます調で」「長年の取引先向け」など関係性を具体的に指定する |
| AIが事実と違うことを書く | AIは事実を知らないので、必ず自分でチェックする。日付や数字は特に注意 |
| 何を指示すればいいかわからない | まずステップ1の「3行メモ」を作る。それがそのまま指示の素材になる |
| 個人情報を入れてしまいそう | 名前は「Aさん」、金額は「約○万円」のようにぼかしてから入力する |

---

## 振り返る

このレッスンであなたは次のことを実践しました。

- クライアントの要望を「依頼内容・背景・期限」の3点で整理する
- AIにテンプレートを使って具体的な指示を出し、返信文の下書きを作ってもらう
- AIの下書きを「事実・トーン・約束」の3観点で確認・修正する
- 修正を依頼するときも具体的に伝える

この流れを一度経験すれば、次回からは **5分以内** に返信文の下書きが作れるようになります。ぜひ次のクライアント対応で試してみてください。
