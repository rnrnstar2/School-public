# 通る提案文をAIで作成する

フリーランスの仕事をもらうとき、最初の関門は「提案文（＝クライアントに『私にこの仕事を任せてください』と伝える文章）」です。

レストランの入り口メニューを想像してみてください。お客さんはメニューを見て「ここで食べよう」と決めますよね。提案文も同じです。クライアントはあなたの提案文を見て「この人に頼もう」と決めます。つまり、提案文はあなたの「お品書き」です。

このレッスンでは、AIチャットツール（＝人工知能と対話できるサービス。ChatGPTやClaudeなど）を使って、クライアントに通る提案文を**15分**で作ります。

![提案文作成の全体フロー](/lesson-assets/atom.ai-freelancer.proposal-writing-ai/diagram.png)

---

## 準備する

始める前に、次の2つを用意してください。

- **AIチャットツールのアカウント** → 無料版で大丈夫です。ChatGPT（chat.openai.com）またはClaude（claude.ai）にアクセスして無料登録を済ませてください
- **応募したい案件の募集文** → クラウドソーシングサイト（＝仕事を依頼したい人と受けたい人をつなぐサービス。ランサーズやクラウドワークスなど）で気になる案件を見つけ、募集文をコピーしておきます

> **ポイント**: まだ応募したい案件が決まっていなくても大丈夫です。練習用に、クラウドワークスやランサーズで「ライティング」と検索して出てくる案件を1つ選んでみてください。

---

## 提案文の「型」を知る

料理でいえば、どんな名シェフもまず「基本のレシピ」を押さえています。提案文にも通るための基本レシピがあります。

**通る提案文の3パーツ**:

1. **挨拶＋応募の意思** → 「はじめまして。〇〇の案件に興味を持ちました」
2. **あなたの強み（具体的に）** → 経験・実績・学んでいることを数字や事実で書く
3. **次のアクションの提案** → 「まずは15分ほどお話できれば幸いです」

### 良い例

> 〇〇様、はじめまして。〇〇の案件に興味を持ちました。
> 私はブログ記事の執筆を2年間行っており、SEO（＝検索サイトで上位に表示されるための工夫）の知識もあります。前の案件では、記事の閲覧数を3ヶ月で2倍に増やしました。
> ぜひ一度、案件の詳細についてお話しできれば幸いです。よろしくお願いいたします。

### 悪い例

> こんにちは。この仕事に応募します。がんばります。よろしくお願いします。

→ 悪い例は「誰でも書ける文章」です。具体的な強みがなく、クライアントが「この人でなくてもいい」と感じてしまいます。

## AIに下書きを作ってもらう

では、AIに提案文の下書き（＝たたき台）を作ってもらいましょう。ここがこのレッスンのメインです。

### ステップ1: 募集文を読み込ませる

AIチャットツールを開き、次のメッセージをコピーして送ります。`（ ）`の中身はあなたの情報に書き換えてください。

```
以下の案件に応募するための提案文を作ってください。

【案件内容】
（ここに募集文をそのまま貼り付ける）

【私の経験・スキル】
（例：ブログ記事の執筆経験2年、SEOの基礎知識あり）
（例：経験がない場合 → 「現在Webライティングを学習中です。毎日1記事を練習で書いています」）

【条件】
・200〜400字くらい
・丁寧で誠実な印象にする
・具体的な実績や強みを1つ以上入れる
・案件内容に触れて「ちゃんと読んだ」と伝わるようにする
```

> **何をしているか**: AIに「募集文の内容」「あなたの情報」「完成イメージの条件」の3つを伝えることで、案件にぴったりの提案文を生成してもらっています。

![AIチャットでの提案文生成画面](/lesson-assets/atom.ai-freelancer.proposal-writing-ai/screen_capture.png)

### ステップ2: 出力が微妙なら追加で指示する

AIの最初の出力がイマイチだと感じたら、そのまま次のメッセージを送ってください。AIは直前の会話を覚えているので、最初から書き直す必要はありません。

**よく使う追加プロンプト（＝AIへの指示）の例**:

| やりたいこと | AIに送るメッセージ |
|---|---|
| 文章が長すぎる | 「200字以内に短くしてください」 |
| 堅すぎる | 「もっとやわらかい、話しかけるような文体にしてください」 |
| 強みが弱い | 「私の強みとして〇〇を追加して、もう一度書き直してください」 |
| 案件への理解が足りない | 「募集文の〇〇という部分に触れた内容を加えてください」 |
| 別パターンがほしい | 「同じ条件で、違う切り口の提案文をもう1パターン作ってください」 |

> **たとえ話**: AIとの会話は、美容院での注文と似ています。「もう少し短く」「前髪はこのくらいで」と伝えれば、どんどん理想に近づきます。1回で完璧を求める必要はありません。

## 提案文を磨き上げる

AIが作った下書きは、**そのまま送信しないでください**。AIの出力はあくまで下書きです。次の4つのポイントを確認して手直しします。

### 確認チェックリスト

| チェック項目 | なぜ重要か | 確認の仕方 |
|---|---|---|
| 案件の内容に合っているか | クライアントは「自分の案件を読んでくれたか」を見ています | 募集文のキーワードが提案文に含まれているか確認 |
| 具体的な数字や実績が入っているか | 「がんばります」より「〇〇の成果を出しました」が信頼されます | 数字が最低1つ入っているか確認 |
| 自然な日本語か | AIは時々不自然な表現を作ることがあります | 声に出して読んでつっかからないか確認 |
| 200〜400字に収まっているか | 長すぎると読んでもらえません | 文字数カウントツールで確認するか、AIに「この文章は何文字？」と聞く |

### 良い手直しの例

**AI出力**: 「私は熱意を持って取り組みます」

**手直し後**: 「前の案件では納期を1週間前倒しで納品し、クライアントから継続依頼をいただきました」

→ 「熱意」のような抽象的な言葉を、具体的な事実に置き換えると説得力が増します。

### 悪い手直しの例

AI出力を一切変更せずそのまま送信する → AIらしい不自然な言い回しが残り、クライアントに「コピペだな」と見抜かれる可能性があります。

---

## 送信前に最終確認する

提案文を送る前に、次の3ステップで最終チェックしましょう。

1. **声に出して読む** → スラスラ読めれば自然な文章です。つっかかる部分は表現を直しましょう
2. **AIにチェックしてもらう** → 完成した提案文をAIに送り、次のように聞きます:

```
この提案文を、クライアント目線でチェックしてください。
改善できるポイントがあれば教えてください。

（ここに完成した提案文を貼る）
```

3. **最後にもう一度、募集文と見比べる** → クライアントが求めていることと、あなたの提案がずれていないか確認します

---

## つまずきやすいポイントと解決策

| つまずき | 解決策 |
|---|---|
| AIが長すぎる文章を作る | 「200字以内で簡潔に書き直してください」と指示し直す |
| どんなスキルを書けばいいかわからない | 募集文にあるキーワード（例：「SEO」「WordPress」）をそのまま使い、「〇〇について学習中です」と書く |
| AIの文章がなんだか不自然 | 「もっと自然な日本語で、友達に話しかけるように書き直してください」と伝える |
| 実績が特にない（初心者の場合） | 「現在〇〇を学習中で、毎日〇〇を練習しています。3ヶ月で〇〇を習得する計画です」と意欲と具体的な行動を書く |
| 使っているAIツールが英語で返答する | 「日本語で回答してください」と最初に一言添える |
