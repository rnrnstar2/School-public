# トレンドの筋書きを説明する

## はじめに

旅行の写真を見せてもらったとき、「売上どうだった？」と数字だけ言われるより、「この時期は新商品がSNSでバズって、売上がぐっと伸びたんだよ」とストーリーを聞くほうが、頭にスッと入ってきますよね。

データのトレンド（＝時系列で見た変化の傾向）も同じです。数字の上がり下がりを並べるだけでは伝わりません。「なぜそうなったのか」という筋書き（＝ストーリー）を添えることで、聞き手が一瞬で理解できるようになります。

このレッスンでは、AIチャットツール（ChatGPTやClaudeなど）を使って、データのトレンドに分かりやすい筋書きをつける方法を15分で身につけます。完了すると、「トレンドを3文で説明するドキュメント」という成果物が手元に残ります。

## 前提を確認する

- 手元に、説明したいトレンドがあること（例：「先月比で売上が20%増えた」「ここ3ヶ月で離脱者が増えている」など）
- AIチャットツール（ChatGPT、Claude など）をブラウザで開いて使える状態であること

プログラミングの知識は一切不要です。すべてAIとの対話だけで進めます。

## トレンドの「主役」を一つ決める

まずは、あなたが説明したいトレンドの中心にある「主役」を一つに絞りましょう。旅行でいうと「この旅行で一番印象に残った場面」を一つ選ぶ感覚です。

次の3つの要素をそろえて、メモに書き出してください。

- **何が** 変わったか（売上・アクセス数・会員数など）
- **いつからいつまで** 変わったか（期間）
- **どれくらい** 変わったか（増加率・減少数など）

**良い例**: 「2024年4月〜6月の間に、オンラインショップの週間売上が前年同期比15%増加した」
→ 数字・期間・変化幅がすべて入っているので、このあとAIに渡しやすい

**悪い例**: 「売上が上がった」
→ いつ・どれくらい上がったのかが不明で、AIも的確な回答を返せない

## AIに「なぜ？」を聞いて理由を3つ出す

主役が決まったら、AIツールに質問して筋書きの種を探ります。料理でいうと、レシピの「隠し味」をAIに提案してもらうイメージです。

次のプロンプト（＝AIへの指示文）をコピーして、`{あなたのトレンド情報}` の部分を先ほど書き出した主役に書き換えて、AIに送ってください。

```text
あなたはデータ分析のアドバイザーです。
以下のトレンドについて、考えられる理由を3つ、それぞれ1〜2文で挙げてください。
業界の一般的な要因も考慮してください。

トレンド：{あなたのトレンド情報}
```

> **ポイント**: 「あなたはデータ分析のアドバイザーです」のように役割を指定すると、AIがより的確な回答を返してくれます。これを「ロール指定」と呼びます。

AIが3つの理由を返してくれたら、あなたの経験や感覚に一番しっくりくるものを**一つだけ**選んでください。迷ったら直感で決めて大丈夫です。あとから変えられます。

## 理由を裏付ける「エピソード」を探す

選んだ理由をよりリアルにするために、小さなエピソード（＝具体的な出来事やデータポイント）を一つ添えます。お弁当でいう「メインのおかずに添える付け合わせ」です。

```text
次のトレンドと理由について、裏付けになるような具体的なエピソードを一つ提案してください。
実際にありそうなリアルな内容にしてください。

トレンド：{主役のトレンド}
理由：{選んだ理由}
```

例えば、「新商品のレビューがSNSで拡散された」という理由なら、「5月の大型連休中に〇〇というインフルエンサーが紹介して、その週の注文が平日の3倍に跳ね上がった」といったエピソードが出てきます。

> **注意**: AIが出したエピソードはあくまで「たたき台」です。あなたが持っている実データや経験に照らして、事実と合っているか必ず確認してください。

## 筋書きを3文でまとめる

ここまで集めた材料を使って、最終的な筋書きを3文で仕上げます。ここでもAIに手伝ってもらいましょう。

```text
次の情報を使って、トレンドの筋書きを「事実→理由→エピソード」の順番で、ちょうど3文にまとめてください。
専門用語を使わず、初めて聞く人でも理解できる平易な表現にしてください。

トレンド：{主役のトレンド}
理由：{選んだ理由}
エピソード：{見つけたエピソード}
```

AIが出力した3文を読んで、次のポイントをチェックしてください。

**良い例**: 「2024年4月〜6月にオンラインショップの週間売上が前年同期比15%増加した。理由は、4月中旬に発売した新商品のレビューがSNSで拡散されたためだ。特に5月の大型連休中は、インフルエンサーの紹介動画がきっかけで注文が平日の3倍に達した。」
→ 事実→理由→エピソードの順で、読み手が自然に追える

**悪い例**: 「売上が上がりました。多分SNSのせいです。あと連休も関係あるかも。」
→ 根拠が曖昧で、推測ばかり。数字や期間がない

## 筋書きを検証する

完成した筋書きが本当に使えるか、次の3点でセルフチェックします。テスト前に味見をする感覚です。

| チェック項目 | 確認方法 |
|---|---|
| **事実と一致するか** | 数字と期間が元のデータと同じか見返す |
| **飛躍がないか** | 理由がデータから読み取れる範囲内か確認する。「宇宙人が来たから売上が伸びた」はさすがに飛躍しすぎです |
| **他人に伝わるか** | 家族や同僚に読んでもらい、「一発で意味が分かったか」を聞く。口頭で15秒以内に説明できたらOK |

3つともクリアしたら、その3文をテキストファイルやドキュメントに保存してください。これがこのレッスンの成果物です。

## つまずき対策

| つまずきポイント | 対策 |
|---|---|
| AIが理由を出してくれない | プロンプトに「業界の一般的な要因も含めて考えてください」と追加する。それでもダメなら、トレンド情報にもっと具体的な数字を追加してみる |
| 筋書きが長くなりすぎる | 「必ず3文以内でまとめてください。それ以上は書かないでください」と強めに指定してAIに書き直してもらう |
| 事実と推測の区別がつかない | 推測部分には「〜と考えられる」と自分で書き添える。AIの出力をそのまま使わず、自分の言葉で確認する |
| どの理由を選べばよいか迷う | 直感で一番しっくりくるものを選ぶ。完璧でなくてOK。あとから差し替えもできる |
| AIの回答が的外れに感じる | 「もう少し〇〇の観点で考えてください」と追加で指示を出す。一度で完璧な回答が出なくても、対話を重ねれば精度が上がる |

## 完了を確認する

以下の3つがすべて「はい」なら、このレッスンは完了です。

- [ ] 3文の筋書きドキュメントが手元に保存されている
- [ ] 筋書きの中に、数字・期間・具体的エピソードが含まれている
- [ ] 家族や同僚など、第三者に読んでもらって「分かった」と言ってもらえた（または自分で15秒以内に口頭説明できた）
