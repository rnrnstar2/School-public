# 複数案件を並行運用する

フリーランスとして仕事が増えてくると、「あのクライアント（＝依頼主）の納品いつだっけ？」「こっちの修正まだだった…」と頭の中がぐちゃぐちゃになりがちです。このレッスンでは、AIを味方につけて複数の案件を混乱なく回す方法を身につけます。

例えて言うなら、レストランの厨房で3品の料理を同時に作るようなものです。鍋を火にかけながら野菜を切り、ソースを煮詰める——それぞれの進捗（＝どこまで進んだか）を一目で把握できれば、焦がす心配はありません。AIは「火加減を見張ってくれるタイマー」のような存在です。

![並行運用の全体像](/lesson-assets/atom.ai-marketer.multi-client-operate/diagram.png)

## 前提を確認する

- あなたはすでに1つのクライアントの仕事を最後までやり遂げた経験があること
- ChatGPTやClaudeなど、AIチャットツールの基本的な使い方を知っていること

まだ1件も仕事を完了していない場合は、まず1件やり遂げてからこのレッスンに戻ってきてください。

## ステップ1: 全案件のタスクをAIで洗い出す

まずは「全部で何があるか」を見える形にします。ここでAIに手伝ってもらいましょう。

**AIへのプロンプト例（ChatGPTやClaudeにそのままコピペしてください）:**

```
私はフリーランスでマーケティングの仕事をしています。
現在、以下のクライアントを担当しています。

- クライアントA（飲食店）：ブログ記事の企画・執筆、月4本、毎週金曜納品
- クライアントB（美容サロン）：Instagram投稿の作成、週3回
- クライアントC（オンラインスクール）：メルマガの配信準備、月2回

それぞれのクライアントについて、今週やるべきタスクを
表形式で整理してください。優先度と締め切りも入れてください。
```

上のプロンプトの「クライアントA」「クライアントB」の部分を、あなたの実際の案件に書き換えて使ってください。

**良い例:**
- クライアントごとに「何を・いつまでに・どのくらいの量」を具体的に書く
- 「ブログ記事の企画・執筆、月4本、毎週金曜納品」のように数字を入れる

**悪い例:**
- 「クライアントAの仕事」だけで中身を書かない
- 「色々やらなきゃ」と頭の中だけで管理する

AIが出してくれた表を、スプレッドシート（＝Googleが提供する無料の表計算ソフト）やNotion（＝情報をまとめる無料ツール）にコピーして保存しましょう。

## ステップ2: 1日のスケジュールをAIと一緒に組む

次に、「いつ何をやるか」を決めます。ここもAIに相談できます。

**AIへのプロンプト例:**

```
以下のタスクを、今日の作業スケジュールに組み込んでください。
作業可能な時間は10:00〜16:00で、昼休みは12:00〜13:00です。

- クライアントAのブログ記事下書き（AIで生成→確認で約45分）
- クライアントBのInstagram投稿3本作成（AIで生成→画像選定で約30分）
- クライアントCのメルマガ構成案（AIで生成→修正で約30分）

各タスクの間に10分の休憩を入れてください。
AIで下書きを作る時間と、自分で確認・修正する時間を
分けてスケジュールに入れてください。
```

**ポイント: 「AI作業」と「確認作業」を分ける**

AIに下書きを作ってもらう時間と、あなた自身が内容を読んで修正する時間は別物です。必ず両方の時間を確保してください。

**良い例:**
- 10:00〜10:30 クライアントAの記事をAIで下書き
- 10:30〜10:45 下書きを読んで修正する
- 10:55〜11:25 クライアントBのSNS投稿をAIで作成

**悪い例:**
- 「今日中に全部やる」と決めて、具体的な時間を割り振らない
- AIに作ってもらったものを確認せずそのまま納品する

## ステップ3: AIで進捗メモと報告文を作る

1日の終わりに5分だけ、各クライアントの進み具合を振り返ります。この振り返りが複数案件を回す上で**最も大切な習慣**です。

**AIへのプロンプト例:**

```
今日の作業結果を報告用にまとめてください。

- クライアントA：ブログ記事の下書き完了、明日クライアントに確認依頼を送る
- クライアントB：Instagram投稿2本完成、残り1本は明日午前中に作成
- クライアントC：メルマガの構成案を作成済み、本文はまだ

それぞれのクライアントに送る短い進捗報告メッセージも作ってください。
カジュアルで丁寧なトーンでお願いします。
```

こうすると、振り返りと報告文の作成が一度にできます。料理で言えば、すべての鍋の火加減を一度にチェックするようなものです。

## つまずきやすいポイントを知る

| よくある失敗 | 対処法 |
|---|---|
| 全部同時に始めようとして混乱する | まず1つのクライアントの最初のタスクを完了させてから次に進む |
| AIの出力をそのまま納品してしまう | 必ず自分の目で読み、事実関係やトーンを確認してから提出する |
| クライアントへの連絡を後回しにする | 1日の終わりの振り返り時に報告文をAIで作り、すぐ送る |
| 案件が増えるたびに管理表を作り直す | 最初から「クライアント名・タスク・締切・状態」の4列で統一しておく |

## 完了を確認する

以下の3つすべてにチェックが入れば、このレッスンは完了です。

- [ ] スプレッドシートやNotionに、担当中の全クライアントのタスク・締切・状態が一覧できる表がある
- [ ] 明日の1日スケジュールに、各クライアントの「AI作業」と「確認作業」の両方が時間付きで入っている
- [ ] AIを使って、今日の進捗報告メッセージを1件以上作成できた

これらが確認できたら、あなたは「AIを使って複数案件を並行して回す」基礎を身につけたと言えます。スクリーンショットを撮って、自分の成果として保存しておきましょう。
